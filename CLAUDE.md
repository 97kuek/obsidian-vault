# Obsidian Vault ガイド

このファイルはClaude Codeがこのvaultで作業する際の規約・ルールをまとめたもの。
ノートの作成・編集・移動を行う際は必ずここを参照すること。

---

## Vault概要

PARAメソッド（Projects / Areas / Resources / Archives）をベースに構成された研究・学習用vault。

---

## フォルダ構成

| フォルダ | 用途 | 注意点 |
|---|---|---|
| `00_Inbox/` | 未整理ノートの一時置き場 | 定期的にレビューして振り分ける |
| `10_Projects/` | 期限・ゴールのあるアクティブな作業 | 完了したら `40_Archives/` へ移動 |
| `20_Areas/` | 継続的に管理するトピック | 終わりのない継続的関心事 |
| `30_Resources/` | 将来再利用できる参考資料・論文 | 特定プロジェクト専用でも可 |
| `40_Archives/` | 完了・不要なノート | 削除せずアーカイブ |
| `99_Templates/` | Templaterテンプレート | 実ノートではない |

## ファイル命名規則

### 通常ノート・実験ノート
```
YYYYMMDD_タイトル.md
```
例：`20260511_音源分離実験用スクリプトの作成.md`

### MOCファイル
```
【MOC】タイトル.md
```
例：`【MOC】プロジェクト研究A.md`

### 論文読書メモ
```
論文タイトル略称_メモ.md
```
例：`TF-Locoformer_メモ.md`

### テンプレートファイル（99_Templates/内のみ）
```
用途.md
```
例：`論文ノート用.md`、`実験ノート用.md`

## テンプレート使い分け

| テンプレート | 使うタイミング |
|---|---|
| `【MOC】.md` | 新しいトピックのMOCを作るとき |
| `プロジェクト用.md` | 10_Projects内の個別ノート |
| `論文ノート用.md` | 論文を読んで読書メモを作るとき |
| `実験ノート用.md` | 実験・作業ログを記録するとき |
| `授業ノート用.md` | 授業・勉強会のノート |

テンプレートファイルには Templater 変数（`<% tp.* %>`）が含まれる。
**実ノートにはテンプレート変数を残さず、必ず実際の値に置換すること。**

```
Home
├── 10_Projects/
│   └── 【MOC】10_Projects

│
├── 20_Areas/
│   └── 【MOC】20_Areas
│       ├── 【MOC】ソフトウェア開発
│       └── 【MOC】機械学習・深層学習
```

各MOCには必ず「関連MOC・上位MOC」セクションを設け、親MOCへのリンクを記載する。

## タグ体系

新規ノート作成時は以下のタグリストから選ぶ。独自タグの追加は最小限にし、追加した場合はここに追記する。

| カテゴリ | タグ | 用途 |
|---|---|---|
| システム | `MOC` | Map of Content |
| システム | `inbox` | Inbox未整理ノート |
| システム | `archive` | アーカイブ済み |
| ノート種別 | `paper` | 論文読書メモ |
| ノート種別 | `experiment` | 実験ノート |
| ノート種別 | `lecture` | 授業・勉強会ノート |
| ノート種別 | `project` | プロジェクトノート |
| ノート種別 | `dataset` | データセット情報 |
| ノート種別 | `corpus` | コーパス情報 |
| トピック | `機械学習` | 機械学習全般 |
| トピック | `深層学習` | 深層学習全般 |
| トピック | `音源分離` | 音源分離・音声処理 |
| トピック | `ソフトウェア開発` | ソフトウェア開発全般 |
| モデル | `transformer` | Transformerアーキテクチャ |
| モデル | `bert` | BERT |
| モデル | `ai` | AI全般（`AI`は使わない） |

**ルール：** タグは小文字英語または日本語に統一。大文字・混在は使わない（`AI` → `ai`）。

## リンクの使い方

- ノート間は Wikiリンク `[[ファイル名]]` で接続する
- MOCは配下ノートへのリンクを集約するハブとして機能する
- 各ノートの「関連MOC・上位MOC」セクションで親MOCにリンクする
- 論文PDFへは**読書メモ経由**でリンクする（`[[論文名_メモ]]`）。PDFへの直リンクは避ける
- Dataviewクエリの `FROM` には具体的なフォルダパスを指定する（`FROM [[]]` は無効）

## Git作業フロー

Claudeに複数ファイルの作成・削除・リネームを依頼する前に必ずcommitすること。

```bash
# 作業前スナップショット
git add -A && git commit -m "作業前スナップショット"

# 作業後に差分確認
git diff HEAD
git status
```
