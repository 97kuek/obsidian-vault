
### 経路制御の概要

#### ルータの役割 ＝ 経路制御

1. **パケットフォワーディング**：受信したパケットのヘッダにある宛先IPアドレスと、**経路制御表（ルーティングテーブル）** を照らし合わせて、次のルータへ転送する。ただし「転送するだけ」で、相手に届いたかは確認しない（＝コネクションレス型通信）
2. **経路制御表の作成・管理**：そもそもどこへ転送すべきかを判断する表を、作って維持する

#### ホップバイホップ・ルーティング

- **ある地点からある地点までの経路は、事前に全部決まっているわけではない**
- パケットを受け取ったルータが、その都度自分の経路制御表を見てnext hopだけを決めて転送する
- これをバケツリレーのように繰り返して宛先まで運ぶ
- 通信の途中で経路が変わることもある

#### 経路制御表をどう作るか

|方式|作り方|長所|短所|
|---|---|---|---|
|**スタティック**（静的）|管理者が手動で固定設定|小規模なら一番簡単|規模拡大で更新作業が大変／障害時は手作業／人的ミスのリスク|
|**ダイナミック**（動的）|ルータ同士が情報交換して自動生成|スケーラブル／障害時に迂回路を自動で有効化|仕組みが複雑|

#### AS（自律システム）と IGP / EGP

- ASとは「経路制御のポリシーを決めて運用する範囲」のこと
- ISPや大学などの組織単位
- それぞれに一意の **AS番号** が振られる（例：早稲田大学は **AS17956**、`133.9.0.0/16` をアナウンス）
- **インターネットとは、無数のAS（6万8千以上）が相互接続してできた「ネットワークのネットワーク」**

経路制御プロトコルは、ASの「内」か「間」かで2種類に分かれます。これは試験頻出の対比です。

||IGP|EGP|
|---|---|---|
|正式名|Interior Gateway Protocol|Exterior Gateway Protocol|
|範囲|AS **内部** の経路制御|AS **間** の経路制御|
|単位|IPアドレス（細かい・ミクロ）|AS番号（粗い・マクロ）|
|代表例|RIP, OSPF|BGP|

### 最短経路問題

- ダイナミックルーティングでは最適（＝最短）な経路を計算する
- グラフ理論の **最短経路問題** そのもの
- グラフの2頂点間で、通る辺の**重み（コスト）の合計が最小**になる経路を求める
#### Bellman-Ford アルゴリズム（→ RIP が採用）

- ある1頂点から**他のすべての頂点**への最短距離を計算
- 計算量は **O(VE)**（V=頂点数, E=辺数）
- 特徴：**負の重みを持つ辺があっても扱える**（汎用性が高い）

#### Dijkstra アルゴリズム（→ OSPF が採用）

- 同じく最短経路を求めるが、計算量は **O(E + V log V)** と効率的
- ただし **負の重みには対応できない**

#### 「情報源」表から「Next hop」表への変換

最短経路計算の結果は「直前のノード（情報源）」付きで出ますが、実際の経路制御表に必要なのは **「最初の一歩＝next hop」** です。たとえば Bellman-Ford の例で B への経路は A→C→D→E→B なので、**A から見た next hop は C**。同様に D, E, F も全部最初の一歩が C なので、next hop はすべて C になります。

#### 両者の比較

- アルゴリズム的には似ている
- 計算コストは Dijkstra が有利
- 1反復あたりのオーバーヘッドが小さいため実用上は Bellman-Ford が優れるという主張もあり、**実質大差はない**とされている
- 違いとして押さえるべきは「**Bellman-Ford は負の重みOK、Dijkstra は不可**」「Dijkstra の方が計算量的に速い」の2点

### 経路制御プロトコル

- ルータが相互に通信して経路情報を交換し、表を自律的に作る手順
- 障害や設定変更で無効になった経路に動的に対応＝**迂回**できる
- インターネットの耐障害性・高可用性を支える

#### 3つの分類

|型|交換する情報|長所|短所|
|---|---|---|---|
|**距離ベクトル型**|距離（ホップ数）＋方向|計算負荷が低い|収束が遅い／障害に弱い|
|**リンク状態型**|ネットワーク全体のマップ（接続状態）|速く同期できれば安定|計算負荷が高い|
|**パスベクトル型**|距離＋方向＋**通過するAS番号の列**（パス）|ループを作らない／ポリシーを柔軟設定|パスが長いと交換データが肥大|

#### 主要プロトコルまとめ

|プロトコル|IGP/EGP|下位レイヤ|方式|アルゴリズム|
|---|---|---|---|---|
|**RIP**|IGP|トランスポート層（UDP）|距離ベクトル|Bellman-Ford ベース|
|**OSPF**|IGP|IPを直接利用（プロトコル番号89）|リンク状態|Dijkstra|
|**BGP**|EGP|トランスポート層（TCP）|パスベクトル|—|

#### RIP（Routing Information Protocol）

- 距離ベクトル型。Bellman-Ford ベースで**分散的に**動作
- 経路情報を定期的にブロードキャスト
- 距離の単位は**ホップ数**（通過ルータ数）
- 候補が複数あればホップ数が小さい方を選ぶ
- 課題としては以下の通り
	- ①収束が遅い
	- ②**ホップ数しか見ない**ので帯域が狭いリンクも選んでしまう
	- ③仕様上 **ホップ数15が上限**
	- ④受信した経路情報も含めて流すので規模拡大で情報が肥大化

### OSPF（Open Shortest Path First）

- リンク状態型
- ネットワークのトポロジー全体を共有して表を作る
- リンクに重み（メトリック、通常は**帯域幅が大きいほど小さいコスト**）を付け、Dijkstra で重み和最小の経路を選ぶ
- HELLOパケットで隣接関係を確立 → リンク情報（**LSA**）を通知・伝播 → 全体で同じ**リンク状態データベース（LSD）**を共有 → Dijkstra で経路計算。
- **「エリア」による階層化**
	- ネットワークが大きいと LSD が肥大化して計算が大変になるため、複数のネットワーク/ホストをまとめて「エリア」を定義し計算負荷を軽減
	- AS内には必ず1つ以上の**バックボーンエリア**が必要で、各エリアはそこに接続されていなければならない
	- エリア内のルータは、エリア外については**距離だけ**を把握しトポロジーは持たない（＝情報削減）

### BGP（Border Gateway Protocol）

- パスベクトル型
- **EGP**として使用
- AS番号（16ビット）で経路制御し、メッセージは TCP で送る
- 距離でもリンク状態でもなく「**通過するAS番号のリスト（ASパス）**」で経路を表現（例：AS17956→AS2907→AS2914→AS3356）。
- AS間の接続は組織間の**契約**に基づく：自分のために相手と対等に繋ぐのが「**ピアリング**」、他ASへパケットを通してもらうのが「**トランジット**」。
- **ポリシールーティング**：基本は短いパスが選ばれるが、回線料や安定性などポリシーに応じて変更可能ASパスを持つことで**ループを防げる**のが大きな利点

### エンドから観察するインターネット経路 

- **traceroute**（Windowsは `tracert`）は、宛先までに通過するルータの一覧を表示するツール
- Van Jacobson が1987年に開発し、経路診断や遅延測定に使える

#### アルゴリズム

- **TTL（Time To Live）はルータを通過するたびに1ずつ減り、0になるとそのルータがパケットを破棄して送信元に「ICMP Time Exceeded（時間超過）」エラーを返す**
- これを逆手に取る

1. まず **TTL=1** のパケットを送る → 最初のルータAでTTLが0になり破棄され、AからICMPエラーが返る → **1番目のルータ＝A** だと判明。
2. 次に **TTL=2** のパケットを送る → Aを通過してTTL=1、次のルータBで0になり破棄され、BからICMPエラーが返る → **2番目＝B** と判明。
3. TTLを 3, 4, … と1ずつ増やしながら繰り返すと、**通過する全ルータが手前から順に判明**し、宛先に到達したら終了。

- 各ステップで往復時間も測れるので、どこで遅延が大きいかも把握できる
