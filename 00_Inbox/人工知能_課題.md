
## Topic 2
#### 8パズル・15パズルの表現

>8パズルの状態のデータ構造、オペレータの動き、ゴール検査の条件を書け。
>15パズルではどうか。

状態は「9区画それぞれに何があるか」を表せばよく、長さ9の配列（または3×3の二次元配列）で表現します。各要素にタイル番号1〜8を入れ、空白を0などの特別値で表します。例：`[5,7,8,4,0,2,6,1,3]`。空白位置を別途インデックスで持っておくとオペレータが書きやすくなります。

オペレータは「空白を上下左右に動かす（隣のタイルと空白を交換する）」操作で、空白のインデックス`b`に対して、上=`b-3`、下=`b+3`、左=`b-1`、右=`b+1`の位置の値と交換します。ただし盤の端では不可能な方向があり、左右移動は行をまたがないという境界条件を課します（`b%3==0`なら左不可、`b%3==2`なら右不可、`b<3`なら上不可、`b>5`なら下不可）。

ゴール検査は、状態配列がゴール配列`[1,2,3,4,0,5,8,7,6]`（スライドのゴール）と一致するかの比較です。15パズルは4×4=16区画、タイル1〜15＋空白で、配列長16、行をまたぐ判定が`b%4`、上下移動が`±4`になるだけで構造は同じです。

#### ハノイの塔の禁止オペレータ条件

状態を `(x1, x2, x3)`（小さい順に各円板のいる塔の番号）としたとき、`(2,1,1)→(2,2,1)` のような移動が禁止される一般条件を、n枚の場合も含めて述べよ。

**解答**

状態を「円板iが乗っている塔の番号 xᵢ」のベクトル `(x1, …, xn)`（添字iが小さいほど小さい円板）とします。1手の移動とは「ある塔の一番上の円板を別の塔へ移す」ことなので、許されるのは次を満たす移動だけです。

まず「動かす円板kは、その塔で一番上にある」必要があります。円板kが塔 `xk` の一番上であるとは、**kより小さい円板がどれも塔 `xk` にいない**こと、すなわち `xj ≠ xk (∀ j < k)` です。次に「移動先の塔 `d` で、kより小さい円板の上に乗せない」必要があり、これは **`xj ≠ d (∀ j < k)`** です。

したがって禁止されるのは、(a) 動かそうとする円板の上により小さい円板が乗っているとき（`∃ j<k, xj = xk`）、または (b) 移動先により小さい円板がいるとき（`∃ j<k, xj = d`）です。例の `(2,1,1)→(2,2,1)` は、小円板1を塔1から塔2へ移そうとしていますが、移動先の塔2にはすでに中円板2（=より大きいので問題ない）…と思いきや、この表記では「小さい円板を大きい円板の上に置く」ことが許され、逆の「大きい円板を小さい円板の上に置く」操作が禁止対象です。一般化すれば、**より小さい円板の上により大きい円板を置く移動がすべて禁止**で、上の(a)(b)がその形式的条件です。

#### ゴールへの経路復元

>Openリストを導入した探索で、あるノードnがゴールと判明したとき、そこへ至る経路をどう得るか。

- Openリスト（およびClosedリスト）に要素を入れるとき、ノードだけでなく **`(子ノード, 親ノード)` の対**を保持しておくのが鍵
- ゴールnが見つかったら、まずnの親pを引き、次にpの親、さらにその親…と**親ポインタを初期状態（親がnull）まで遡り**、得られた列を逆順に並べれば、初期状態からゴールへの経路（オペレータの列）になる
- 各ノードについて親は1つ記録されているので、この遡りは一意に定まる

#### 各探索法での探索順序

下図の状態空間グラフ（初期状態A、ゴールE、数値はコスト）を各探索法で探索したときの状態の探索（展開）順序を書け。双方向は1ステップずつ交互。

辺とコスト：A–B:2, A–F:1, B–C:3, B–F:2, B–G:1, C–H:2, D–G:2, F–G:3, G–H:3, H–E:2, H–I:1

**解答**

隣接関係（昇順）：A:{B,F} / B:{A,C,F,G} / C:{B,H} / D:{G} / E:{H} / F:{A,B,G} / G:{B,D,F,H} / H:{C,E,G,I} / I:{H}。先に最適経路を確認しておくと、Dijkstraより **A→B→G→H→E（コスト 2+1+3+2＝8）が最短**です。

**幅優先探索**：浅い層から順に展開。 A → B → F → C → G → D → H（ここでHの子としてゴールEが生成され終了） （コスト一定でないため、見つかる経路A-B-?-H-Eが最適とは限らない点に注意）

**均一コスト探索**：累積コスト最小を取り出す（ゴールは取り出し時に判定）。 A(0) → F(1) → B(2) → G(3) → C(5) → D(5) → H(6) → I(7) → E(8) で終了。 得られる経路は **A→B→G→H→E、コスト8（最適）**。

**深さ優先探索**：スタックで最深を優先（子は昇順に探索）。 A → B → C → H（Hの子にゴールEが現れ終了）。 経路 **A→B→C→H→E、コスト9（最適ではない）**。深さ優先が最適を保証しない好例です。

**深さ制限探索**：上の深さ優先を上限limitで打ち切るだけ。ゴールEの最浅深さは4（A-B-C-H-E あるいは A-B-G-H-E）なので、limit≦3では解なし、limit≧4で深さ優先と同じく A→B→C→H→E。

**反復深化探索**：limit=0,1,2,…と増やす。 limit0：A／limit1：A,B,F／limit2：A,B,C,G,F／limit3：A,B,C,H,G,…（Hの子Eはd=4でlimit外なので未発見）／**limit4：A→B→C→H→E で発見**。深さ最小（4手）の解を返します。

**双方向探索**：前向きBFS（Aから）と後向きBFS（Eから）を1手ずつ交互。 前向き：Aを展開（縁{B,F}）→ 後向き：Eを展開（縁{H}）→ 前向き：Bを展開（縁に{C,G}追加）→ 後向き：Hを展開（縁に{C,G,I}追加）。この時点で前向き側と後向き側の双方にC・Gが現れ**交差**。Gで合成すると **A→B→G→H→E（コスト8、最適）**。展開順は A, E, B, H（で交差）。

#### 反復深化が不利になる木／深さ優先が有利な条件

> 反復深化が深さ優先よりかなり効率が悪くなるのはどんな探索木か。
> 逆に深さ優先が有利になる条件は。

反復深化の総展開ノード数は、1回の深さ優先に対して**おおよそ b/(b−1) 倍**のオーバーヘッドを持ちます。したがって**分岐数bが小さく（1に近く）木が深いとき**、上層を何度も探索し直すコストが相対的に大きくなり、反復深化は深さ優先よりかなり不利になります（b=2で約2倍、b=10では約1.1倍に過ぎず、bが大きいほど大半のノードは最深層に集中するので繰り返し分は無視できる）。

逆に深さ優先が有利なのは、(a) 木が一本道に近く（bが小さく）非常に深い、(b) ゴールが深い位置にあり、(c) 最初に選ぶ枝が当たりやすい（探索順序が良い）場合です。このとき深さ優先は再探索なしに一気にゴールへ到達できます。ただし無限の深さや「赤がすべてゴール」のような配置では最短を逃すリスクと裏表である点に注意します


#### その木での反復深化と幅優先の計算量・メモリ量

**解答**

深く分岐数の小さい木を想定すると、**計算量はどちらも O(b^d)** のオーダー（反復深化は前問の繰り返しオーバーヘッドが乗るが、オーダーは同じ）です。決定的に異なるのは**メモリ量**で、幅優先は次層展開のため各層全体を保持し **O(b^d)** と指数的に膨張します（前掲の表でd=10→1TB、d=12→111TBの世界）。一方、反復深化は現在の経路上のノードだけ保持すればよく **O(bd)** と線形に収まります。よってこの種の木では「時間は同程度・メモリは反復深化が圧倒的に有利」となり、反復深化が選ばれます。

### 問題7（スライド16）｜課題2-3②でのOpen/Closedリストの変化

特定の探索戦略を決め、課題2-3②のグラフでOpen/Closedリストの変化を追え。

**解答（幅優先探索の場合）**

要素は`(ノード 親)`で記します。

|展開ノード|展開後のClosed|展開後のOpen（縁）|
|---|---|---|
|開始|{}|(A null)|
|A|{A}|(B A)(F A)|
|B|{A,B}|(F A)(C B)(G B)|
|F|{A,B,F}|(C B)(G B)（Fの子G,Bは既出）|
|C|{A,B,F,C}|(G B)(H C)|
|G|{A,B,F,C,G}|(H C)(D G)（Gの子HはOpenに既存）|
|D|…,D|(H C)|
|H|…,H|Hの子にE（ゴール）→終了|

Closedは展開済みノードが単調に増え、Openは「縁＝まだ展開していないノード」を親付きで保持しています。問題3で述べた親ポインタの遡りで、終了時にHの親C…と辿れば経路が復元できます。均一コストやA*なら、Open要素にコストcも併記し、Step4の置き換え規則（より低コストが来たら更新／Closedからの再オープン）に従って変化を追うことになります。

#### 地下鉄の例で最短経路が見つかることを考えよ

> 各駅の地図上の直線距離をhとするA\*が、最短経路を見つけることを示せ。

- A\*が最適解を返す十分条件は **hが許容的（h(n) ≤ 真の残コスト）** であること
- 地下鉄では、ある駅nから目的地までの実際の路線に沿った距離は、二点間の**地図上の直線距離以上**になる
- したがって「直線距離」を h(n) とすれば必ず h(n) ≤ 実際の残距離となり、hは許容的
- 許容的なhを使うA\*は完全かつ最適なので、地下鉄の例でも最短経路が得られる

### 問題9（課題2-7）｜h(n)=0 のときのA*

h(n)=0 も許容的である。このときのA*探索はどうなるか。

**解答**

h(n)=0 は「残コストを0と見積もる」もので、明らかに h(n) ≤ 真の残コスト を満たすので許容的です。このとき評価関数は f(n)=g(n)+h(n)=g(n) となり、**Openから常に累積コストg最小のノードを取り出す＝均一コスト探索（Dijkstra法）に一致**します。完全・最適は保たれますが、ゴール方向への情報がまったくないため、全方向に等コストで広がり効率は最も悪くなります。h=0はA*の下限（最も弱い許容的ヒューリスティック）に相当します。

#### 8パズルで $h_0(n)=0$ のときの探索

> ヒューリスティックを $h_0(n)=0$ とすると、h1, h2を使った探索木はどうなるか。

- f(n)=g(n) となるので、A*(h0) は均一コスト探索になる
- 8パズルは各手のコストが一律1なので、これは実質的に幅優先探索と等価
- つまり浅い層からしらみ潰しに展開し、深さ（手数）順に解を見つける
- h1, h2のようにゴール方向へ絞り込む情報がないため、展開ノード数は h1・h2 を使う場合より桁違いに多くなる
- 最適解（最小手数）は得られるが、効率は最悪という位置づけ

#### 比較関数 h(n,n′) でも最良優先探索が可能か

各ノードの評価値h(n)は計算できないが、2状態を比べてどちらが良さそうかを返す関数 h(n,n′) があるとする。これでも最良優先探索ができることを説明せよ。

**解答**

最良優先探索が本質的に必要としているのは「Openリストの中で**最も良いノードを1つ選ぶ**」操作だけで、各ノードに絶対的な数値スコアを付けることそのものは必須ではありません。比較関数 h(n,n′) が全順序（または少なくとも最良要素を決められる順序）を与えるなら、それは比較ソートの比較演算子として使えます。すなわちOpenリストを h(·,·) で比較しながら整列、あるいは線形走査で「現状の最良候補と次の候補を h で比較し、良い方を残す」ことを繰り返せば、数値スコアなしに最良ノードを取り出せます。よって展開順を決める最良優先探索は h(n,n′) だけでも実行可能です（ソートに必要なのは大小比較であって距離の数値ではない、というのがポイント）。ただし、g+hのように**コストを加算するA*流の計算はできない**ため、欲張り探索的な使い方に限られます。

#### A\*の応用例

> A\*（およびその発展形）の応用を挙げ、その一つについて具体的な使い方を述べよ

- 経路探索は多くのアプリの基盤技術で、応用例としては、カーナビ・地図アプリの最短/最速経路案内、ゲームAIのキャラクター移動（グリッドやナビメッシュ上）、ロボット・自動搬送車(AGV)の動作計画、ネットワークのルーティング、配送・物流の配車計画、ロボットアームの動作計画などがある
- 一例としてゲームAIのキャラ移動を取り上げると、マップをグリッドに区切り、各マスをノード、隣接マス間の移動を辺（コストは地形に応じて平地1・茂み2など）とする。
- ヒューリスティックには目的地までのマンハッタン距離を使うと許容的になり、A\*が最短経路を効率的に返す
- 動的に障害物が動く場合はD_ Lite、多数のキャラが同時に動く場合はCooperative A*（時空間を予約して衝突回避）といった発展形へ拡張する、という設計になる

### 問題13（課題2-12：RTA*例題）｜この例題の最短経路を確認せよ

RTA*例題のグラフ（緑＝経路長）で、SからGへの最短経路を求めよ。

辺コスト：S-2:3, S-3:2, 2-4:2, 2-5:4, 3-4:2, 4-5:3, 4-6:3, 5-7:3, 5-8:3, 6-8:4, 7-8:2, 7-9:5, 7-G:5, 8-G:6, 9-G:2

**解答**

Dijkstra法で各ノードの最短コストを求めると、S:0, 3:2, 2:3, 4:4, 5:7, 6:7, 7:10, 8:10, 9:15, **G:15** となります。最短経路は2通りあり、いずれもコスト**15**です。

- S→3→4→5→7→G ＝ 2+2+3+3+5 ＝ 15
- S→2→5→7→G ＝ 3+4+3+5 ＝ 15

一方、図中で太線で示されたRTA_が実際に通る経路（S→2→5→7→8→G ＝ 3+4+3+2+6 ＝ 18）はこれより長く、**RTA_が最適性を保証しない**ことを示す例になっています。

### 問題14（課題2-14）｜RTA*で最短経路が得られるときのhの性質

RTA*探索でも最短経路を得られるのは、hがどんな性質を持つときか検討せよ。

**解答**

RTA*は各地点で先読み範囲内のf値が良い方へ「進んでしまう」ため、一般の許容的hでは最適性を保証しません（問題13がその反例）。最短経路が保証されるのは、本質的に**hが完全（正確）なヒューリスティック h(n)=h*(n) に一致するとき**、すなわち各ノードからゴールまでの真の残コストをhが与えるときです。このとき f(n)=g(n)+h*(n) はその点を通る真の最短コストに等しく、局所的に最良の後継を選ぶことが常に最適経路上の選択と一致するため、後戻りなしに最短経路をたどれます。

より弱く言えば、「**各地点の先読みで選ばれる最良後継が、必ず何らかの最短経路上にある**」ことを保証できる程度にhが正確であれば最適になります。逆に、hが楽観的すぎて実際には遠いノードを近いと誤認させる（=h*との差が大きい）と、RTA_はそちらへ進んでしまい、hの書き換え＋バックトラックで修正はされるものの、最終的にたどった経路は最短とは限らなくなります。なお、RTA_のh書き換え自体は許容性を保つ（戻ったなら真の距離はさらに大きいはず）ので、繰り返せば「受け入れ可能(acceptable)な経路」には収束しますが、それが最短である保証は完全なhを仮定しない限り得られません。
