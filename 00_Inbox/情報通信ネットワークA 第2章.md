
## 物理層

### 通信路の性能

#### ① アナログ信号とデジタル信号、そして標本化

- **アナログ信号**＝連続値（なめらかに変化する波）
- **デジタル信号**＝離散値（0か1のような飛び飛びの値）
- **サンプリング**＝連続なアナログ信号から、一定の時間間隔で値を「点」として抜き出すこと

#### ② 標本化定理（ナイキスト＝シャノンの定理）

- アナログ信号を、どれくらい細かい時間間隔で標本化すれば、元の波形をきちんと復元できるのか？
- **元のアナログ信号に含まれる最高周波数の「2倍以上」の周波数で標本化すれば、元の波形を完全に復元できる**

#### ③ ノイズとS/N比

- 現実の通信路には必ず**ノイズ**＝送りたい信号とは別の不要な信号が混入する
- **熱雑音**：導体中の電子の不規則な運動で生じる
- **外部信号の混入**：太陽・雷・電源・送電装置など
- ノイズの量を表す指標が **S/N比**
$$ \text{SNR [dB]} = 10 \log_{10}\frac{P_s}{P_n} $$

- $P_s$ ＝信号電力、$P_n$ ＝雑音電力
- 単位は **デシベル***
- S/N比が大きいほど「信号がノイズに対してくっきりしている＝良い回線」

#### ④ シャノン容量（通信路容量）

- 標本化定理は「ノイズなし」を想定しているが、現実にはノイズがある
- **ノイズがある通信路で達成できる最大伝送速度**を与えるのがシャノン容量

$$ C = B \log_2\left(1 + \frac{P_s}{P_n}\right) $$

- $C$ ＝通信路容量（bit/s）：これ以上速くは送れないという**理論的上限**
- $B$ ＝帯域幅（Hz）
- $P_s/P_n$ ＝S/N比（こちらは**真値**で、dBではない点に注意）

- ポイントは **「帯域幅を広げるか、S/N比を上げれば、速度の上限が上がる」** ということ
- 物理層の媒体が出せる最高速度はこの式で頭打ちになります。

#### 演習問題

> **問題**：ノイズを無視できる理想的な回線で、10 Hz までの周波数は減衰なしに送れる（カットオフ周波数 = 10 Hz）。この回線で**2段階の信号**を使って2進信号を送るとき、達成可能な最大データ転送速度は？

**解答：20 bps**

考え方：

- 回線を通過できるのは高々 10 Hz の信号成分まで。
- 標本化定理より、10 Hz の信号を扱うには **2 × 10 = 20 Hz**（＝毎秒20回）の標本化が可能。
- 「2段階の信号」＝1回の標本で 1 ビット（0か1）を表現。
- したがって **20 標本/秒 × 1 ビット/標本 = 20 bps**。

> 補足：もし信号レベルを増やせば（例：4段階なら1標本で2ビット）速度は上がりますが、この問題は2段階なので 20 bps が上限です。これは「ナイキストレート＝2B（シンボル/秒）」の典型問題です。

### 伝送媒体の種類

#### 有線伝送媒体

| 媒体               | 構造・原理                                                        | 長所                 | 主な用途                                     |
| ---------------- | ------------------------------------------------------------ | ------------------ | ---------------------------------------- |
| **同軸ケーブル**       | 内部導体（コア）を絶縁体・網状の外部導体・保護皮膜で覆う。外部導体が電磁シールドになる                  | 広帯域＋ノイズに強い         | 旧来の長距離電話、旧LAN(10BASE2/5)、**CATVインターネット** |
| **より対線(ツイストペア)** | 絶縁した2本の銅線を螺旋状にひねり合わせる。磁束が打ち消し合いノイズに強い。**2本の電圧差**で信号を運ぶ(差動信号) | 安価                 | **LAN/Ethernet**、電話回線                    |
| **光ファイバー**       | 光のパルスで信号を送る。臨界角以上で入射した光が**全反射**を繰り返し、何kmも減衰なく進む              | 広帯域・高速、減衰小、軽量、盗聴困難 | インターネット長距離回線(WAN)、**FTTH**(光回線)          |

- シールドの有無：**UTP**（シールド無し）／**STP**（シールド有り）
- 扱える周波数による分類：**カテゴリー（CAT1〜CAT8）**。CAT6まではUTP/STP両方、CAT7以降はSTPのみ。

#### 無線伝送媒体（電磁波）

- 無線は電磁波を使い、配線が不要で利便性が高い一方、**干渉の影響を受け品質が不安定**
- 周波数が高くなるほど「光」に近い性質（直進性が増し、壁を通り抜けにくい）が出てくる

|分類|周波数帯のイメージ|代表用途|
|---|---|---|
|長波(LF)〜短波(HF)|〜30 MHz|電波時計、船舶無線、アマチュア無線|
|超短波(VHF)|30〜300 MHz|業務移動通信、航空・船舶無線|
|**マイクロ波(UHF)**|0.3〜3 GHz|**携帯電話、GPS、無線LAN、Bluetooth**|
|センチ波(SHF)|3〜30 GHz|無線LAN、ETC|
|**ミリ波(EHF)**|30〜300 GHz|**5G新無線**、衛星通信|
|赤外線|3〜400 THz|リモコン、IrDA|
|可視光|405〜709 THz|可視光通信(壁を通らない・生体無害だがノイズ源多い)|

### デジタルデータの伝送

- データリンク層の「ビット（0/1）」を、物理層では**アナログ信号として送る**必要がある
#### ① ベースバンド伝送

- **ベースバンド＝送信対象の信号と同じ周波数帯のまま送る方法**
- 搬送波に振幅・周波数・位相の変調をかけず、電圧パターンで直接ビットを表現
- 主に**有線通信**で使われます。
- ビットを電圧パターンに変換する規則を **伝送路符号**と呼ぶ

|伝送路符号|特徴|
|---|---|
|**NRZ** (Non-Return-to-Zero)|最も単純。1を正の電圧、0を負の電圧で表現|
|**マンチェスタ符号**|後述。クロック情報を埋め込む|
|AMI / 4B5B / 8B10B|より高度な符号化（Ethernet等で使用）|

**マンチェスタ符号**

- 信号の中に**クロック（同期）情報**を埋め込んだ符号化方式。
- ビットの「値」ではなく「**電圧の遷移（変化）**」で 0/1 を表現する。
    - 例：低→高の遷移を 1、高→低の遷移を 0、のように割り当てる（逆の流儀もあり：Ethernet 10BASE-T）
- 必ず各ビットの途中で電圧が変化するので、送受信のタイミングを合わせやすい。
- 欠点：同じビットレートを出すのに **NRZの2倍の周波数**が必要。

#### ② パスバンド伝送（デジタル変調）

- **搬送波を、データに応じて変調して送る方法**
- 主に**無線・光**で使われる
- 搬送波の「振幅・周波数・位相」のどれを変えるかで方式が分かれる

|方式|変調する量|特徴|
|---|---|---|
|**ASK** (振幅偏移変調)|振幅|干渉に弱く、無線では効率が悪い|
|**FSK** (周波数偏移変調)|周波数|ASKよりエラー・干渉に強い（干渉は特定周波数に限定されやすいため）|
|**PSK** (位相偏移変調)|位相|3つの中で**最もエラーに強い**|
|**QAM** (直交振幅変調)|振幅＋位相|振幅と位相を組み合わせ多値化（高速化）|

**パスバンドの利点**：

- 無線や光では**使える周波数帯域が決められている**ので、信号を任意の周波数帯に「持ち上げる」必要がある
- 変調すれば元信号と異なる特定の帯域に乗せられる
- 変調しておくと、次節の**周波数分割多重・符号分割多重**が適用できる
- エラー耐性の順は **PSK > FSK > ASK** 

### 多重化

|方式|分け方|例え話（教科書のアナロジー）|
|---|---|---|
|**TDM**（時分割多重）|**時間**で分ける|各人が**順番に**割り当てられた時間に話す|
|**FDM**（周波数分割多重）|**周波数**で分ける|各グループが**異なる周波数**で話す（互いに干渉しない）|
|**CDM**（符号分割多重）|**符号**で分ける|各グループが**異なる暗号(言語)**で話す（理解できない言語は無視）|

#### TDM（時分割多重）

- 複数の信号を順番に並べて1本の回線に流す
- 集約後の回線はより高い帯域幅を持つ（例：64 kbps × 3本 → 192 kbps）。

#### FDM（周波数分割多重）

- 各チャネルに、互いに重ならない周波数領域を割り当てる
- 各信号を別々の周波数帯に乗せて同時に流す。

#### CDM（符号分割多重

- 各端末に **チップ** と呼ばれる、互いに**直交する**ベクトルを割り当てる

**具体例**

- 4つの端末に次のチップを割り当てる

```
c₁ = (+1, +1, +1, +1)
c₂ = (+1, +1, −1, −1)
c₃ = (+1, −1, −1, +1)
c₄ = (−1, +1, −1, +1)
```

- 各端末は、送りたいビットが **+1 のときは自分のチップ $c_i$ を、−1 のときは $-c_i$ を**送信する
- いま、端末 **1,3 が +1** を、端末 **2,4 が −1** を送ったとする
- 回線上では全員の信号が**重なり合って**観測される

```
合成信号 S = c₁ − c₂ + c₃ − c₄

  c₁  = (+1, +1, +1, +1)
 −c₂  = (−1, −1, +1, +1)
  c₃  = (+1, −1, −1, +1)
 −c₄  = (+1, −1, +1, −1)
 ─────────────────────────
  S   = (+2, −2, +2, +2)
```

- 受信側は、この**ごちゃ混ぜになった S** から、知りたい端末のチップとの**正規化内積**を計算するだけで、その端末が送ったビットを取り出せる

```
S・c₁ / 4 = ( 2 − 2 + 2 + 2)/4 = +1  → 端末1 は +1 を送った ✓
S・c₂ / 4 = ( 2 − 2 − 2 − 2)/4 = −1  → 端末2 は −1 を送った ✓
S・c₃ / 4 = ( 2 + 2 − 2 + 2)/4 = +1  → 端末3 は +1 を送った ✓
S・c₄ / 4 = (−2 − 2 − 2 + 2)/4 = −1  → 端末4 は −1 を送った ✓
```
