

## フィルタから直交変換へ

### 線形シフト不変システムとたたみ込み

- ディジタル信号処理の基本は、入力 $x(n)$ をシステム $h(n)$ に通して出力 $y(n)$ を得る、という枠組み
- システムが**線形**かつ**シフト不変**であれば、入出力関係は必ず**たたみ込み演算**で書ける

$$y(n) = x(n)\otimes h(n) = \sum_{k=-\infty}^{\infty} x(k),h(n-k) = \sum_{k=-\infty}^{\infty} x(n-k),h(k)$$

- ここで $h(n)$ は**インパルス応答**で、システムの特性を完全に決める
- 最後の等号は、$x$ と $h$ の役割を入れ替えても同じになる（たたみ込みの可換性）ことを示している

### ディジタルフィルタの構造

- このたたみ込みをハードウェア／プログラムで実現したものがディジタルフィルタ
- 構成要素は3つだけ

- **遅延** $z^{-1}$：信号を1サンプル遅らせる（$x(n)\to x(n-1)$）
- **乗算**：各遅延信号にフィルタ係数 $h(k)$ を掛ける
- **加算**：それらを足し合わせて出力にする

### DFTの行列表現

- 8点DFTを思い出すと、

$$X(k) = \sum_{n=0}^{N-1} x(n),W_N^{nk}, \qquad W_N = e^{-j\frac{2\pi}{N}}$$

- これを8個すべての $k$ についてまとめて書くと、**行列とベクトルの積**になる

$$\vec{X} = F\vec{x}$$

- ここで重要な見方は、**行列 $F$ の各行が1つのディジタルフィルタになっている」という点
- 第 $k$ 行は係数 ${W_N^{0\cdot k}, W_N^{1\cdot k}, \dots}$ を持つフィルタで
- これは入力 $x(n)$ から周波数 $k$ の成分だけを取り出す働きをする

### フィルタバンクという見方

- 各行が「特定の周波数を抜き出すフィルタ」
- DFT行列全体は**フィルタバンク**（多数のフィルタの集合）と解釈できる
- これを発展させると、次のような信号処理の基本構造になる

$$x(n) \to \underbrace{h_i(n)}_{\text{分析フィルタ}} \to \underbrace{\downarrow M}_{\text{サンプリング}} \to \underbrace{\uparrow M}_{\text{補間}} \to \underbrace{g_i(n)}_{\text{合成フィルタ}} \to y(n)$$

## 直交変換とは

### 直交行列とユニタリ行列

- 直交変換は、特別な性質を持つ行列 $A$ による線形変換

$$\vec{X} = A\vec{x}, \qquad \vec{x} = A^{-1}\vec{X}$$

- **直交行列**の定義は、

$$A^t A = A A^t = I \quad(\text{転置行列が逆行列})$$

- つまり $A^{-1} = A^t$ 
- 逆変換を求めるのに、わざわざ逆行列計算をせず「転置するだけ」でよい、というのが最大の利点
- **ユニタリ行列**は、DFTのように成分が複素数の場合への拡張

$$A^{t} A = A A^{t} = I \quad(A_\text{複素共役行列})$$

- 逆行列が**共役転置**（転置して複素共役をとったもの）で得られる
- 直交行列は、ユニタリ行列の実数版と考えればよい

### なぜ「直交」が嬉しいのか

- 逆変換が転置だけで計算でき軽い
- 変換によって信号のエネルギーが保存される（パーセバルの定理）
- 信号を「互いに干渉しない成分」に分解できる

## ブロック変換とオーバーラップ変換

### ブロック変換

- 信号を互いに重ならない $N$ 点ブロックに区切り、それぞれに $N\times N$ 行列を施す
- DFT・DCT・アダマール変換・KLTがこれにあたる
- シンプルだが、ブロック境界で不連続が生じやすい欠点がある

### オーバーラップ変換

- 隣り合うブロックを互いに重ねながら変換する
- MDCTや離散ウェーブレット変換がこれで、ブロック境界の不連続（ブロックノイズ）を抑えられるのが利点

## 各直交変換の詳細

### DFT（離散フーリエ変換）

$$X(k) = \frac{1}{\sqrt{N}}\sum_{n=0}^{N-1} x(n)W_N^{nk}, \qquad x(n) = \frac{1}{\sqrt{N}}\sum_{k=0}^{N-1} X(k)W_N^{-nk}$$

1. **高速計算アルゴリズム（FFT）が存在する** ―― 演算量を $N^2$ から $N\log_2 N$ に削減
2. **積・畳み込みの特性** ―― 時間領域のたたみ込みは周波数領域の積に、時間領域の積は周波数領域のたたみ込みになる。この性質を持つのは**フーリエ変換系だけ**で、フィルタリングを効率化できる理由
3. **周期性を仮定** ―― ブロック単位で信号が繰り返す（周期 $T=N$）と想定している

### アダマール変換

$N = 2, 4, 8, 16, \dots$ と再帰的に作れます。

$$H_2 = \frac{1}{\sqrt{2}}\begin{pmatrix}1 & 1\ 1 & -1\end{pmatrix}, \qquad H_{2N} = \frac{1}{\sqrt{2}}\begin{pmatrix}H_N & H_N\ H_N & -H_N\end{pmatrix}$$

**特徴：** 係数が **±1だけ**なので、**乗算が一切不要**（足し引きだけ）。ハードウェア性能が乏しかった時代に重宝され、現在は携帯電話のCDMA（符号分割多重）で各ユーザーを区別する直交符号として使われます。

### 3. DCT（離散コサイン変換）― JPEG / MPEGの心臓部

$$X(k) = \sqrt{\frac{2}{N}},\alpha(k)\sum_{n=0}^{N-1} x(n)\cos\frac{(2n+1)k\pi}{2N}, \quad \alpha(k)=\begin{cases}1/\sqrt{2} & (k=0)\ 1 & (k\neq 0)\end{cases}$$

**特徴3つ：**

1. **高速計算アルゴリズム（FDCT）が存在する**
2. **ブロック境界で信号が鏡像的に繰り返すことを仮定**（信号の鏡像性、周期 $T=2N$）
3. **隣接信号間の相関が高い信号（画像など）に対して、後述のKLTに漸近する** ―― これが圧縮で広く使われる決定的な理由です

**DFTとDCTの周期性の違い：** DFTは信号がそのまま周期 $N$ で繰り返すと仮定するため、ブロックの端と端が一致せず大きな不連続（＝高周波の偽成分）が出ます。一方DCTは端で**折り返した鏡像**をつなげて周期 $2N$ とするため、境界が滑らかにつながり、エネルギーが低周波に集中します。これが圧縮効率の高さに直結します。

**FDCTの手順：** ①鏡像性を使って $x(n)$ から長さ $2N$ の $x'(n)$ を作る

$$x'(n) = \begin{cases} x(n) & (n=0,\dots,N-1)\ x(2N-1-n) & (n=N,\dots,2N-1)\end{cases}$$

②これに **2N点FFT** を施し、③DCTの定義に従って位相補正すると $X(k)$ が得られます。演算量は加算・乗算とも $2N\log_2 N$ です。

### 4. KLT（カルーネン・レーベ変換）― 理論的な最適解

KLTは固定の式で与えられず、**入力信号の自己相関行列の固有ベクトル**から作られる、信号依存の変換です。

$$R_{xx} = E[\vec{x},\vec{x}^t]\ (\text{自己相関行列}), \qquad R_{xx},\vec{l}_k = \lambda_k,\vec{l}_k\ (\text{固有値問題})$$

固有ベクトル ${\vec{l}_k}$ を並べたものがKLT行列です。

**算出手順：** ①自己相関行列を計算 → ②固有値・固有ベクトルを計算 → ③固有ベクトルからKLT行列を構成

**特徴：**

1. **変換係数の自己相関行列が対角行列になる**（＝変換後の係数が互いに無相関）。これはKLT固有の性質で、データ圧縮として**最適な変換行列**を与えます。主成分分析（PCA）としてパターン認識でも広く使われます
2. **しかし効率的な高速アルゴリズムが存在しない**。信号ごとに固有値分解が必要で重いため、**圧縮の目標性能（ベンチマーク）としては使われるが、実際の圧縮にはほぼ使われない**

つまり「DCTが実用的にKLTへ漸近する」という関係が、DCTが画像圧縮の標準である理由を裏付けています。

### 5. MDCT（修正離散コサイン変換）― MP3のコア

$$X(k) = \frac{1}{\sqrt{N}}\sum_{n=0}^{2N-1} x(n)\cos\frac{(2n+1+N)(2k+1)\pi}{4N}, \quad k=0,\dots,N-1$$

**オーバーラップ変換の代表例**です。$2N$ 点を入力して $N$ 個の係数を出し、逆変換後に**隣接ブロック間で加算**して信号を再構成します（重ね合わせ加算）。この重ね合わせにより、ブロック境界の不連続を消せます。**MP3（MPEG-1 Audio Layer 3）** で採用されています。

## 応用例

### 動画像圧縮

- DCT → 量子化 → 符号化、という流れに、フレームメモリと動き予測を組み合わせた予測符号化（逆量子化・逆DCTでループを閉じる）
- MPEGの基本構造です

#### オーディオ圧縮（MP3）

- 帯域分割 → MDCT → 量子化 → 符号化
- 並行してFFTで心理聴覚分析を行い、人間に聞こえにくい成分を粗く量子化することでデータ量を削減

## Exercise (1)：ユニタリ行列であることの証明

**$F_2 = \dfrac{1}{\sqrt{2}}\begin{pmatrix}1&1\1&-1\end{pmatrix}$ について**

$F_2$ は実数かつ対称なので、共役転置は $F_2$ 自身です。

$$F_2^{*t}F_2 = \frac{1}{2}\begin{pmatrix}1&1\1&-1\end{pmatrix}\begin{pmatrix}1&1\1&-1\end{pmatrix} = \frac{1}{2}\begin{pmatrix}2&0\0&2\end{pmatrix} = I ;\checkmark$$

**$F_4 = \dfrac{1}{2}\begin{pmatrix}1&1&1&1\1&-j&-1&j\1&-1&1&-1\1&j&-1&-j\end{pmatrix}$ について**

各成分は $(F_4)_{kn} = \dfrac{1}{2}W^{kn}$（$W = e^{-j2\pi/4} = -j$）です。共役転置との積の $(i,j)$ 成分は、

$$(F_4^{*t}F_4)_{ij} = \frac{1}{4}\sum_{k=0}^{3}\overline{W^{ki}},W^{kj} = \frac{1}{4}\sum_{k=0}^{3}W^{k(j-i)}$$

- $i=j$ のとき：$\dfrac{1}{4}\sum_{k=0}^{3}W^0 = \dfrac{1}{4}\cdot 4 = 1$
- $i\neq j$ のとき：等比数列の和で、$\dfrac{1}{4}\cdot\dfrac{W^{4(j-i)}-1}{W^{(j-i)}-1}$。$W^4 = (-j)^4 = 1$ より分子が0なので $= 0$

よって $F_4^{*t}F_4 = I$。**ユニタリ行列である** $\checkmark$

（具体的に列ベクトル同士の内積を計算しても確認できます。たとえば第1列と第2列の内積は $\frac{1}{4}(1\cdot1 + 1\cdot\overline{(-j)} + 1\cdot\overline{(-1)} + 1\cdot\overline{j}) = \frac{1}{4}(1+j-1-j)=0$。）

## Exercise (2)：AR(1)モデルの自己相関係数

モデルは $x(n+1) = \rho,x(n) + \varepsilon(n)$、$\varepsilon$ は平均0・無相関（白色雑音、分散 $\sigma_\varepsilon^2$）です。$x(n)$ は過去の雑音 $\varepsilon(n-1),\varepsilon(n-2),\dots$ で決まるので、未来の雑音とは無相関：$E[x(n)\varepsilon(n)] = 0$。

**まず分散** $\sigma_x^2 = E[x(n)^2]$ を求めます。定常性 $E[x(n+1)^2]=E[x(n)^2]$ より、

$$\sigma_x^2 = \rho^2\sigma_x^2 + 2\rho\underbrace{E[x(n)\varepsilon(n)]}_{=0} + \sigma_\varepsilon^2 ;\Longrightarrow; \boxed{\sigma_x^2 = \frac{\sigma_\varepsilon^2}{1-\rho^2}}$$

**次に自己相関** $R(k) = E[x(n)x(n+k)]$（$k\ge 1$）。$x(n+k)=\rho x(n+k-1)+\varepsilon(n+k-1)$ を代入：

$$R(k) = \rho,E[x(n)x(n+k-1)] + \underbrace{E[x(n)\varepsilon(n+k-1)]}_{=0} = \rho,R(k-1)$$

漸化式を解いて、

$$\boxed{E[x(n)x(n+k)] = R(k) = \rho^{|k|},\sigma_x^2 = \frac{\rho^{|k|}}{1-\rho^2},\sigma_\varepsilon^2}$$

ラグ $k$ が増えるほど相関が $\rho^{|k|}$ で指数的に減衰します（$|\rho|<1$ が定常の条件）。

## Exercise (3)：自己相関行列 $R_{xx}$ の簡略化

$\vec{x} = [x(0),x(1),\dots,x(N-1)]^t$ とすると、$(R_{xx})_{ij} = E[x(i)x(j)] = R(|i-j|) = \rho^{|i-j|}\sigma_x^2$。したがって、

$$R_{xx} = \sigma_x^2\begin{pmatrix} 1 & \rho & \rho^2 & \cdots & \rho^{N-1}\ \rho & 1 & \rho & \cdots & \rho^{N-2}\ \rho^2 & \rho & 1 & \cdots & \rho^{N-3}\ \vdots & & & \ddots & \vdots\ \rho^{N-1} & \rho^{N-2} & \cdots & & 1 \end{pmatrix}$$

これは $(i,j)$ 成分が $\rho^{|i-j|}$ の**対称テプリッツ行列**です。$\rho\to 1$（強い相関）では全成分が1に近づき、画像のような相関の高い信号を表すモデルになります。

## Exercise (4)：KLTが自己相関行列を対角化することの証明

KLTでは、$R_{xx}$ の固有ベクトルを**列**に並べた行列 $A=[\vec{l}_0,\vec{l}_1,\dots,\vec{l}_{N-1}]$ を使い、変換係数を $\vec{X} = A^t\vec{x}$ とします。$R_{xx}$ は実対称なので固有ベクトルは正規直交にとれ、$A^tA=I$（直交行列）です。

変換係数の自己相関行列は、

$$R_{XX} = E[\vec{X}\vec{X}^t] = E[A^t\vec{x},\vec{x}^tA] = A^t,E[\vec{x}\vec{x}^t],A = A^t R_{xx} A$$

固有値問題 $R_{xx}A = A\Lambda$（$\Lambda = \mathrm{diag}(\lambda_0,\dots,\lambda_{N-1})$）と $A^tA=I$ を使うと、

$$R_{XX} = A^t R_{xx} A = A^t A,\Lambda = \Lambda = \begin{pmatrix}\lambda_0 & & \ & \ddots & \ & & \lambda_{N-1}\end{pmatrix}$$

**対角行列になった** $\checkmark$。非対角成分が0なので、変換後の各係数は**互いに無相関**です。これが「KLTがデータ圧縮として最適」と言われる根拠で、エネルギーが各固有値 $\lambda_k$ に分離され、重要な係数だけを残せば効率的に圧縮できます。

## Exercise (5)：アダマール変換によるCDMAシミュレーション

```matlab
d1=8; d2=12; d3=18; d4=10;        % 4ユーザーの送信データ
N=8; H=hadamard(N);               % 8×8アダマール行列
code1=H(1,:); code2=H(3,:);       % 各ユーザーに直交符号を割当
code3=H(5,:); code4=H(8,:);
x = d1.*code1+d2.*code2+d3.*code3+d4.*code4+0.01*randn(1,N); % 多重化送信
dd1=sum(code1.*x)/N               % 受信：相関による復調
```

**ポイント：** アダマール行列の各行は互いに直交し、$\text{code}_i\cdot\text{code}_j = N\delta_{ij}$ が成り立ちます。受信側で

$$\frac{1}{N}\sum \text{code}_i\cdot x = \frac{1}{N}\sum_j d_j(\text{code}_i\cdot\text{code}_j) + (\text{雑音}) = d_i + (\text{微小雑音})$$

となるため、**結果は dd1≈8, dd2≈12, dd3≈18, dd4≈10** と元データがほぼ復元されます。複数ユーザーが同じ帯域を直交符号で共有する**CDMAの原理**そのものです。

## Exercise (6)：DCTによる音声圧縮

```matlab
x = her'; X = dct(x);                       % 音声をDCT
[XX,ind] = sort(abs(X),'descend');          % 係数を大きい順にソート
need = 1;
while norm(X(ind(1:need)))/norm(X)<0.999    % エネルギー99.9%に必要な数を探す
    need = need+1;
end
X(ind(need+1:end)) = 0;                     % それ以外の係数を0に
xx = idct(X);                               % 逆DCTで復元
```

**ポイント：** DCTは音声のエネルギーを少数の係数に集中させるため、**全体のごく一部（数%）の係数だけでエネルギーの99.9%を保持**できます。残りを捨てて逆DCTしても、元の波形（`Original`）と復元波形（`xx`）の差（`Difference`）はごくわずかです。これが圧縮の本質で、`xpc` に「必要な係数の割合（%）」が表示されます。男声・女声で必要係数の割合が変わる点も観察してみてください。

## Exercise (7)：DCTによる画像圧縮（JPEGの原理）

```matlab
I = im2double(imread('cameraman.tif'));
T = dctmtx(8);                              % 8×8 DCT行列
dct = @(b) T*b.data*T';                     % ブロックごとに2次元DCT
B = blockproc(I,[8 8],dct);
mask = [...];                               % 左上の低周波だけ残す三角マスク
B2 = blockproc(B,[8 8],@(b) mask.*b.data);  % 高周波成分を捨てる
invdct = @(b) T'*b.data*T;
I2 = blockproc(B2,[8 8],invdct);            % 逆DCTで復元
```

**ポイント：** 画像を8×8ブロックに分け、各ブロックを2次元DCT（$T\cdot\text{block}\cdot T'$）します。`mask` は**左上（低周波）の係数だけを残し、右下（高周波）を0にする**フィルタです。画像のエネルギーは低周波に集中しているので、この三角マスクで多くの係数を捨てても、復元画像 `I2` は元画像 `I` とほぼ見分けがつきません ―― これがまさに**JPEG圧縮の核心**です。マスクで残す係数を増減させると、画質とデータ量のトレードオフが体感できます。
