
## 知能(Intelligence)とは何か

- 辞書的には「外界の状況の知覚をもとに認識・理解を行い、得られた知識・情報を使って問題を適切に処理し、適応する能力」
- 哲学辞典では「**学習能力・抽象思考能力・環境適応能力**」の3つに整理される
- **「では、何ができたら『知能がある』と言えるのか?」** 
- 定義を並べても、判定基準ははっきりしない
### 「機械は考えるか?」という思考実験

知能の本質を考えさせる有名な比較が出てきます。

- 機械は**飛ぶ**か? → Yes(ただし鳥や昆虫のように羽ばたきはしない)
- 機械は**泳ぐ**か? → No?(船は航行 sail するが「泳ぐ」とは言わない)
- 機械は**走る**か? → Yes?
- 機械は**駈ける(gallop)**か? → No?

つまり、機械は同じ「移動」という結果を達成しても、**生物と同じやり方でやっているわけではない**。ここから核心の問いに至ります——**「では機械が『考える』とは Yes か No か?」**

この問いに関連して2つの論点が示されます。1つは「Deep Blue(チェス)やAlphaGo(囲碁)は確かに人間に勝ったが、そこに知性があると思うか?」。もう1つが哲学者**サール(Searle)の「中国語の部屋」**という反論です。これは「規則に従って正しい中国語の応答を返せても、部屋の中の人は中国語を理解していない=記号操作ができることと理解していることは別だ」という思考実験で、「結果が出せること=知性があること」ではないかもしれない、と問題提起しています。

そして、「何ができたら知能と言えるか」の基準として2つの方向性が対比されます。**①生物(人間)の判断を基準にする**(ただし人間には発想・思いつきもあれば誤りもある。誤りまで真似るのか?という問題、文化依存の問題がある)。**②可能な限り広く・深く推論する『合理性』を基準にする**(評価基準が明確で、必ずしも人間的でない=人間は時に非合理だから)。この①と②の対立軸が、次のスライドの中心的なフレームワークになります。

## 3. AI研究の4象限(スライド4)★最重要

ここがこの講義で最も大事な図の一つです。AI研究を**2つの軸で4つに分類**します。

||**人間を基準**(誤りも真似るかも)|**合理性を基準**(可能な範囲で最善解を求める理想像)|
|---|---|---|
|**行動を基準**(外から見た振る舞い)|**人間のように行動する**|**合理的に行動する**|
|**思考を基準**(内部の思考方法)|**人間のように考える**|**合理的に考える**|

縦軸が「**思考か行動か(内部のプロセスか、外から見える振る舞いか)**」、横軸が「**人間を基準にするか、合理性を基準にするか**」です。各象限を見ていきます。

**① 人間のように行動する**：「人間が行う場合には知能を必要とする機能を達成する機械を創る技術」。評価方法は**チューリングテスト**(人間と区別がつかない応答ができれば合格。emacsの Eliza が例として挙げられています)。

**② 合理的に行動する**：「知的行動の自動化に関する計算機科学の一分野」。これが**「合理的エージェント」アプローチ**で、知識や信念から可能な限りの合理的結論を得る。

**③ 人間のように考える**：「計算機が考えるようにする。真の意味で心を持った機械を造る、刺激的で新しい試み」。これが**認知科学的アプローチ**(人間の記憶や思考の過程を考える)。スライドではこの象限が青枠で強調され、「機械で意識や人間性(humanity)を実現したい、科学的に解明したい」という究極の目標を持つ領域とされています。

**④ 合理的に考える**：「計算機モデルを用いた心の機能の研究/認識・推論・行為を可能にする計算の研究」。**形式的な推論や思考の規則=論理学**のアプローチ。

重要な注意点が2つあります。**4枠どれもAIの研究**であること(AIは知性を広く科学的に解明する目的を持つため)。そして、**人間も時に合理的・時に非合理的、時に論理的・時に直感的**であり、これらの枠はきれいに分かれているわけではない、ということです。

### 強いAIと弱いAI

- **強いAI**：心(意識や自我)を実現する。アルゴリズムで心が実現可能だという立場。**サール自身はこれに否定的**
- **弱いAI**：機械は心・意識を持たない。そこまでは追求しない(できない)。

## 合理的エージェントアプローチと限定合理性

### 合理的エージェントとは

- 保持している情報(知識・時間・使える資源)に基づき、常に最善の決定を下すもの
- この意味で「知的」
- ここでポイントは「**調べれば分かるのに『知らなかったことにしよう』は非合理**」という点
- **合理性には、必要な知識を取得するという行為も含まれる**

### 限定合理性

- 可能な限り、限りある範囲での合理性
- 完全な合理性は無理だが、与えられた資源の範囲内で最善を尽くす
- これがこのアプローチの現実的な立場
- **人間の思考過程を真似ても真似なくてもよい**

## 現代のAIの定義


1. 「環境を知覚し、**成功の可能性を最大化する**行動をとるシステム」——しかし"成功"は曖昧。
2. 「**期待効用(expected utility)を最大化する**システム。ただし効用(目的)は人間の設計者が与える」——しかし多くの場合、効用は固定できず、設計者が与えきれない。
3. 「**人間の真の目的が不明確な場合でも、その人間の代わりに動作するシステム。したがって、何を最大化すべきかを学習し、目的が不明確なままでも適切に機能する必要がある**」

この3段階目は、近年のAIの考え方を象徴しています。「目的を与えられて最適化する機械」から「**目的そのものが不確実で、それを学習しながら適切に振る舞う機械**」へと定義が深化しているのです。

知的に行動するエージェントの条件として、最終的に4つが挙げられます：**①ゴールと状況に適切に行動する、②環境やゴールの変化に柔軟に対応する(適応)、③経験から学ぶ(学習)、④知覚的・計算的限界の中で適切に選択する(限定合理性)**。

## エージェントとは何か

- ラテン語の **agere(行動する)** から **Agent(行動する者、動作主)** が来ている
- つまりエージェントは本質的に「**行動する主体**」
- エージェントとは、**センサー**を通して環境を認知し、**エフェクター**により環境に対して自律的・合理的に行為を行う**実体(entity)**。
- ここで「実体」には、機械・ロボット・人間のような**物理的実体だけでなく、仮想的なソフトウェア・プログラムも含まれる**
- エージェントは**センサーで環境を知覚し → 内部で判断し → エフェクタ(アクチュエータ)で環境に働きかける**。
- そしてその行為が環境を変え、変わった環境をまた知覚する……という**ループ**を回す

### エージェントの具体例

- **ロボット**：認知=超音波・接触・距離計・カメラ、行為=移動・方向転換
- **ソフトウェアエージェント**：認知=プログラムへの入力(数値・文字・音・画像)、行為=プログラムからの出力
- **組織の代理(delegates)**：Microsoftや日本政府の代理として動く専門家(教師・医者・トレーダー・旅行代理店…)。あるいはサーモスタット、飛行機の制御プログラム、Google Car、Mars rover など
- **生き物**：犬・魚・鳥・昆虫・細菌・ウイルス(物理学でよく扱う対象)

### 合理性・自律性・創発

エージェントの性質をさらに掘り下げます。3つの重要概念があります。

**合理性**：「分かっていることから予測できる最大限の成功を考えること」。先ほどの「赤信号になりそうだが車がいないので渡ろうとしたら鉄骨が落ちてきた」という例——これは**知り得ない事態なので不合理とは言えない**。理想合理的エージェントとは、知覚の履歴に対して、持っている知識を使って最大の性能を引き出す動作を選ぶもの。「最大の性能」を測るには**性能基準(performance measure)**が必要で、これはゴール達成・速度・使用資源・環境への影響などで、問題ごとに変わります。

**自律性(autonomy)**：行為を**環境や自らの経験から決定できる**こと。外部からあらかじめ決められた命令だけに従うなら自律的とは言わない。ただし経験ゼロの段階でも適切に動くための**事前知識(生物の本能のようなもの)**は必要で、本能的に盲目的に動くだけでなく**環境の変化に適応する**ことが求められます。

**創発(emergence)**：これは面白い概念です。**個々が単純な動作しかしないエージェントでも、全体として見ると何らかの知的な行動・現象が現れる**ことがある。例として**渡り鳥のV字編隊**や**イワシの群れ**が挙げられます。どちらも、各個体の自律的な最適判断をベースに、全体としての秩序だった形が「創発」しているわけです。

なお歴史的補足として、「エージェント」という語は1988年頃からAI分野で使われ始め、1992-3年頃から「自律した計算主体・プログラム」として計算機科学やソフトウェア工学に広く普及した、と述べられています。

## エージェントが備えるべき能力

「自律的・知的に振る舞うには、何をプログラムとして内部に実装すればいいか」を整理します。図に沿って**4つの要素**が挙げられます。

**① 認知・判断能力**：可能な行動のモデル、環境の認知とその内部表現(モデル)、自分の行為が環境に与える影響のモデル。 → 課題：**計算機内に環境を表現(モデル化)する必要がある**。

**② ゴール・優先順位・重要さ**：希望する状態とその重要さ。これと①を合わせて、希望状態に達しうる行為を選択する。 → 課題：**希望状態を与えるか学習する必要があり、そこへ達する行為の計画=プランニング(planning)が必要**。

**③ 知識と経験(メモリ)**：行為と結果の知識、過去の行為と環境変化の記憶。これにより似た状況で即座に対応できる。 → 課題：**「選択可能な行為と想定結果」(行動モデル)や「望ましい結果の記録」が必要**。

**④ 環境の観測・刺激**：現在の状況の把握、行為結果の確認、想定外の結果や回避すべき状態の認識。 → 課題：**環境の観測と内部モデルとの照合が必要**。

そしてスライドは「**これらの能力をどうやって実現するか……**」と問いかけて次へ続きます。実はこの①〜④の「課題」が、この後の講義トピック(探索・プランニング・機械学習・知識表現など)に対応しているのです。導入講義が後の全トピックの「地図」になっている構造が見えます。

## エージェントのアーキテクチャ

「内部をどう設計するか」の代表的な4つのアーキテクチャが、**単純なものから高度なものへと積み上がる形**で説明されます。これは重要なので順に見ます。なお、スライド左上の▲マークは「少し進んだ内容/飛ばすこともある」を示します。

### ① 単純反射エージェント(Simple reflex agents)

**知覚と行為の対応表(lookup table)や if-then 形式のルール(プロダクションルール)**だけで行動を選ぶ。**状態(メモリ)を持たない**——過去の環境・知覚・行動を一切記録しません。動物の反射的な単純行動に対応します。

擬似コードはこうです：

```
Function simple-reflex-agent(percept) returns action
  static: rules  // 条件-行動ルールの集合
  state  ← Interpret-Input(percept)   // 知覚を解釈
  rule   ← Rule-Match(state, rules)   // 合致するルールを探す
  action ← Rule-Action-Part(rule)     // ルールの行動部を取り出す
  return action
```

**問題点**が明確に列挙されています：(a)対応表が巨大になる(チェスは約 10¹²⁰ の状態がある)、(b)知覚への対応しかできず「知覚→行動→新状態→行動」と先を読めない(新状態を知覚しないと次に動けない)、(c)環境変化に対して固定的、(d)学習でテーブルを書き換えるには情報が少ない、(e)**過去を覚えるメモリがないのでループが起こりうる**。

### ② 記憶を持つ反射エージェント(Agents with memory)

①に**内部メモリ**を加えたもの。現在の環境の内部モデルや過去に知覚した内容を記憶します。内部モデルとは「環境はこうである」とエージェントが考える環境の表現です。外から与えると良いものとして、**(1)自分の行為が環境をどう変えるか(状態遷移のモデル)**と**(2)自分の行為と独立に環境がどう変化するか(確率的モデル)**の2つが示されます。これにより、行為がもたらす環境変化に内部モデルを追随させ、**少し先の行為まで推論できる**ようになります。

### ③ ゴール主導エージェント(Agents with goals)

②に**ゴール(エージェントが望む状態)**を加えたもの。単に状態をメモリに持つだけでなくゴールを持つことで、「どの状態に変わるのがより適切か」を判断できる。**ある状態で可能な行為が複数あるとき、そのうち適切なものを選べる**ようになります(それまでは状態に対し基本的に単一行動が対応)。例：「授業に遅刻しそうなのにラーメンは食べに行かない(ゴール次第だが)」。

この性質は、①②の**反射的(reactive)**に対して**熟考的(deliberative)**と呼ばれます。さらに内部モデルに基づき数手先の状態を考え、ゴールへ達する道筋を作る。この「**ゴールに向けた行為の列をプラン(plan)**」、それを生成する過程を「**プランニング(planning)**」と呼びます。

### ④ 効用に基づくエージェント(Utility-based agents)

②に**効用関数(utility function)** `U: State → 数値` を加えたもの。各状態の「望ましさの度合い」を数値で表します。ゴールの達成/未達成という二択だけでなく、**効率・安全性・速度・成功確率**なども数値として考慮できます。

なぜこれが必要かというと、**ゴールを達成する方法が複数あったり、評価が単一指標でなかったりする**から。例えば「早く達成したい(迅速性)」「確実に達成したい(成功確率)」「安全に達成したい(安全性)」は、**時に同時には達成できない**(安全性と迅速性のトレードオフなど)。効用関数を使えば「何が大切か」を数値で表して行為を選べます。**ゴールに近づくことも効用関数に含められるので、効用ベースはゴール主導を包含する**(より一般的な)エージェントと言えます。

この①→②→③→④という積み上げの構造を押さえておくと、後のトピック(探索・強化学習など)で「このアルゴリズムはどのレベルのエージェントの話か」が見通せるようになります。

## 8. 環境の性質による分類(スライド13)

**エージェントの能力は、置かれた環境の性質に左右されます**。環境を分類する4つの軸が示されます。

**① アクセス可能性(accessible / inaccessible, observable)**：行為決定に必要な情報をセンサーで**完全に取得できる**なら「アクセス可能(観測可能)」。部分的にしか取れない場合もある(partially observable)。

**② 決定的・非決定的(Deterministic / Nondeterministic)**：次の状態が**現在の状態と行為によって完全に決まる**なら「決定的」。アクセス可能かつ決定的な環境なら、エージェントは不確実性を考えなくてよい。逆に、本当は非決定的でも取得できない情報があると決定的に見えることがある(その場合でも非決定的として扱う)。

**③ 静的・動的(Static / Dynamic)**：エージェントが行為しなければ変化しないのが「静的環境」(扱いやすい)。「動的環境」では、情報を取得せずに熟考していると結果が無意味になることもある。

**④ 離散的・連続的(Discrete / Continuous)**：知覚や行為が**離散値**で表せれば離散的(例：チェス)、**連続値**なら連続的(例：自動運転の速度やハンドル角)。両者の融合もあり、連続値も計算機内部では細かく分けて離散的に扱うことが多い。

これらの軸は、後の探索アルゴリズムや強化学習で「どんな問題設定か」を規定する重要な前提になります。

## AIの歴史

### 前史

- 学問としての問いは2000年以上前(哲学)
- 計算機の誕生としては、Zuse の Z3(1941、リレー式2進計算機)、Wilkes らの EDSAC(1949、最初のプログラム格納式電子計算機、真空管式)
- AIに直結する萌芽として、**McCulloch & Pitts の神経細胞の数理モデル(1943、ニューラルネットの始まり)**、Shannon と Turing のチェスプログラム(1950, 1953)、Minsky と Edmonds のニューラルネット(1951、パーセプトロン)
- ただしこの後しばらくブランクがある

### 第1次ブーム(1956〜60年代後半)

- 基本原理創出の時代**Artificial Intelligience**という言葉は、1956年のダートマス会議で John McCarthy が初めて使った
- これがAIの公式な誕生
- この時期は、一般問題解決器、定理証明、ゲーム理論、論理学などの基礎が築かれた
- McCarthy は MIT で **LISP言語**を作り、Shakey ロボット(’69-71頃、最初の自走式AI制御ロボット、視覚あり)も登場する

### 第2次ブーム：知識の時代とAI産業化

- 「知識こそ力」の時代
- **エキスパートシステム**——特定問題領域に絞ってAIを実用化する試み
- 専門知識を**プロダクションルール(if-then)、フレーム、セマンティックネットワーク、論理式**で表現
- 医療診断の **MYCIN**(血液感染症)、分子構造解析の **Dendral** が代表例
- 専門家にインタビューして知識を計算機に表現する「**知識エンジニア**」という職業も生まれた。
- 商用では DEC の構成助言システム **R1**(年間4千万ドルの経費節減)が成功し、日本の**第五世代プロジェクト(1982-1992)**、米国の MCC、LISP専用マシンなどが登場、1988年には20億ドル市場に成長した

### 1980年代後半〜90年代：転換

- ニューラルネットの復活、遺伝的アルゴリズム、Bio-inspired アルゴリズムが登場(85年〜)
- 一方で「**単にルールを蓄積するだけでは限界**」が見え、専門家からの知識獲得も難しい(**知識獲得ボトルネック**)という問題が明らかになる
- 90年代からは実世界事例ベースの研究へ
- NNなどの機械学習が音声処理・画像処理で有用となり、実環境とのインタラクション、**自律エージェント、マルチエージェント**へ関心が移る

### 2000年〜現在：第3次ブーム

- **機械学習、特に統計に基づく学習が普及**
- インターネットやIoTによる**ビッグデータ**の存在が背景
- ビッグデータに基づく回帰分析・クラスタリング・NN
- これが「今(第3次ブーム)」
- 実世界・ネットワークを介したエージェント間の相互作用と調整のための社会学習、複雑系・複雑ネットワーク、マルチエージェントシステム研究へと広がっている
