## 双対問題

### 双対問題とは

- ある工場が製品 A・B を作って利益を最大化する、という問題を考える(**主問題**)

- 製品Aの生産量を $x_1$、製品Bの生産量を $x_2$
- 資源1(たとえば材料)は A に1個・B に2個必要で、在庫は16
- 資源2(たとえば労働)は A に2個・B に1個必要で、在庫は14
- 利益は A が20、B が30

$$\max z = 20x_1 + 30x_2 \quad \text{s.t.}\quad x_1 + 2x_2 \le 16, 2x_1 + x_2 \le 14, x_1,x_2 \ge 0$$

- ここで視点を変える
- **「この工場の資源をぜんぶ買い取りたい人」**を想像してみる
- その人は資源1・資源2に値段 $y_1, y_2$ をつけて買おうとする
- 買う側は**支払総額を最小にしたい**$\min; 16y_1 + 14y_2$
- ただし工場が「売るより自分で作ったほうが得」と思わない値段でないと売ってもらえない
- 製品A・Bを作って得られる利益以上の価値を、資源の値段で保証する必要がある
    - A 1個分の資源価値:$1\cdot y_1 + 2\cdot y_2 \ge 20$
    - B 1個分の資源価値:$2\cdot y_1 + 1\cdot y_2 \ge 30$
- 値段は負にできない:$y_1, y_2 \ge 0$
- これが**双対問題**

### 具体例で答えが一致することを見る

- **主問題の最大値も、双対問題の最小値も、ぴったり 260 で一致**
- これが双対性
- 「利益を最大化した値」と「資源を最小コストで評価した値」が一致する、というのは決して偶然ではない

### 一般形での対応関係

- 主問題と双対問題は次のように対応する

$$ \textbf{主問題(P)}:\quad \max_{x}; c^T x \quad \text{s.t.}; Ax \le b,; x \ge 0 $$ $$ \textbf{双対問題(D)}:\quad \min_{y}; b^T y \quad \text{s.t.}; A^T y \ge c,; y \ge 0 $$

|主問題(最大化)|双対問題(最小化)|
|---|---|
|変数 $x$($n$個)|制約 $A^Ty \ge c$($n$本)|
|制約 $Ax \le b$($m$本)|変数 $y$($m$個)|
|目的関数の係数 $c$|制約の右辺 $c$|
|制約の右辺 $b$|目的関数の係数 $b$|
|係数行列 $A$|転置 $A^T$|

### 弱双対定理

- なぜこの双対問題が出てくるのか。これを**ラグランジュ緩和**で導く
- 発想は「主問題の目的関数の **上界** を作り、その上界を一番きつく(=最小に)する」というもの
- 主問題 $\max, c^T x$ s.t. $Ax \le b$ に対し、$y \ge 0$ を使ってラグランジュ関数を作る

$$L(x, y) = c^T x - y^T(Ax - b)$$

ここで制約 $Ax \le b$ は $Ax - b \le 0$ なので、$y \ge 0$ のもとでは $-y^T(Ax-b) \ge 0$。したがって

$$L(x, y) \ge c^T x$$

つまり $L$ は **常に目的関数以上の値(=上界)** になります。次に $L$ を $x$ について最大化します。

$$\max_x L(x,y) = \max_x \big[(c^T - y^T A)x + y^T b\big]$$

$x \ge 0$ のもとでこの値が**発散せず有限にとどまる**には、$x$ の係数が正であってはいけません。すなわち

$$c^T - y^T A \le 0 \quad\Longleftrightarrow\quad A^T y \ge c$$

(これがまさに双対問題の制約!)この条件が満たされるとき、第1項は $x=0$ で最大となり、

$$\max_x L(x,y) = b^T y$$

以上をつなげると、次の**不等式の鎖**ができます。

$$c^T x ;\le; \max_x c^T x = c^T x^* ;\le; L(x^*, y) ;\le; \max_x L(x,y) = b^T y$$

最後にこの上界 $b^T y$ を $y$ について最小化すれば、

$$\max_x c^T x = c^T x^* ;\le; b^T y^* = \min_y b^T y \qquad \textbf{(弱双対定理)}$$

**弱双対定理**は「主問題の任意の実行可能解の値 ≤ 双対問題の任意の実行可能解の値」を保証します。重要な副産物として、**主問題のラグランジュ乗数 $y$ が、そのまま双対問題の設計変数になる**ことが分かります。

### 強双対定理

弱双対定理は $\le$ どまりでしたが、実は線形計画では等号まで到達します。

$$\max_x c^T x = c^T x^* = b^T y^* = \min_y b^T y \qquad \textbf{(強双対定理)}$$

証明の流れ(スライドの基底分解による議論)を要約します。スラック変数で標準形にした $x$ を、基底変数 $x_D$ と非基底変数 $x_I$ に分け、対応して $A = [A_D; A_I]$、$c = \begin{bmatrix}c_D\ c_I\end{bmatrix}$ と分けます。

制約 $A_D x_D + A_I x_I = b$ から $x_D = A_D^{-1}(b - A_I x_I)$。これを目的関数に代入すると

$$z = c_D^T A_D^{-1} b + \big(c_I^T - c_D^T A_D^{-1} A_I\big)x_I \tag{4}$$

最適基底解では非基底変数 $x_I^* = 0$。**最適性の条件**(最大化なので、どの非基底変数を増やしても $z$ が増えてはいけない)は、$x_I$ の係数(被約費用)が非正であること:

$$c_I^T - c_D^T A_D^{-1} A_I \le 0$$

ここで $y^T = c_D^T A_D^{-1}$、すなわち $y = (A_D^{-1})^T c_D$ と置くと、上の条件は $A^T y \ge c$ となり、**$y$ は双対問題の実行可能解**になります。さらに

$$b^T y = b^T (A_D^{-1})^T c_D = (A_D^{-1}b)^T c_D = c_D^T x_D^* = c^T x^*$$

(最後は $x_I^_=0$ より $c^T x^_ = c_D^T x_D^_$)。$b^T y$ が主問題の最適値に一致し、これは双対の下界そのものなので、**この $y$ は双対の最適解 $y^_$**。よって強双対定理 $c^T x^* = b^T y^*$ が示されます。

### 相補性定理

双対性の応用として最も使い出があるのが**相補性定理**です。主問題(P)・双対問題(D)に対し、

> $x^_$ が主問題の最適解、$y^_$ が双対問題の最適解であるための**必要十分条件**は $$(A^T y^* - c)^T x^* = 0 \quad \text{かつ} \quad (Ax^* - b)^T y^* = 0$$

**必要性**:2つの条件 $=0$ を辺々引くと $$(A^Ty^_-c)^Tx^_ - (Ax^_-b)^Ty^_ = -c^Tx^* + b^Ty^* = 0 ;\Rightarrow; c^Tx^* = b^Ty^*$$ 強双対定理より、これは両者が最適であることを意味します。

**十分性**:$x^_, y^_$ が最適なら強双対より $c^Tx^* = b^Ty^_$。よって $$(A^Ty^_-c)^Tx^* = (Ax^_-b)^Ty^_ \tag{11}$$ 一方、実行可能性から $x^* \ge 0,; Ax^_-b \le 0,; y^_ \ge 0,; A^Ty^_-c \ge 0$ なので $$(Ax^_-b)^Ty^* \le 0 \le (A^Ty^_-c)^Tx^_ \tag{12}$$ (11)(12)を合わせると、両辺とも $0$ でなければならず、条件が成立します。

**この定理の "意味" がとても大事です。** 和の各項はすべて符号がそろっている(片方が非負、片方が非正)ので、項ごとに $0$ になります。つまり制約ごとに次が言えます。

- $y_i^* > 0$(影の価格が正)$;\Rightarrow;$ 主問題の第 $i$ 制約は**アクティブ**($A^{(i)}x^* = b_i$、資源を使い切っている)
- 第 $i$ 制約が**非アクティブ**($A^{(i)}x^* < b_i$、資源が余っている)$;\Rightarrow; y_i^* = 0$(余っている資源に値段はつかない)

「余った資源の影の価格はゼロ」「価格がつく資源は使い切っている」という、経済的にも自然な対応です。スラック変数の言葉で言えば「スラック変数(制約からの距離)とラグランジュ乗数のどちらか一方が必ずゼロ」となります。これは**Kuhn–Tucker(KKT)条件そのもの**でもあります。

### 一般の最適化問題への拡張

線形に限らない一般の問題 $$\min_x f(x) \quad \text{s.t.}; g_i(x) \le 0,; i=1,\dots,m$$ でも同じ枠組みが使えます。ラグランジュ関数

$$L(x,\lambda) = f(x) + \sum_{i=1}^m \lambda_i g_i(x),\qquad \lambda_i \ge 0$$

を作ると、最適点で $\lambda_i \ge 0,; g_i(x^_) \le 0$ より $\lambda_i g_i(x^_) \le 0$ なので $L(x^_,\lambda) \le f(x^_)$。さらに $q(\lambda) = \min_x L(x,\lambda)$ とおくと $$q(\lambda) \le f(x^_) \le f(x),\qquad \max_\lambda q(\lambda) = f(x^_) = \min_x f(x)$$

**双対問題による解き方の手順**は次の3ステップです。

1. ラグランジュ乗数 $\lambda$ を導入してラグランジュ関数を定義する
2. $L(x,\lambda)$ を $x$ で最小化して $q(\lambda)$ を求める($\partial L/\partial x = 0$ の結果を代入して $x$ を消去)
3. $\lambda_i \ge 0$ のもとで $q(\lambda)$ を最大化する

複雑な変数 $x$ を消去して $\lambda$ だけの簡単な問題に落とせる点が、双対化のうれしさです。

## 整数計画問題と分枝限定法

### 整数計画問題とは何か

-  線形計画問題に「整数」という条件を足したもの
- **線形計画問題**を考える

$$\max_{\boldsymbol{x}} \ \boldsymbol{c}^T\boldsymbol{x} \quad \text{s.t.} \quad \boldsymbol{a}_i^T\boldsymbol{x} \le a_i$$
- 目的関数も制約も全部一次式（線形）でできた最適化問題
- 目的関数 $\boldsymbol{c}^T\boldsymbol{x}$ をできるだけ大きくする $\boldsymbol{x}$ を、いくつかの不等式制約の中から探す
- ここに **「$\boldsymbol{x}$ は整数でなければならない」($\boldsymbol{x} \in Z^n$)** という条件を付け加えたものが**整数計画問題**

### 0-1整数計画問題は「選ぶ／選ばない」

- さらに整数を $0$ か $1$ のどちらかだけに限定したものを**0-1整数計画問題**と呼ぶ
- これは「$i$ 番目のものを**選ぶ($1$)か選ばない($0$)か**」という選択を表している
- 組合せ最適化の多くの問題がこの形で書ける
- ナップザック問題やTSPなどがこれに該当

### なぜ整数計画は「数段難しい」のか

- 線形計画問題なら、**シンプレックス法**という効率的な解法があり、最適解は必ず多角形の「頂点」に来ることが分かっているので、わりと簡単に求まる
- ところが整数を要求した瞬間、話が変わる
- LPの最適頂点はたいてい整数座標ではない
- では「その近くの格子点なら良いだろう」と思っても、**どの格子点が本当に最良かは簡単には分からない**
- 素朴に全部の格子点を調べようとすると、変数が増えるほど候補が指数的に爆発する

### 線形計画緩和という発想

#### 「整数」という縛りを一旦外す

- 難しいなら、難しくしている原因（整数制約）を一旦外してみよう、というのが**線形計画緩和**
- 整数計画問題から整数制約 $\boldsymbol{x}\in Z^n$ を取り除くと、ただのLP
- このLPを、元のIPの**緩和問題**と呼ぶ
- 「緩和」とは、制約を緩めて実行可能領域を広げること
- 元のIPの解（格子点）は全部このLPの中にも入っているので、緩和問題の領域は元の領域より必ず広い
#### 緩和問題の2つの大事な性質

**性質1：緩和問題の最適値は、元の問題の最適値の「上界」になる。** 
- 領域を広げたのだから、その中で最大化すれば、元の狭い領域での最大値以上になる
- つまり緩和問題の答えは「元の問題の答えがどんなに頑張ってもこれを超えない」という天井（上界）を教えてくれる
- これは**楽観的な見積もり**だと考えると分かりやすい

**性質2：緩和問題の最適解が、たまたま整数（=元の問題でも実行可能）だったら、それはそのまま元の問題の最適解でもある。** 
- 楽観的な見積もりが、ちょうど現実に実現できる解だった場合、それより良い解はあり得ない（上界に達しているから)ので、確定で最適
### 解法の全体像

**近似解**
- 線形計画緩和を解いて、得られた非整数の解を一番近い整数に**丸める**だけ
- 手軽だが、丸めた点が制約からはみ出したり、最良でなかったりするので、あくまで近似

**厳密解**
- 緩和を使いながら、賢く探索する
- 代表的な2つが
	- 線形計画緩和 + **分枝限定法**
	- 線形計画緩和 + **切除平面法**
### 分枝限定法

- 全部の格子点を調べるのは無理だけど、調べなくていい部分は早めに切り捨てて、効率よく全探索する」という考え方

### アルゴリズムの流れ

```
1. 問題リストに初期問題を入れる
2. 実行可能解をひとつ求め、暫定解とする
3. while (問題リストが空でない) {
   3-1. リストから問題をひとつ取り出し、緩和解を求める
   3-2. if (緩和解が暫定解より悪い) continue   // 見込みなし、切り捨て
   3-3. if (緩和解が整数解) 暫定解を入替えて continue  // 解が確定
   3-4. 緩和解の近傍に候補を探し、暫定解より良ければ入替える
   3-5. 変数ひとつを切り上げ／切り捨てして問題を2つに分け、リストに入れる
}
4. 暫定解を最適値として出力
```

- ここで**暫定解**とは「今までに見つかった一番良い実行可能解」のこと
- 探索を進めるうちにこれがだんだん良くなっていく

### 各ステップの意味

**ステップ2（暫定解の用意）**
- まず何でもいいから実行可能な整数解をひとつ持っておく

**3-1（緩和解を求める）** 
- 取り出した部分問題について、整数制約を外したLPを解く

**3-2（限定操作：見込みのない枝を切る）** 
- ここが「限定」の核心
- 緩和解は上界＝楽観的な見積もりであった
- **その楽観的な見積もりですら、今持っている暫定解より悪い**なら、この部分問題をどれだけ細かく調べても暫定解を超える解は出てこない
- だから調べずに切り捨てる
- これにより、本来なら指数的にある探索を大幅に省ける

**3-3（整数解が出たら確定）
- 緩和解がたまたま整数なら、性質2より、その部分問題の最適解が見つかったことになる
- 暫定解を更新して次へ進む

**3-4（近傍探索で暫定解を底上げ）**
- 緩和解が非整数のときも、その近くの整数点を試して、暫定解より良ければ更新
- 暫定解が良くなるほど、3-2でより多くの枝を切れるので、探索が速くなる

**3-5（分枝操作：問題を2つに割る）**
- 緩和解が非整数の変数、たとえば $x_1 = 2.4$ だったとする
- 本当の解では $x_1$ は整数なので、$x_1 \le 2$ か $x_1 \ge 3$ のどちらか
- そこで「$x_1 \le 2$ という制約を足した問題」と「$x_1 \ge 3$ を足した問題」の2つに分けてリストに入れる
- これが切り上げ／切り捨てによる**分枝**
- 分けても両方を合わせれば元の解の候補を漏らさずカバーしているので、最適解を見逃さない

## 切除平面法

### 凸包が分かれば線形計画で解ける

- すべての実行可能な整数解を含む最小の凸集合を**凸包**と呼ぶ
- 整数点だけを囲む一番小さな多面体
- もしこの凸包の不等式表現が手に入れば、その頂点はすべて整数になるので、**ただのLPとしてシンプレックス法で解けてしまう**

### でも凸包を直接求めるのは困難

- 凸包の不等式を全部求めるのは一般に非常に困難
- そこで妥協案として、**毎回ちょっとずつ凸包に近づける**ことを考える
- 具体的には、緩和解が整数でないとき、
	- すべての整数解（実行可能解）は満たすが、今の非整数の緩和解だけは満たさないような不等式（=平面）を1本見つけて、制約に追加
	- この追加した制約が、邪魔な非整数解を「切り落とす」ので**切除平面**と呼ぶ
	- 「すべての実行可能解が満たす不等式」のことを**妥当不等式**と言います。

### チャバタル・ゴモリーの不等式（妥当不等式の作り方）

- では、その切除平面をどう作るのか
- 基本道具が**Chvátal-Gomory不等式**
- 制約条件が次のように与えられているとする

$$\sum_{j=1}^{n} a_{ij}x_j \le b_i, \quad i=1,2,\dots,m$$
- 整数問題の実行可能領域を $S$、緩和したLPの実行可能領域を $\bar{S}$ とする（$S \subset \bar{S}$）
- **手順1：制約を重み付けして足す。** 各制約に非負の重み $u_i \ge 0$ をかけて全部足し合わせると、

$$\sum_{j=1}^{n}\Big(\sum_{i=1}^{m} u_i a_{ij}\Big)x_j \le \sum_{i=1}^{m} u_i b_i$$

- これは $\bar{S}$ の点なら必ず満たすので、$\bar{S}$ の、したがって $S$ の妥当不等式
- **手順2：左辺の係数を切り捨てる。** $x_j \ge 0$ なので、左辺の各係数を切り捨てて小さくすれば、左辺はさらに小さくなります：

$$\sum_{j=1}^{n}\Big\lfloor\sum_{i=1}^{m} u_i a_{ij}\Big\rfloor x_j \le \sum_{j=1}^{n}\Big(\sum_{i=1}^{m} u_i a_{ij}\Big)x_j \le \sum_{i=1}^{m} u_i b_i \tag{4}$$

これもまだ妥当不等式です。

**手順3：右辺を切り捨てる。** ここがミソです。$x_j$ が整数で係数も整数になったので、左辺は**必ず整数値**をとります。整数が右辺以下なら、右辺を切り捨てた整数以下でもあるはずです：

$$\sum_{j=1}^{n}\Big\lfloor\sum_{i=1}^{m} u_i a_{ij}\Big\rfloor x_j \le \Big\lfloor\sum_{i=1}^{m} u_i b_i\Big\rfloor \tag{2}$$

これが Chvátal-Gomory 不等式で、$S$(整数解全体)の妥当不等式になります。元のLP領域 $\bar{S}$ では成り立たない場合がある（右辺を切り捨てた分だけきつい）ので、これを切除平面に使えるわけです。整数だからこそ右辺を切り捨てられる、という整数性の活用がポイントです。

## ゴモリーの小数切除平面法

実際の計算では、シンプレックス法の途中で出てくる表（基底）から自動的に切除平面を作ります（スライド38〜39)。

スラック変数を含めた変数を、基底変数 $\boldsymbol{x}_D$（従属変数）と非基底変数 $\boldsymbol{x}_I$（独立変数）に分けます。シンプレックスの計算では基底変数が

$$x_{Di} = b_i' - \sum_k a_{ik}' x_{Ik}, \quad\text{つまり}\quad x_{Di} + \sum_k a_{ik}' x_{Ik} = b_i'$$

という形で表されます（$\boldsymbol{b}' = A_D^{-1}\boldsymbol{b}$, $A' = A_D^{-1}A_I$）。この各行に上のChvátal-Gomoryの操作（係数と右辺の切り捨て）を施すと妥当不等式が得られ、これを切除平面として追加します。追加したうえで再び緩和問題を解き、まだ整数解でなければまた切除平面を作って追加…と繰り返すことで、整数解にたどり着きます。

### いろいろな整数計画問題（定式化の例）

#### ナップザック問題

- 重さの上限 $C$ のナップザックに、$n$ 個の荷物（荷物 $i$ は重さ $w_i$・価値 $p_i$）から選んで詰め、価値の合計を最大にする問題
- 荷物 $i$ を入れるなら $x_i=1$、入れないなら $x_i=0$ として

$$\max_{\boldsymbol{x}} \sum_{j=1}^{n} p_j x_j \quad \text{s.t.} \quad \sum_{j=1}^{n} w_j x_j \le C, \quad x_j \in {0,1}$$

- 目的が「価値の合計の最大化」、制約が「総重量が容量以下」
- 0-1変数の最も基本的な例

#### 文書要約（ナップザックの一種）

- $n$ 個の文があり、文 $j$ の長さが $l_j$
- 要約の長さの上限が $L$
- 重要な文を選んで、長さ $L$ 以内で重要度の合計が最大の要約を作る
- 文の重要度 $p_j$ は、文が含む「概念」の重要度の合計 $p_j = \sum_i w_i a_{ij}$ で決める（$a_{ij}=1$ は文 $j$ が概念 $i$ を含むの意）
- 文 $j$ を採用するなら $x_j=1$

$$\max_{\boldsymbol{x}} \sum_{j=1}^{n} p_j x_j \quad \text{s.t.} \quad \sum_{j=1}^{n} l_j x_j \le L, \quad x_j \in {0,1}$$

- 重さ→文の長さ、価値→文の重要度、と読み替えるとナップザックそのもの
- 同じ数学構造が全然違う応用に使えるという好例

#### ビンパッキング問題

- 重さ $w_i$ の $n$ 個の品物を、容量 $C$ のビン（箱）に全部詰めるとき、使うビンの数を最小化する問題
- 品物 $j$ をビン $i$ に入れるなら $x_{ij}=1$、ビン $i$ を使うなら $y_i=1$

$$\min_{\boldsymbol{x}} \sum_{j=1}^{n} y_j$$ $$\text{s.t.} \quad \sum_{j=1}^{n} w_j x_{ij} \le C y_i, \quad \sum_{i=1}^{n} x_{ij} = 1, \quad x_{ij}, y_i \in {0,1}$$

ポイントは2つの制約です。$\sum_j w_j x_{ij} \le C y_i$ は「ビン $i$ の中身の総重量は容量以下、ただしビンを使っていない($y_i=0$)なら右辺が $0$ になるので何も入れられない」という巧みな表現。$\sum_i x_{ij} = 1$ は「品物 $j$ はどれかちょうど1つのビンに入る」という制約です。

#### 割当問題

$m$ 人の子供に $n$ 種類の果物を配り、満足度の合計を最大にします（スライド48〜49)。果物 $j$ は $u_j$ 個あり、子供 $i$ が果物 $j$ をもらうと満足度 $p_{ij}$。子供 $i$ に果物 $j$ を渡すなら $x_{ij}=1$:

$$\max_{\boldsymbol{x}} \sum_{i=1}^{m}\sum_{j=1}^{n} p_{ij} x_{ij}$$ $$\text{s.t.} \quad \sum_{j=1}^{n} x_{ij} = 1 \ (\text{各子供は1つもらう}), \quad \sum_{i=1}^{m} x_{ij} \le u_j \ (\text{果物 } j \text{ は在庫以下}), \quad x_{ij}\in{0,1}$$

「各子供はちょうど1つ」「各果物は在庫数まで」という2方向の制約で配分を表します。

#### 1機械スケジューリング問題

- 1台の機械で $n$ 個の仕事を順番に処理し、納期遅れを最小に
- 仕事 $i$ の処理時間 $p_i$、納期 $d_i$。$i$ を $j$ より先に処理するなら $x_{ij}=1$、開始時刻 $s_i$、遅れ $t_i$。
- ここで重要なのが**Big-M法**というテクニック
- 「$i$ が $j$ より先なら $s_i + p_i \le s_j$」という条件を、大きな定数 $M$ を使って

$$s_i + p_i \le s_j + M(1 - x_{ij})$$

と書きます。$x_{ij}=1$（$i$ が先）なら右辺の $M$ 項が消えて $s_i+p_i\le s_j$ が効きます。$x_{ij}=0$ なら右辺が $s_j+M$ になり、$M$ が十分大きいので制約は実質無効になります（=この向きの順序は課されない)。これで「if-then」的な論理を一本の線形不等式で表現できます。全体は：

$$\min \sum_{i=1}^{n} t_i$$ $$\text{s.t.} \quad s_i + p_i \le s_j + M(1-x_{ij}), \quad x_{ij}+x_{ji}=1, \quad s_i+p_i-d_i \le t_i, \quad s_i, t_i \ge 0$$

$x_{ij}+x_{ji}=1$ は「2つの仕事には必ず前後関係がある」、$s_i+p_i-d_i\le t_i$ は「終了時刻が納期を超えた分が遅れ $t_i$」を表します。

#### 巡回セールスマン問題（TSP）

$n$ 都市を一度ずつ訪れて元に戻る最短経路を求めます（スライド54〜57)。都市 $i$ から $j$ へ行くなら $x_{ij}=1$:

$$\min \sum_{i\in V}\sum_{j\in V, j\neq i} d_{ij} x_{ij}$$ $$\text{s.t.} \quad \sum_{j} x_{ij} = 1 \ (\text{各都市から出る経路は1本}), \quad \sum_{j} x_{ji} = 1 \ (\text{各都市に入る経路は1本})$$ $$\sum_{(i,j)\in E(S)} x_{ij} \le |S| - 1 \quad (\text{部分巡回路の除去})$$

最初の2つの制約だけだと、「全体で1つの大きな輪」ではなく「小さな輪が複数できる」解（部分巡回路＝subtour）が許されてしまいます。3つ目の**部分巡回路除去制約**がそれを禁じます。都市の部分集合 $S$ の内部だけで閉じた輪を作ると、辺の数がノード数 $|S|$ と一致してしまう。そこで内部の辺の数を $|S|-1$ 以下に抑えることで、$S$ の中だけで閉じる輪を禁止し、必ず外から入って外へ出る形を強制します（スライド56〜57の説明）。これがTSP定式化の難所です。

### 分枝限定法の一般論

#### 組合せ最適化と分枝限定法

- 組合せ最適化とは、目的関数 $f(x)$ を最小（または最大）にする $x$ を、$x \in S$($S$ は有限個または可算無限個の離散集合）の中から選ぶ問題
- 分枝限定法はその代表的解法

#### 列挙木と枝の意味

配分問題（総量 $K$ の資源を $N$ 工場に整数で分け、収益合計を最大化）を例に（スライド63〜67)、素朴な解法は**全列挙**です。$x_1$ の値を決め、次に $x_2$、次に $x_3$…と枝分かれさせると、葉が許容解になる**列挙木**ができます。

各枝は「解に $x_1=1$ を含める」「$x_1=2$ を含める」…という**部分問題への分解**を意味します（スライド66）。$P_0$ から $P_1, P_2, P_3$ という部分問題に分けているわけです。分枝限定法の基本は、この木を作りつつ、**調べる必要のない部分木を途中で打ち切る**ことです（スライド67：分枝操作と限定操作)。

#### 終端（限定）の2つの条件

部分問題を打ち切る（終端する）条件は2種類あります（スライド68）。

**下界値による限定（スライド69〜72）：** 部分問題 $P_i$ の緩和問題 $P_i'$ を解いて得る最適値 $g(P_i)$ を**下界値**と呼びます。これは $g(P_i) \le f(P_i)$ を満たす、$P_i$ の最適値の**楽観的な見積もり**です。性質は次の3つ：

- $P_i'$ に許容解がなければ、$P_i$ にもない（楽観的に見ても解がないなら、現実にもない）。
- $P_i'$ の最適解がたまたま $P_i$ で実行可能なら、それが $P_i$ の最適解（楽観的な見積もりが実現できたなら、それが答え)。
- これまでに見つかった暫定値 $z$ について $g(P_i) \ge z$ なら、$P_i$ とその先には解はない（楽観的に見積もっても今の暫定解に勝てないなら、調べる価値なし)。

3つ目が、前半の「3-2 緩和解が暫定解より悪ければ切る」を最小化の言葉で言い直したものです。

**優越関係による限定（スライド73）：** 2つの部分問題に $f(P_i) \ge f(P_j)$（$P_j$ の方が必ず良い）という関係があるとき「$P_j$ は $P_i$ に優越する」と言います。$P_i$ に優越する $P_j$ がすでにあれば、$P_i$ は調べる必要がないので終端できます。

#### 形式的な処理手順

記号（スライド74）：$A$ はアクティブリスト（生成済みだがまだ分解も終端もしていない部分問題)、$N$ は生成済み部分問題全体、$z$ は暫定値、$O$ は暫定解、$s(A)$ は $A$ から1つ選ぶ探索関数。

手順（スライド75）：

```
1. A = {P0}, N = {P0}, z = ∞, O = ∅
2. A が空でない間:
   - P_i = s(A)  で1つ選ぶ
   - u(P_i) < z なら（より良い許容解が見つかったら）z と O を更新
   - 緩和問題 P'_i に許容解がなければ終端
   - g(P_i) = u(P_i) なら（下界と上界が一致＝解が確定）終端
   - g(P_i) ≥ z なら（下界値による限定）終端
   - 優越する P_j があれば（優越関係による限定）終端
   - どれにも当てはまらなければ子節点を作り、A と N に追加（分枝）
   - A から P_i を除く
3. 最適値は z、最適解は O
```

前半の具体的なwhileループを、限定条件を明示して一般化したものになっています。

#### 探索順序の選び方

最後に、$s(A)$ で「どの部分問題から調べるか」の戦略です（スライド76)。

**盲目的探索：** 深さ優先探索（深いノードを優先＝とにかく葉まで早く潜って暫定解を得る）と、幅優先探索（浅いノードを優先)。

**ヒューリスティック探索：** 最良下界探索（best-bound search、下界値が最も良いノードを優先）。$A^*$ 探索がこの仲間です。

探索順序によって、暫定解が早く良くなれば限定操作がよく効き、無駄な探索を減らせます。
