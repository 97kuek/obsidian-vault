# PDF → Markdown 変換

`00_Inbox/` に置かれた PDF を精査し、Markdown に変換したうえで元の PDF を削除する。

## 原則

- 変換前に対象 PDF の一覧を提示し、ユーザーの承認を得てから実行する。
- 元の PDF は変換後に削除する（削除前に確認を取る）。
- 数式・図表・コードブロックは可能な限り保持する。図は `[図: キャプション]` のようにプレースホルダーで代替する。
- 変換後のファイルは `/inbox` コマンドで分類・整理する前段として扱う。

## 手順

1. `00_Inbox/` 直下の PDF ファイルを列挙する。PDF が見つからなければ「対象PDFがありません」と報告して終了。
2. 対象一覧をユーザーに提示し、変換するものを確認する（全件 or 選択）。
3. 各 PDF について `anthropic-skills:pdf` スキルを使ってテキストと構造を抽出し、以下のルールで Markdown に変換する：
   - タイトル → `# タイトル`
   - セクション見出し → `##`・`###`（階層を維持）
   - 箇条書き・番号付きリスト → そのまま保持
   - 数式 → `$$...$$`（ブロック）または `$...$`（インライン）
   - 図・画像 → `[図: キャプションまたはalt text]`
   - 表 → Markdownテーブル形式
4. frontmatter を付与する：
   ```yaml
   ---
   date: YYYY-MM-DD（変換実行日）
   tags:
     - inbox
   source: "元PDFファイル名"
   ---
   ```
5. `00_Inbox/元PDFのベース名.md` として保存する。
6. Markdown の内容をユーザーに確認してもらい、問題なければ元の PDF を削除する。
7. 全件完了後にサマリーを出す。

## 出力サマリー

```markdown
## PDF → Markdown 変換完了（YYYY-MM-DD）
- 変換: N件
  - 元PDF → 保存先Md（ページ数、変換できなかった要素）
- 削除したPDF: N件
- 次のステップ: /inbox で整理する
```
