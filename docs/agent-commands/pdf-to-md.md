# PDF → Markdown 変換

PDFファイルを読み込み、Markdown形式に変換して `00_Inbox/` に保存する。

## 原則

- 変換後のファイルはそのまま確定ではなく、`/inbox` で整理する前段として扱う。
- 数式・図表・コードブロックは可能な限り保持する。図は `[図: キャプション]` のようにプレースホルダーで代替する。
- 変換後にユーザーへ確認し、必要なら調整してから保存する。
- 元のPDFは削除しない。

## 手順

1. ユーザーが指定したPDFファイルパスを確認する。パスが省略された場合は確認する。
2. `anthropic-skills:pdf` スキルを使ってPDFを読み込み、テキストと構造を抽出する。
3. 以下のルールで Markdown に変換する：
   - タイトル → `# タイトル`
   - セクション見出し → `##`・`###`（階層を維持）
   - 箇条書き・番号付きリスト → そのまま保持
   - 数式 → `$$...$$`（ブロック）または `$...$`（インライン）
   - 図・画像 → `[図: キャプションまたはalt text]`
   - 表 → Markdownテーブル形式
4. frontmatter を付与する：
   ```yaml
   ---
   date: YYYY-MM-DD（変換実行日）
   tags:
     - inbox
   source: "元PDFファイル名またはパス"
   ---
   ```
5. ファイル名を `元PDFのベース名.md` として `00_Inbox/` に保存する。
6. 変換結果のサマリーを出す。

## 出力サマリー

```markdown
## PDF → Markdown 変換完了（YYYY-MM-DD）
- 元PDF: パス
- 保存先: 00_Inbox/ファイル名.md
- ページ数: N
- 変換できなかった要素: （図N点、表N点など）
- 次のステップ: /inbox で整理する
```
