# Vault GC

Vault の重い棚卸しを行う。月次から四半期を目安に、重複・統合・分割・陳腐化を検出し、整理案を提案する。

## 原則

- この手順は提案までで止まる。
- 統合・削除・リネームはユーザー承認後に実行する。
- 統合・削除の前に対象ファイルを必ず読む。
- 知識を消すより、つなぐ・束ねることを優先する。
- 破壊的操作の前に作業前コミットを提案する。

## 事前準備

`VAULT_INDEX.md`、vault の命名規則、主要 MOC を読む。機械的な候補抽出には `tools/vault-gc.ps1` を使う。

```powershell
.\tools\vault-gc.ps1
```

## 見る観点

### 統合候補

- 同義語・表記ゆれ。
- 親子関係で内容が大きく重複しているノート。
- 同じ論文・概念について複数のメモがあるケース。

各ペアについて、残すノート、移植する内容、リンク方針を提示する。

### 分割候補

- 永続ノートとして長すぎる。
- 1ノートに複数の独立概念が入っている。
- 見出しごとに独立ノート化できる。

### 陳腐化・棚上げ候補

- 長期間更新されていない。
- 被リンクが少ない、または MOC から孤立している。
- 現在の研究・学習テーマから外れている。

即削除せず、MOC へ接続、`40_Archives/` へ移動、統合後削除のいずれかを提案する。

## 出力

```markdown
## Vault GC 提案（YYYY-MM-DD）

### 統合候補
| 主ノート | 統合元 | 理由 | 移植/リンク方針 |

### 分割候補
| ノート | 含む概念 | 切り出し案 |

### 陳腐化・棚上げ候補
| ノート | 最終更新 | 被リンク | 推奨アクション |

### 実行プラン（承認後）
1. 作業前コミット
2. 統合・分割・移動
3. 被リンク張り替え
4. MOC 更新
5. VAULT_INDEX 更新
6. 切れリンク確認
```
