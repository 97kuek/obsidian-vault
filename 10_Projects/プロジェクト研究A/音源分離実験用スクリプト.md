TF-Locoformerの事前学習済みモデルで音源分離・音声強調を試すためのユーティリティスクリプト群。

## 前提

```bash
# tmuxセッションを作成（SSH切断対策）
tmux new -s sep

# conda環境を有効化（tmux内では毎回必要）
source ~/anaconda3/etc/profile.d/conda.sh
conda activate tflocoformer
```

## スクリプト一覧

|スクリプト|用途|
|---|---|
|`convert_audio.py`|任意の音声ファイルをモノラルWAVに変換|
|`mix_audio.py`|2つのWAVを重ね合わせて混合音声を生成|
|`generate_tts.py`|gTTSでテスト用音声を生成（混合音声も同時に作成可能）|

スクリプトの場所：`/home/kueki/audio/scripts/`

## 対応モデル

|データセット|sample_rate|num_spk|タスク|
|---|---|---|---|
|WSJ0-2mix|8000 Hz|2|話者分離（クリーン）|
|WHAMR!|8000 Hz|2|話者分離（ノイズ+残響）|
|Libri2mix|8000 Hz|2|話者分離（クリーン）|
|DNS-Interspeech2020|16000 Hz|1|音声強調（ノイズ除去）|

## スクリプトの使い方

### convert_audio.py

```bash
# 8kHz変換（WSJ0-2mix / WHAMR! / Libri2mix 用）
python /home/kueki/audio/scripts/convert_audio.py input.m4a /home/kueki/audio/output_8k.wav

# 16kHz変換（DNS用）
python /home/kueki/audio/scripts/convert_audio.py input.m4a /home/kueki/audio/output_16k.wav --sr 16000
```

### mix_audio.py

```bash
python /home/kueki/audio/scripts/mix_audio.py /home/kueki/audio/a_8k.wav /home/kueki/audio/b_8k.wav /home/kueki/audio/mix_8k.wav
```

注意：2つの入力はサンプリングレートが同じであること。引数はフルパスで指定すること。

### generate_tts.py

```bash
# デフォルト（英語 US + UK、8kHz、混合音声も生成）
python /home/kueki/audio/scripts/generate_tts.py /home/kueki/audio/tts_test

# テキストを指定
python /home/kueki/audio/scripts/generate_tts.py /home/kueki/audio/tts_test \
    --text_a "Hello world" --text_b "Good morning"

# 日本語で生成
python /home/kueki/audio/scripts/generate_tts.py /home/kueki/audio/tts_test \
    --lang ja --tld_a com --tld_b com

# DNS用（16kHz、1話者のみ）
python /home/kueki/audio/scripts/generate_tts.py /home/kueki/audio/tts_dns \
    --sr 16000 --no_mix
```

生成されるファイル（8kHzの場合）：

```
tts_test/
├── tts_a.mp3
├── tts_b.mp3
├── tts_a_8k.wav
├── tts_b_8k.wav
└── tts_mix_8k.wav
```

## 音源分離の実行

**重要：`separate.py` は `egs2/wsj0_2mix/enh1/` にしかない。** 他のデータセットのモデルでもこの `separate.py` を使い、`--model_path` でモデルを切り替える。

```bash
PYTHONPATH=/home/kueki/espnet:$PYTHONPATH python /home/kueki/tf-locoformer/egs2/wsj0_2mix/enh1/separate.py \
    --model_path <モデルのパス> \
    --audio_path <入力音声のパス> \
    --audio_output_dir <出力先ディレクトリ>
```

### 各モデルの具体的なコマンド

**WSJ0-2mix（8kHz、2話者分離）**

```bash

    --model_path /home/kueki/tf-locoformer/egs2/wsj0_2mix/enh1/exp/enh_train_enh_tflocoformer_raw/valid.loss.ave_5best.pth \
```

## tmux操作

|操作|コマンド|
|---|---|
|セッション作成|`tmux new -s sep`|
|デタッチ（抜ける）|`Ctrl+b` → `d`|
|アタッチ（戻る）|`tmux attach -t sep`|
|セッション一覧|`tmux ls`|
|セッション終了|セッション内で `exit`|

tmux内では毎回最初に以下を実行：

```bash
source ~/anaconda3/etc/profile.d/conda.sh
conda activate tflocoformer
```

## 注意事項

- 分離実行前に `nvidia-smi` でGPUが空いているか確認する
- スクリプトの引数はフルパスで指定する（相対パスだとカレントディレクトリ依存でエラーになる）
- モデルは英語音声で学習されているため、日本語音声では性能が低下する
- 同一話者の音声では分離がうまくいかない（声質の違いが分離の手がかりになるため）
- WHAMR!には通常版（6層）とsmall版（4層）の2つのモデルがある

## トラブルシューティング

| 症状                                | 対処                                                |
| --------------------------------- | ------------------------------------------------- |
| `No module named 'pkg_resources'` | `pip install setuptools==69.5.1`                  |
| `conda activate` が効かない            | `source ~/anaconda3/etc/profile.d/conda.sh` を先に実行 |
| NAS上で `Permission denied`         | ローカル（`/home/kueki/`）で作業する                         |
| `.pth` ファイルが読めない（ASCII text）      | `cd /home/kueki/tf-locoformer && git lfs pull`    |
| `ffmpeg` が見つからない                  | `conda install -c conda-forge ffmpeg`             |
| `separate.py` が見つからない             | `egs2/wsj0_2mix/enh1/separate.py` を使う             |
