## 対応データセットの特徴

TF-Locoformerのリポジトリでは4つのデータセットに対応している。
### WSJ0-2mix

- **タスク：** 2話者の音源分離（クリーン環境）
- **言語：** 英語（Wall Street Journalコーパスの朗読音声）
- **データ量：** 学習20,000 / 検証5,000 / テスト3,000 ペア（学習約30h）
- **サンプリングレート：** 8kHz / 16kHz
- **特徴：** 音源分離研究の最も標準的なベンチマーク。スタジオ録音のクリーンな音声を合成的にミックスしたもの。ノイズ・残響なし。ただし汎化性能の低さが指摘されている
- **入手：** 有償（LDC: CSR-I WSJ0 Complete, LDC93S6A）
### Libri2mix

- **タスク：** 2話者の音源分離（クリーン or ノイズあり）
- **言語：** 英語（LibriSpeech由来、オーディオブックの朗読音声）
- **特徴：** WSJ0-2mixの汎化性能問題を解決するために作られたオープンソースの代替データセット。話者数が多く多様性がある。WHAM!のノイズを加えたバージョンもある
- **入手：** 無料（LibriSpeechベース）
- **TF-Locoformerでの設定：** train-360サブセット、noise-free条件（clean）、8kHz
### WHAMR!

- **タスク：** 2話者の音源分離 + 残響除去（ノイズ＋残響環境）
- **言語：** 英語（WSJ0ベース）
- **特徴：** WSJ0-2mixにWHAM!（カフェ、レストラン、バーなどの環境ノイズ）と残響（Room Impulse Response）を追加したデータセット。よりリアルなシナリオを想定
- **入手：** WSJ0ライセンスが必要
- **TF-Locoformerでの設定：** noisy reverberant speech separation with dereverberation
### DNS-Interspeech2020

- **タスク：** 単一話者の音声強調（ノイズ除去）
- **言語：** 英語が中心
- **特徴：** Microsoftが主催したDeep Noise Suppression Challengeのデータセット。クリーン音声とノイズをさまざまなSNR（0〜40dB）でミックスして合成。リアルタイム処理を意識した評価基準（主観評価MOS）
- **入手：** 無料（https://github.com/microsoft/DNS-Challenge）
- **TF-Locoformerでの設定：** 3000時間の合成ノイズ音声（SNR: -5〜15dB）

## 実験1：日本語・同一話者

### 入力

自分で2回録音した日本語音声（m4a → 8kHz WAVに変換後、重ね合わせ）

### 音声生成スクリプト

```python
import numpy as np
import soundfile as sf
from pydub import AudioSegment

def convert_to_8k_mono(input_path, output_path):
    audio = AudioSegment.from_file(input_path, format="m4a")
    audio = audio.set_channels(1).set_frame_rate(8000).set_sample_width(2)
    audio.export(output_path, format="wav")

convert_to_8k_mono("test1_raw.m4a", "speech_a_8k.wav")
convert_to_8k_mono("test2_raw.m4a", "speech_b_8k.wav")

a, sr = sf.read("speech_a_8k.wav")
b, _ = sf.read("speech_b_8k.wav")
max_len = max(len(a), len(b))
a = np.pad(a, (0, max_len - len(a)))
b = np.pad(b, (0, max_len - len(b)))
mix = a + b
mix = mix / np.max(np.abs(mix))
sf.write("mix_8k.wav", mix.astype(np.float32), 8000)
```

### 分離コマンド

```bash
cd /home/kueki/tf-locoformer/egs2/wsj0_2mix/enh1
PYTHONPATH=/home/kueki/espnet:$PYTHONPATH python separate.py \
    --model_path ./exp/enh_train_enh_tflocoformer_raw/valid.loss.ave_5best.pth \
    --audio_path /home/kueki/audio/mix_8k.wav \
    --audio_output_dir /home/kueki/audio/separated2
```

### 結果

**分離失敗。** ほとんど分離できていなかった。

### 考察

- 同一話者の声なので声質の違いがなく、分離の手がかりを得られなかった
- モデルが英語（WSJ0-2mix）で学習されており、日本語音声との特徴の差も影響した可能性
- この2つの要因が複合的に作用していると考えられる

## 実験2：英語TTS・異なる話者

### 入力

Google TTSで生成した2つの英語音声（アメリカ英語 + イギリス英語）を重ね合わせ

### 音声生成スクリプト

```python
from gtts import gTTS
from pydub import AudioSegment
import numpy as np
import soundfile as sf

text_a = "The weather was really nice today, so I went for a walk in the morning. The cherry blossoms were still blooming in the park, and I took some photos."
text_b = "I recently found a new cafe and the coffee there is really good. The place is quiet and perfect for studying. I'm planning to go again next week."

tts_a = gTTS(text=text_a, lang="en", tld="us")
tts_a.save("tts_a.mp3")
tts_b = gTTS(text=text_b, lang="en", tld="co.uk")
tts_b.save("tts_b.mp3")

for name in ["tts_a", "tts_b"]:
    audio = AudioSegment.from_file(f"{name}.mp3")
    audio = audio.set_channels(1).set_frame_rate(8000).set_sample_width(2)
    audio.export(f"{name}_8k.wav", format="wav")

a, sr = sf.read("tts_a_8k.wav")
b, _ = sf.read("tts_b_8k.wav")
max_len = max(len(a), len(b))
a = np.pad(a, (0, max_len - len(a)))
b = np.pad(b, (0, max_len - len(b)))
mix = a + b
mix = mix / np.max(np.abs(mix))
sf.write("tts_mix_8k.wav", mix.astype(np.float32), 8000)
```

### 分離コマンド

```bash
cd /home/kueki/tf-locoformer/egs2/wsj0_2mix/enh1
PYTHONPATH=/home/kueki/espnet:$PYTHONPATH python separate.py \
    --model_path ./exp/enh_train_enh_tflocoformer_raw/valid.loss.ave_5best.pth \
    --audio_path /home/kueki/audio/tts_mix_8k.wav \
    --audio_output_dir /home/kueki/audio/separated3
```

### 結果

**分離成功。** 2話者の音声がきれいに分離された。

### 考察

- 英語音声かつ声質が異なる2話者で、学習データ（WSJ0-2mix）の条件に近かったため成功
- モデルの分離性能は学習データとの類似度に大きく依存する
## わかったこと

- TF-Locoformerは**同時に重なった音声を分離する**モデルであり、交互に話す対話音声の話者分割（ダイアライゼーション）には向かない
- WSJ0-2mix学習済みモデルは**英語・8kHz・2話者**の条件で性能を発揮する
- 同一話者や異なる言語では性能が大きく低下する
- 日本語音声の分離には日本語データセットでの再学習が必要と考えられる
- 交互に話す対話音声の話者分離には、pyannote-audioやWhisperXなどのダイアライゼーション手法が適している
