## コンセプト

**PARAメソッド** をベースに、4つのカテゴリで知識を管理する。

> **P**rojects — **A**reas — **R**esources — **A**rchives

すべてのノートはこの4分類のどれかに属する。分類に迷ったときは「このノートはいつ使うか」で判断する。

| 問い | 分類 |
|---|---|
| 特定のゴールに向けて今動いている | Projects |
| 継続的に維持・学習しているトピック | Areas |
| いつか使えそうな参考資料 | Resources |
| 終わった・不要になった | Archives |

## フォルダ構成（PARA）

```
obsidian-vault/
├── 00_Inbox/          ← キャプチャした未整理ノートの一時置き場
├── 10_Projects/       ← 期限・ゴールのあるアクティブな作業
├── 20_Areas/          ← 継続的に管理するトピック
├── 30_Resources/      ← 将来再利用できる参考資料・論文
├── 40_Archives/       ← 完了・不要になったノートの保管庫
├── 99_Templates/      ← Templaterテンプレート（実ノートではない）
├── Home.md            ← ダッシュボード
├── CLAUDE.md          ← Claude Code用の作業規約（自動参照）
└── README.md          ← このファイル
```

### 00_Inbox — 一時置き場

思いついたこと・気になったリンク・メモなどをとにかくここに書く。  
**週1回** レビューして、適切なフォルダに振り分けるか削除する。

### 10_Projects — プロジェクト

期限やゴールが明確な作業単位。プロジェクトごとにサブフォルダを作り、MOCで管理する。  
完了したプロジェクトは `40_Archives/` に移動する（削除しない）。

```
10_Projects/
├── 【MOC】10_Projects.md     ← Projects全体のハブ
└── プロジェクト研究A/
    ├── 【MOC】プロジェクト研究A.md   ← プロジェクトの目的・タスク・リンク集
    ├── 20260511_音源分離実験用スクリプトの作成.md
    └── 使用モデルと学習データセットの特徴.md
```

### 20_Areas — エリア

終わりのない継続的な関心事。スキル・習慣・知識領域など。

```
20_Areas/
├── 【MOC】20_Areas.md
├── ソフトウェア開発/
│   ├── 【MOC】ソフトウェア開発.md
│   ├── Git・GitHub.md
│   └── AI駆動開発.md
└── 機械学習・深層学習/
    ├── 【MOC】機械学習・深層学習.md
    ├── ロジスティック回帰.md
    └── B4勉強会/
        ├── 【MOC】B4勉強会.md
        └── 20260423_勉強会02.md
```

### 30_Resources — リソース

論文・書籍・外部資料のメモ。特定プロジェクトに紐づくものでもここに置いてよい。  
PDFは `30_Resources/` に置き、対応する読書メモ（`_メモ.md`）をセットで作成する。

```
30_Resources/
├── 【MOC】30_Resources.md
└── 音源分離/
    ├── TF-Locoformer.pdf
    ├── TF-Locoformer_メモ.md    ← PDFと1対1で対応する読書メモ
    └── TasNet.pdf
```

### 40_Archives — アーカイブ

削除せずに保管しておく場所。検索はできるが基本的に参照しない。

### 99_Templates — テンプレート

Templater プラグインが使うテンプレートファイル。**直接編集しない**。  
新規ノート作成時はTemplaterから呼び出す。

## 日々の使い方

### 典型的な1日の流れ

```
1. Home.md を開く
   → Inboxの未整理ノートと未完了タスクを確認

2. 作業する
   → 実験ノート・作業メモは 10_Projects/該当プロジェクト/ に作成
   → 気づき・参考リンクは 00_Inbox/ に一時保存

3. 週1回：Inboxレビュー
   → 各メモを PARA のどれかに振り分ける
   → 振り分け先が決まらないものは削除を検討
```

### 新しいノートの作り方

Templater を使う（`Ctrl+T` またはコマンドパレットから「Templater: Open Insert Template Modal」）。

| 状況 | 使うテンプレート |
|---|---|
| 論文を読む | `論文ノート用.md` |
| 実験・作業ログを書く | `実験ノート用.md` |
| 授業・勉強会のメモ | `授業ノート用.md` |
| プロジェクト内のノート | `プロジェクト用.md` |
| 新しいMOCを作る | `【MOC】.md` |

## ファイル命名規則

### 通常ノート・実験ノート
```
YYYYMMDD_タイトル.md
```
例：`20260511_音源分離実験用スクリプトの作成.md`

**理由：** 日付プレフィックスにより、ファイル名だけで「いつ書いたか」が分かる。時系列ソートも自動で正しくなる。

### MOCファイル
```
【MOC】タイトル.md
```
例：`【MOC】プロジェクト研究A.md`

**理由：** `【MOC】` プレフィックスにより、フォルダ内でMOCが先頭に表示され、他のノートと視覚的に区別できる。

### 論文読書メモ
```
論文タイトル略称_メモ.md
```
例：`TF-Locoformer_メモ.md`

**理由：** PDFファイル名と対応させることで、どのPDFのメモかが一目で分かる。

## テンプレートの使い方

### 論文ノート（`論文ノート用.md`）

論文1本につき1ファイル。PDFと同じフォルダに置き、`![[論文名.pdf]]` で埋め込む。

```markdown
---
date: 2026-05-12
tags: [paper]
title: "論文タイトル"
authors: "著者名"
year: 2023
status: read
---
## 一言要約
## 提案手法
## 気づき・メモ
```

### 実験ノート（`実験ノート用.md`）

実験・作業1セッションにつき1ファイル。`10_Projects/プロジェクト名/` に置く。

```markdown
---
date: 2026-05-12
tags: [experiment]
project: プロジェクト研究A
environment: g12（RTX 2080 Ti）
---
## 実験概要
## 手順
## 結果
## 次のアクション
```

## MOCとは

**MOC（Map of Content）** はノート同士をつなぐハブファイル。  
フォルダの代わりにリンクで構造を表現する。

### MOCの役割

- 関連ノートへのリンクを集約する
- プロジェクトの目的・進捗・タスクをまとめる
- 上位MOCへのリンクで階層を形成する

### MOC階層

```
Home.md（ダッシュボード）
├── 【MOC】10_Projects
│   └── 【MOC】プロジェクト研究A   ← 実験ノート・論文・タスクが集まる
├── 【MOC】20_Areas
│   ├── 【MOC】ソフトウェア開発
│   └── 【MOC】機械学習・深層学習
│       └── 【MOC】B4勉強会
└── 【MOC】30_Resources
    └── 音源分離フォルダ内の読書メモ群
```

### MOCを作るタイミング

- 新しいプロジェクトを始めたとき
- 特定トピックのノートが3つ以上になったとき
- 関連ノートを一か所から参照したくなったとき

## タグ体系

| カテゴリ | タグ | 用途 |
|---|---|---|
| システム | `MOC` | Map of Content |
| システム | `inbox` | 未整理ノート |
| システム | `archive` | アーカイブ済み |
| ノート種別 | `paper` | 論文読書メモ |
| ノート種別 | `experiment` | 実験・作業ログ |
| ノート種別 | `lecture` | 授業・勉強会ノート |
| ノート種別 | `project` | プロジェクトノート |
| ノート種別 | `dataset` | データセット情報 |
| トピック | `機械学習` | 機械学習全般 |
| トピック | `深層学習` | 深層学習全般 |
| トピック | `音源分離` | 音源分離・音声処理 |
| トピック | `ソフトウェア開発` | ソフトウェア開発全般 |
| モデル | `transformer` | Transformer |
| モデル | `bert` / `ai` 等 | 各モデル・技術 |

**ルール：** タグは**小文字英語または日本語**に統一。`AI` → `ai`、`Git` → `git`。

## プラグイン一覧

| プラグイン | 用途 | 重要度 |
|---|---|---|
| **Dataview** | `Home.md` のクエリ（タスク一覧・更新履歴など） | 必須 |
| **Templater** | テンプレートの変数展開（`<% tp.* %>`） | 必須 |
| **Local REST API** | Claude Code との MCP 連携 | 必須 |

## Claudeとの連携

Claude Code（CLIまたはデスクトップアプリ）からこのVaultを直接操作できる。

### 基本的な使い方

```bash
# Vaultフォルダで起動
cd C:\Users\yamashita\Documents\obsidian-vault
claude
```

### できること

- ノートの作成・編集・リネーム
- Vault全体の検索（キーワード・意味検索）
- MOCの構造整理
- 実験ログの整形・要約
- 論文読書メモの作成補助

### Claudeに依頼するときのコツ

```
良い例：「プロジェクト研究AのMOCに今日の実験ノートへのリンクを追加して」
良い例：「音源分離に関連するノートをすべて教えて」
良い例：「30_Resourcesに新しい論文メモを作って。論文名はXXX」

悪い例：「適当にノートを整理して」（範囲が広すぎる）
```

### 大規模操作の前に

複数ファイルの作成・削除・リネームを依頼する前に必ずcommitしておく。

```bash
git add -A && git commit -m "作業前スナップショット"
```

## Gitワークフロー

Vaultはgitで管理されている。obsidian-gitプラグインが定期的に自動commitする。

### ブランチ構成

- `main` — 常に安定した状態を保つ

### 手動操作が必要なとき

```bash
# 状態確認
git status

# 差分確認（Claudeの作業内容チェックに使う）
git diff HEAD

# コミット
git add -A
git commit -m "コミットメッセージ"

# プッシュ
git push origin main
```

### .gitignore の対象

- `.obsidian/workspace.json` — ワークスペース状態（頻繁に変わるため）
- `.obsidian/plugins/obsidian-local-rest-api/` — APIキー（セキュリティ）
- `.obsidian/plugins/mcp-tools/` — MCPツール設定
- `.trash/` — Obsidianのゴミ箱
