## インターネット層

### 概要

#### 「インターネット」とは

- Inter-net = **ネットワークのネットワーク**。
- 複数のネットワークが地球規模で相互接続した集合体である。
- 1組織内のLANではなく、それらが無数につながった「全体」が The Internet である。
- 魅力は「遠くまでつながる」こと。接続端末なら家から世界中の端末と通信でき、距離によらず通信料金は基本的に同じである（物理的には海底ケーブル等が支える）。

### 【問題1】(スライド9)

> 世界中に遍在する無数のコンピュータが物理層・データリンク層で接続されている。これらの相互通信を実現するには何が必要か。

- **答え：「目的地のアドレス」と「経路（経路表）」の2つ。**
- 遠くの端末とつなぐには、本質的に2つの問いに答える必要がある。
	- **どこに接続するか？** → 目的地の**アドレス**が必要。
	- **どう辿り着くか？** → **経路（経路表）**が必要。
- 電車のアナロジー：「どこに行くか」に駅名（=アドレス）、「どう辿り着くか」に路線図（=経路表）が要る。

### 【問題2】(スライド15)

> 各端末のMACアドレスで、インターネット上の端末間通信を実現できるか。

- **答え：できない（実用的でない）。**
- MACアドレスは6バイト `XX:XX:XX:YY:YY:YY`。前半3バイトが **OUI**（製造組織の識別子）、後半3バイトを組織が各デバイスへ割り当てる。
- 要点：**デバイスの地理的位置とMACアドレスの間に構造的関係がない**。
	- MACアドレスを見ても所在が分からない。
	- 世界中の膨大な端末を「所在が分かる形で」管理できず、経路を決められない。
- ゆえに大規模ネットワークの統一管理には、**アドレスに階層構造を導入する必要がある** → IPアドレス導入の動機。

### インターネット層の役割（まとめ）

- 多数の端末に割り当てる、**階層構造を持つアドレス空間とその管理方法を定義する**こと。
- アドレスで指定された端末へ到達するための**経路制御**を行うこと。

### 通信の3つの特徴（重要キーワード）

- **コネクションレス型通信**：簡略化・高速化のためコネクション確立をしない。確立が要る場合は上位層(TCP)が担う。
- **ベストエフォート**：可能な範囲で最大限届けようとするが、到達は保証しない。途中で消失・順序逆転がありうる。
- **エンドツーエンド原理**：ネットワーク内部（インターネット層以下）は簡素にし、複雑な処理（トランスポート層以上）は端末側で行う。

### 砂時計モデル

- このシンプルさを表すのが**砂時計(hourglass)モデル**である。
- くびれ部分にインターネット層(IP)があり、上下に多様なアプリ層技術・物理層技術がぶら下がる。
- IPをアドレスと経路制御に特化した単純機能（"dumb network"）に絞ったため、上位・下位の多様な技術を**統一プラットフォームで統合**できた。

## 4-2 IPアドレス

### 基本

- 「IPアドレス」は通常 **IPv4** のアドレスを指す（IPv6なら「v6アドレス」と呼び分ける）。
- インターネット層の「住所」。端末やルータに割り当て、パケットのヘッダに記載する。

### 形式と表現

- **32ビット（4バイト）の符号なし整数**で表す。
- 32桁の2進数は人間が打てないので、**1バイトずつドット区切りの10進表記(dotted decimal)** を使う。
- 例：`10000101.00001001.00000001.00000001` → **`133.9.1.1`**（16進 `0x85090101`）。
- 名前をIPアドレスに変換する仕組みが **DNS (Domain Name System)**（詳細は別章）。

### 【問題3】(スライド41)

> `13.114.124.245` を2進表記・16進表記で表せ。

- **答え**
	- 2進：`00001101 01110010 01111100 11110101`
	- 16進：`0x0d727cf5`
- **解き方**：各オクテット(0〜255)を8桁2進/2桁16進へ直す。2進は 128, 64, 32, 16, 8, 4, 2, 1 の重みで分解する。

|10進|計算|2進|16進|
|---|---|---|---|
|13|8+4+1|00001101|0d|
|114|64+32+16+2|01110010|72|
|124|64+32+16+8+4|01111100|7c|
|245|128+64+32+16+4+1|11110101|f5|

### IPアドレスの割り当て単位

- **ネットワーク・インターフェース(IF)ごと**に割り当てる。
- 通常は1IFに1IP（MACとペア）だが、1IFに複数も、1端末が複数IFを持つことも可能。
- **ルータは通常2つ以上のIFを持つ → 2つ以上のIP（とMAC）を持つ。**
- 確認コマンド：自分のIPは Windows `ipconfig` / macOS `ifconfig`、サーバは `nslookup` `dig`。

### ネットワーク部とホスト部（核心）

- 32ビットを **ネットワーク部** と **ホスト部** に分ける。

```
[ ネットワーク部 ][ ホスト部 ]
←―――― 32ビット ――――→
```

- **ネットワーク部**：組織単位（企業・大学・研究室など）で割り当てる。
- **ホスト部**：その組織のネットワーク内で各ホストに割り当てる。
- どこからをホスト部にするかは管理主体が自由に決めてよい（LANでは先頭24・末尾8が定番）。
- 重要：**ネットワーク部が同じアドレス同士は同一セグメント（同じLAN）にいる**。

### 境界の表し方① プレフィックス表記

- 境界を `/数字` で表す。例 `192.168.128.10/24` = 先頭24ビットがネットワーク部、残り8ビットがホスト部（「スラ24」）。
- `192.168.128.0/24` の場合：
	- アドレス範囲：`192.168.128.0` 〜 `192.168.128.255`
	- **ホストアドレス**：`192.168.128.1` 〜 `192.168.128.254`
	- **ネットワークアドレス**（ホスト部全0）：`192.168.128.0`
	- **ブロードキャストアドレス**（ホスト部全1）：`192.168.128.255`

### 境界の表し方② サブネットマスク

- 2進で**ネットワーク部を全1、ホスト部を全0**にしたもの。
- 例：`/24` → `11111111 11111111 11111111 00000000` = **`255.255.255.0`**。
- 使い方：**IPアドレスとマスクの論理積(AND)でネットワークアドレスが求まる**。
- 例（スライド54）：X=`133.9.222.10`、/25マスクとANDで `133.9.222.0` → `133.9.222.0/25`。
- ⚠️ 補足：スライド54は「25ビットマスク」と書きつつ `255.255.255.192` と記載している。**/25の正しいマスクは `255.255.255.128`**（`192`は/26のマスク）。今回はホスト部 `.10` が128未満なので、どちらで計算しても結果は `133.9.222.0`。試験では「/25 = 255.255.255.128」と正確に覚える。

### ブロードキャストアドレス

- **ホスト部を全1にしたアドレス**。同一セグメント内の全ホストへ送るのに使う。
- このときMACも `FF:FF:FF:FF:FF:FF`（全1）になる。
- **ブロードキャストはセグメントを越えない。**
- 例：`172.20.0.0/16` → `172.20.255.255`／`133.9.203.128/27` → `133.9.203.159`。

### 【問題4】(スライド57)

> セグメントX（`192.168.5.0/25`）のブロードキャスト・ネットワーク・サブネットマスクを書け。

- **答え**
	- ネットワークアドレス：`192.168.5.0`
	- ブロードキャストアドレス：`192.168.5.127`
	- サブネットマスク：`255.255.255.128`
- **解き方**：/25なので先頭25ビットがネットワーク部、残り7ビット（最後オクテットの下位7ビット）がホスト部。

```
最後の8ビット:  N H H H H H H H   (N=ネットワーク部1ビット、H=ホスト部7ビット)
```

	- サブネットマスク：ネット部1・ホスト部0 → `10000000`=128 → `255.255.255.128`
	- ネットワークアドレス：ホスト部全0 → `00000000`=0 → `192.168.5.0`
	- ブロードキャスト：ホスト部全1 → `01111111`=127 → `192.168.5.127`
- `192.168.5.7` や `.23` はホスト部が0〜127の範囲＝同じ `192.168.5.0/25` に属する。

## 4-3 IPアドレスの管理

### 昔のやり方：クラス（現在は不使用）

- かつてはIPアドレスをクラスA〜Eに分けていた。

|クラス|ネットワーク部|ホスト数の目安|先頭ビット|
|---|---|---|---|
|A|/8|約1678万|0|
|B|/16|65,534|10|
|C|/24|254|110|
|D|マルチキャスト用|—|1110|
|E|予約（将来用）|—|1111|

- **利点**：先頭数ビットでネットワーク部の長さが決まる → ルータの経路制御処理が単純。
- **欠点1（非効率）**：1000台しか要らない組織にクラスB(65534)は無駄、逆にクラスC(254)では小さすぎる。サイズが合わない。
- **欠点2（経路数が多い）**：クラスCは約210万個あり、管理経路が多くルータ負荷が高い。

### 今のやり方：CIDR

- 1993年、クラス分けを廃止し **CIDR (Classless Inter-Domain Routing)** を導入。
- **任意のビット長でネットワークを区切れる**（VLSM=可変長サブネットマスク）。
	- 欠点1克服：組織サイズに合った単位でアドレスブロックを配布できる。
	- 欠点2克服：アドレスの**集約**（複数経路を1つにまとめる）ができる。
- 例：`nsl.cs.waseda.ac.jp`(`13.114.124.245`) はドメイン間経路制御では `13.112.0.0/14`（AmazonのAS）で集約される。

### 【問題5】(スライド65)

> IPアドレスを全世界の端末・ルータに割り当てる方法は。

- **答え：階層的な委任(delegation)による分散管理。**
- 相反する2つの要請を両立させる。
	- 同じアドレスの重複（矛盾）は絶対に避けたい → 一貫した管理主体が必要。
	- だがその管理は中央集権的であってはいけない。
- 階層的委任の流れ：
	1. 国際的非営利組織 **ICANN** が全体を統括。
	2. 地域ごとの **RIR (Regional Internet Registry)** に委託（AFRINIC, APNIC, ARIN, RIPE NCC, LACNIC）。
	3. 国ごとの **NIR (National Internet Registry)** に委託（日本は **JPNIC**、中国 CNNIC、韓国 KISA）。
	4. そこから各組織へ。
- 早稲田大学の例：

```
ICANN
 └ APNIC（アジア太平洋地域）
    └ JPNIC（日本）
       └ 早稲田大学            133.9.0.0/16
          └ メディアネットワークセンター
             └ 森研究室         133.9.86.0/24
```

- 大きなブロックを上位→下位へ細分化して委任することで、矛盾なく・分散的に割り当てる。（`133.0.0.0/8` は日本の大学・研究機関、`133.9.0.0/16` が早稲田。）

## 4-4 経路制御の概要

### ルータの役割

- インターネットは**ルータ(router)**で相互接続される（route=経路、進む方向を指し示すもの）。
- ルータの仕事は2つ：
	1. **パケットフォワーディング**：宛先IPと**経路表**を参照し、次のルータへ転送する。**転送するだけで到達確認はしない**（コネクションレス型）。
	2. **経路表の作成・管理**。
- この2つをまとめて**経路制御(routing)**と呼ぶ。

### ホップバイホップ・ルーティング

- **経路は事前に丸ごと決まっているわけではない。**
- パケットを受け取った各ルータが、その都度、経路表に従って「次の転送先」だけを決めて送り出す（1歩=hopずつ進む）。
- 途中で経路が変わることもある。
- 資料の図：A→Bのパケットが各ルータで「経路表→IF3」「→IF2」「→IF4」と出口IFを選びながら進む。

### IPアドレスと経路表 — 最長一致

- 経路選択は**IPアドレスのネットワーク部**を使う。
- 経路表には「ネットワークアドレス」と「次に送るべきルータのアドレス（1IFに対応）」が載る。
- **一致するネットワークアドレスが複数あるときは、一致ビット列が最も長いものを選ぶ**＝**最長一致(longest prefix match)**。
- 例：宛先 `133.9.86.x` で `133.9.0.0/16` と `133.9.86.0/24` の両方があれば、より長く一致する `/24` を選ぶ（より具体的な経路が優先）。

## 第4章のまとめ

- 一言でいうと、**「インターネット層は、階層構造を持つIPアドレスでどこにいるかを表し、ルータの経路制御でそこへ届ける層」**である。
- **役割**＝アドレス体系の定義＋経路制御。砂時計の「くびれ」として簡素さに徹し、多様な技術を統合した。
- **通信の特徴**＝コネクションレス／ベストエフォート／エンドツーエンド原理。
- **IPアドレス**＝32ビット。ネットワーク部とホスト部に分かれ、プレフィックス(`/24`)やサブネットマスク(`255.255.255.0`)で境界を表す。ネットワークアドレス（ホスト部全0）とブロードキャスト（ホスト部全1）を計算できることが重要。
- **管理**＝クラス制 → CIDR。ICANN→RIR→NIR→組織の階層委任で、中央集権を避けつつ矛盾なく割り当てる。
- **経路制御**＝ルータがホップバイホップで転送。選択はネットワーク部、複数一致時は**最長一致**。
- 試験で手を動かしやすいのは **2進/16進変換（問題3）** と **ネットワーク/ブロードキャスト/サブネットマスク計算（問題4）**。「ネット部全1=マスク」「ホスト部全0=ネットワークアドレス、全1=ブロードキャスト」を押さえれば確実に解ける。
