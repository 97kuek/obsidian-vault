# 情報通信ネットワークA：データリンク層

- 親ノート: [[情報通信ネットワークA]]

## データリンク層

### データリンク層とは

- TCP/IPモデルの中で**物理層とインターネット層の架け橋**になる層
- 物理層は「ビット（0と1）をケーブルや電波に流す」だけの世界で、ノイズによる誤りや、どこからどこまでが一つの意味あるデータかという区切りもない。一方インターネット層は「パケットを送りたい」と考えている
- データリンク層はこの間に立って、上位層から見たときに**隣接するコンピュータどうしが、あたかも誤りのない理想的な専用回線でつながっているように見せる**役割を担う

#### 接続形態：LAN と WAN

- **LAN（Local Area Network）**：同じ構内などの近距離で、**多数の端末が一つの伝送路を共有**する。複数台が同時に喋ると衝突するので「次に誰が使うか」を決める制御（**多重アクセス**）が必要
- **WAN（Wide Area Network）**：拠点間など遠距離で、ルーター同士を**1対1**で結ぶ。共有相手がいないので誰が使うかを決める制御は不要

#### データリンク層のプロトコルスタック（2層構造）

- **LANの場合、データリンク層は2層に分かれる**（論理リンク層＋MAC副層）
- WANはポイントツーポイントで多重アクセスがいらないのでMAC副層は不要
- **「LANでは誰が回線を使うかを決める必要がある → だからMAC副層が追加される」**

#### データリンク層が提供する4つの機能

|機能|処理層|内容|
|---|---|---|
|サービスインターフェイス|論理リンク層|上位層に対しプロトコル（約束事）を提供|
|フレーム化|論理リンク層|ビットストリームをフレームに分割し、パケットとして上位へ渡す|
|誤り制御|論理リンク層|物理層で生じた伝送誤りに対処し、上位層からできるだけ隠す|
|フロー制御|論理リンク層|速い送信者が遅い受信者をデータであふれさせないよう流量を調整|
|多重アクセス|MAC副層|複数端末がある時、次に誰が使うかを決定|

#### NIC（ネットワークインターフェースカード）

- データリンク層は具体的には **NIC** というハードウェアで実装される
- **MAC controller**（フレーム化・衝突検知などMAC副層の処理）／**PHY controller**（デジタル変換など物理層の処理）
- PCとはPCI express、LANケーブルとはRJ45コネクタでつながる
- 最近のPCではNICは独立した周辺機器ではなく**マザーボードに内蔵**されているのが普通

#### 上位層に提供する3種類のサービス

- 論理リンクが上位層に提供するサービスは、**物理回線の品質に応じて**3段階ある

| サービス               | 前提         | 例          | トランスポート層での類似物 |
| ------------------ | ---------- | ---------- | ------------- |
| **Ackなし・コネクションレス** | 誤りはあまり起きない | 有線イーサネット   | UDP           |
| **Ackあり・コネクションレス** | 誤りは常に起こりうる | 無線LAN      | （中間）          |
| **Ackあり・コネクション指向** | 信頼性が極めて低い  | 衛星通信・長距離電話 | TCP           |

- 回線が綺麗なら確認応答すら省いて速く送る（イーサネット＝UDP的）
- 回線が悪いほど確認と接続管理で信頼性を担保する（衛星通信＝TCP的）

### フレーム化

#### なぜフレームに区切るのか

- **誤り制御がしやすい**（小さなフレーム単位で検出・訂正）
- **フロー制御がしやすい**（フレーム単位でフィードバック）
- **多重化しやすい**（複数の送信元・宛先を混在＝パケット交換の基本）

#### 境目をどう決めるか

> ビットストリームからフレームを切り出すとき、**どこがフレームの境目か**をどう判別するか？

- **方法1：バイト数**：先頭に「このフレームは何バイト」と書く。その数のビットが化けると区切りがずれ、**それ以降が連鎖的にエラー**になる
- **方法2：バイト詰めを伴うフラグ・バイト**：開始と終了を特別なパターン（例 `01111110` = `0x7E`）で示す。データ中に同じ列が出たら直前に **ESC（例 `0x7D`）を挿入**。ESC自身が現れたら**ESCの前にもう一つESC**を置く。受信側は「先頭のESCは常に削除」で復元
- **方法3：ビット詰めを伴うフラグ・パターン**：バイト詰めを**1ビット単位**で行う（HDLC・PPP）。**連続6個の1（`111111`）でフレーム境界**を表し、送信側はペイロード中に**1が5個連続したら直後に0を挿入**。受信側は「1が5連続した直後の0」を取り除く
- **方法4：物理層符号化違反**：符号化の冗長性（例 **4B/5B**）の未使用パターンに境界の意味を持たせる。**詰め物によるオーバーヘッドがない**のが最大の利点

> 実際は4つを組み合わせて使う。IEEE 802ファミリーは**プリアンブル**（7バイト=56ビット）で到着を示し、終了はヘッダ内の**フレーム長フィールド**（バイト数方式）で判断する。

### 誤り制御

- **誤り検出**：誤りが起きたことだけ気づける冗長性をつけ、気づいたら**送信側に再送要求**
- **誤り訂正**：受信側が中身を復元できる十分な冗長情報をつけ、その場で直す（再送不要）

|手法|適した状況|例|
|---|---|---|
|**誤り検出**|誤り率が極めて低い回線（検出して再送する方が効率的）|光ファイバー、品質の高い銅線|
|**誤り訂正**|誤り率が高い、または再送が原理的に不可能|無線リンク、CD/DVD/BD、QRコード|

- CDやQRコードは「もう一度送ってもらう」ができないので訂正符号が必須

#### 誤り検出の用語

- **データ** $m$ ビット
- **検査ビット** $r$ ビット
- **符号語** $n=m+r$ ビット → **(n, m)符号**／**符号化率** $m/n$

#### ① パリティビット

- **1の個数が偶数（または奇数）になるよう**検査ビットを1ビット付ける。受信側は1の個数の偶奇を数えるだけ
- 弱点：**ランダムエラー**（多くは1ビット＝奇数）は検出できるが、**バーストエラー**（連続して化け、エラー数が偶数になりうる）は単一パリティでは検出できない
- **インターリービング**：パリティを「転送順序とは違う順序（縦方向）」で計算し、バーストエラーを散らして孤立誤りとして検出できるようにする

#### ② チェックサム

- 検査対象部分の**和（2の剰余系）がゼロになる**数を検査ビットとしてデータ末尾に置く。受信側は全部足してゼロなら誤りなし

#### ③ CRC（巡回冗長検査）

- 1961年 Peterson・Brown。パリティ・チェックサムより強力（多項式符号）。Ethernet, HDLC, Bluetooth, USB, gzip/png, MPEG-2 など適用例が広い
- ビット列を**係数が0/1の多項式**として扱い、演算はすべて**2の剰余系**（桁上げ・借りがなく、加減算は XOR と同じ）。例 `110001` ↔ $x^5+x^4+1$

**計算手順**（$G(x)$＝生成多項式 r+1ビット、$D(x)$＝データ、$R(x)$＝チェックサム r ビット）:

1. 元データの後ろに **r個の0** を付けて $S(x)=x^r D(x)$ を作る
2. $S(x)$ を $G(x)$ で割り算（mod 2、XORで引き算）する
3. 余り $R(x)$（rビット）がチェックサム
4. 送信データ $T(x)=S(x)-R(x)$ を送る
5. 受信側が $T(x)$ を $G(x)$ で割り、**余りが0なら誤りなし**

**性質**：長さ $r$ 以下のすべてのバースト誤りを検出できる。ただしエラーパターン $E(x)$ が $G(x)$ を因数に持つと検出できない（$T(x)+E(x)$ がともに $G(x)$ の倍数で割り切れてしまう）。だから $G(x)$ の選び方が重要（CRC-4: $x^4+x+1$、CRC-16/32 など）。

### 誤り訂正

#### ① 反復符号

- 0→`000`、1→`111` と3回くり返し、受信側は**多数決**で復号。**(3,1)符号**、符号化率 $1/3$。1ブロックで1ビットまで訂正できるが効率が悪い

#### ② 水平垂直パリティ検査符号 ((8,4)符号)

- データを2次元（縦横）に並べ、縦と横の両方でパリティを計算。**誤った行と列の交点**として化けたビット位置を1点に絞り込める
- 例：1ビット化けると、その行と列のパリティだけが合わなくなり、交点が誤りビットと特定できる → 反転して訂正

#### ③ パリティ検査方程式と検査行列

符号語 $w=(x_1\ x_2\ x_3\ x_4\ p_1\ p_2\ p_3\ p_4)$ について検査ビットは次を満たす：

$$\begin{aligned} x_1 \oplus x_2 \oplus p_1 &= 0 \\ x_3 \oplus x_4 \oplus p_2 &= 0 \\ x_1 \oplus x_3 \oplus p_3 &= 0 \\ x_2 \oplus x_4 \oplus p_4 &= 0 \end{aligned}$$

係数を行列にまとめたものが**検査行列 H**（4行×8列）:

```
       x1 x2 x3 x4 p1 p2 p3 p4
H =  [  1  1  0  0  1  0  0  0 ]
     [  0  0  1  1  0  1  0  0 ]
     [  1  0  1  0  0  0  1  0 ]
     [  0  1  0  1  0  0  0  1 ]
```

「w が正しい符号語」の必要十分条件は $w H^{T} = \mathbf{0}$。

#### ④ シンドローム（症候群）

受信語 $y$ を検査方程式に入れた結果が**シンドローム** $s = y H^{T}$。$i$ ビット目だけ誤った（誤りベクトル $e_i$）なら $y = w \oplus e_i$ より

$$s = (w \oplus e_i)H^{T} = \underbrace{w H^{T}}_{=\,0} \oplus\ e_i H^{T} = e_i H^{T}$$

これは **H の第 $i$ 列**そのもの。**H の各列がすべて異なれば、シンドロームを見るだけで何ビット目が誤ったか一意に分かり訂正できる**。誤りなしなら $s=0$。

#### ⑤ (7,4)ハミング符号

7ビットで4ビットの情報を運び1ビット誤りを訂正。データ $x_1,x_2,x_3,x_4$、パリティ $p_1,p_2,p_3$：

$$\begin{aligned} p_1 &= x_1 \oplus x_2 \oplus x_3 \\ p_2 &= x_2 \oplus x_3 \oplus x_4 \\ p_3 &= x_1 \oplus x_2 \oplus x_4 \end{aligned}$$

シンドローム $(s_1,s_2,s_3)$ と誤り位置の対応:

|誤った位置|s1|s2|s3|
|---|---|---|---|
|x1|1|0|1|
|x2|1|1|1|
|x3|1|1|0|
|x4|0|1|1|
|p1|1|0|0|
|p2|0|1|0|
|p3|0|0|1|
|誤りなし|0|0|0|

シンドロームは3ビット＝$2^3=8$通り。「000」が誤りなし、残り7通りが7ビットの各誤りに1対1対応（だから(7,4)で過不足なく効率的）。表を引いて該当ビットを反転すれば訂正。

#### ⑥ 主要な誤り訂正符号

- **ハミング符号(1950〜)**：高速だが訂正能力は低い。メモリ・RAID2
- **畳み込み符号(1955〜)**：GSM・衛星・802.11、NASAボイジャー
- **リード・ソロモン符号(1960〜)**：訂正能力が高く**バーストエラーに強い**。CD/DVD/BD、DSL、衛星
- **LDPC(1995〜)**：理論限界（**シャノン限界**）に近い。10Gイーサ、802.11n/ac、WiMAX、4K/8K放送

### フロー制御

- 相手の状況に合わせて送信ペースを調整する
	- ①フィードバックなし（理想）
	- ②逐次確認 
	- ③スライディングウィンドウ

#### ① 理想的なネットワーク

- 処理時間ゼロ・受信バッファ無限・通信路に誤りなし → **フィードバック不要で最大転送率で送りっぱなし**

#### ② 逐次確認と ARQ

- **1フレーム送ったらACKが返るまで次を送らない**
- 誤りがあるとフレームやACKが失われるので **自動再送要求 ARQ**：ACKが来なければ**タイムアウト**で再送
- 「初回フレーム」と「再送された重複」を区別するためSeqを付ける（1フレームずつなら0と1の交互で十分）

#### 演習1：東京―ワシントンD.C.間の最高伝送速度

##### 問題

> フレーム長を1,500バイト、伝播速度を光速、帯域を無制限、雑音なし、距離を約11,000 kmとする。東京―ワシントンD.C.間で逐次確認を行う場合の最高伝送速度を求めよ。

##### 解答

- 逐次確認では「1フレーム送る→ACK戻る」に**往復遅延(RTT)**だけかかり、その間1フレームしか送れない。上限＝1フレームのビット数 ÷ RTT
- RTT $= \dfrac{2 \times 11000\,\text{km}}{3.0\times10^8\,\text{m/s}} \approx 0.073$ 秒、1フレーム $1500\times8=12000$ ビット → $\dfrac{12000}{0.073} \approx \mathbf{163.6\ kbps}$
- 帯域無制限でも距離（遅延）が大きいと逐次確認は極端に非効率＝**long pipe problem**

#### 演習2：RTTが大きい場合の高速化

##### 問題

> RTTが大きい場合、どのように伝送速度を向上させるか。

##### 解答

- ACKを待たずに複数フレームを先送りする、**スライディングウィンドウ**を用いる。

#### ③ スライディングウィンドウプロトコル

- **ウィンドウサイズの範囲内ならACKを待たず複数フレームを同時送信**
- ACKが来るたびウィンドウを前へずらす

#### 演習3：パイプライン中のフレーム破損

##### 問題

> パイプライン中のフレームが破損した場合、どのように再送するか。

##### 解答

- **一括再送 (Go-back-N ARQ)**：誤り以降を全部捨てACKを返さない。受信側は順序通りしか受けない（受信ウィンドウ=1、送信ウィンドウ=N）。実装は簡単だが後続を全部送り直す
- **選択的再送 (Selective Repeat ARQ)**：届いたフレームをバッファに溜め、タイムアウトしたフレームだけ再送。**NAK**で再送を早められる。両側ウィンドウ=N、効率は良いが受信側にバッファが必要

### メディアアクセス制御 (MAC)・多重アクセス

- 1本の共有チャネルを複数端末が使うとき「**次に誰が送信してよいか**」を決める制御＝**MAC**（データリンク層のMAC副層で実現）

|区分|レイヤー|方式|特徴|
|---|---|---|---|
|**多重化（静的）** TDM/FDM/CDM|物理層|時間・周波数・符号で固定分割|回線交換的。バースト性データに非効率|
|**多重アクセス（動的）** CSMA等|データリンク層|フレーム単位で動的割当|パケット交換的。バースト性に向く|

#### ① ALOHA

- 1970年代初頭、ハワイ大学が離島とホノルルを短波無線でつなぐために開発
- **ピュアALOHA**：送りたいデータがあればいつでも送り、衝突したらランダム時間待って再送
	- フレーム送信に $t$ 秒、$t$ 秒間の平均発生数 $G$（ポアソン分布 $P(k)=\tfrac{G^k e^{-G}}{k!}$）。成功には前後 $2t$ 秒に他フレームがゼロ＝$P_\text{成功}=e^{-2G}$。スループット $S=Ge^{-2G}$、最大は $G=1/2$ で $S_{\max}=\tfrac12 e^{-1}\approx 0.184$（チャネルの18%程度）
- **スロット化ALOHA**：時間をスロットに区切り先頭でのみ送信。衝突区間が $2t$→$t$ に半減し $S=Ge^{-G}$、最大は $G=1$ で $S_{\max}=e^{-1}\approx 0.368$（ピュアの**2倍**）

#### ② CSMA（搬送波検知多重アクセス）

- 送信前に**搬送波（チャネルが使用中か）を確認し、空いてから送る**。イーサネット・無線LANの基本
- **永続CSMA(1-persistent)**：空いたら即送信／**p-永続**：空いたら確率 p で送信／**非永続**：継続監視せず空きを見つけたら送る。p-永続・非永続のほうが高スループット（皆が空気を読むと全体最適）
- **CSMA/CD（衝突検出）**：有線クラシックイーサネット(802.3)。送信中にジャム信号で衝突を知る。スイッチングハブの登場で使われなくなった
- **CSMA/CA（衝突回避）**：無線LAN(802.11)。物理的に衝突検出できないので、空いた後**ランダムなバックオフ**を入れてから送信。衝突有無は**ARQ（ACKが返るか）**で推定
- **指数バックオフ**：衝突のたび待ち候補スロットを倍々に広げる（1回目0〜1、2回目0〜3、… n回目0〜$2^n-1$）。混雑時に皆が散らばる
- CSMA/CAの分散制御モードを **DCF**（集中制御の PCF はほぼ使われない）

### データリンクプロトコルの具体例

#### ① IEEE 802 ファミリー

- IEEE 802委員会のLAN規格群。**データリンク層だけでなく物理層も含めて標準化**。論理リンク制御(LLC)=802.2、有線=802.3、無線=802.11

#### ② イーサネット (Ethernet)

- 1973-74年 Xerox PARC の**ロバート・メトカーフ**が開発（語源は仮想媒質「エーテル」）
- **クラシック**：同軸ケーブル共有・半二重・CSMA/CD（指数バックオフ）／**スイッチ式**：スイッチングハブで専用回路・原理的に衝突なし・全二重・CSMA/CD不要、100Mbps〜100Gbps超
- 規格表記 `N BASE-X`（N=速度、X=媒体: 2/5=同軸、T=ツイストペア、S/L=レーザー波長）
- **フレーム構造**：プリアンブル+SoF(8B)→宛先DA(6B)→送信元SA(6B)→Type(2B)→データ(0〜1500B)→Pad(0〜46B)→**FCS（チェックサムで誤り検出）**
- **MACアドレス**：6バイト。前半3バイト=**OUI（組織識別子）**、後半3バイト=各組織がデバイスへ割当。スイッチは受信履歴から「どのポートにどのMAC」を**学習**
- **ループ対策**：ループは**ブロードキャストストーム**を起こすので、**スパニングツリー(STP/RSTP, 802.1D)**で木構造を作り余分なポートを閉じる

#### ③ 無線 IEEE 802.11 / Wi-Fi

- **802.11(技術規格) と Wi-Fi(Wi-Fi Alliance の相互接続認定) は別物**。2.4/5GHz帯のISMバンド。MAC副層は**半二重・CSMA/CA・Stop-and-Wait ARQ**
- **動作モード**：AP経由の**インフラストラクチャモード**／端末直結の**アドホックモード**。BSS（基本）→ESS（拡張、同じ**SSID**）
- **AP接続手順**：Probe→Beacon→Authentication→Association→Data。フレームは データ/制御/管理 の3種

#### ④ PPP (Point-to-Point Protocol)

- 1対1接続用
- 主にWAN（アナログ電話・ISDN・ADSL）
- イーサネットと違い**物理層の規定を含まない**
- コネクション型（確立→ネゴシエーション→通信）
- **LCP**（上位層に依存しない機能）／**NCP**（上位層依存、IPなら IPCP）
- 認証：**PAP**（ID+パスワードを平文＝弱い）／**CHAP**（チャレンジ&レスポンスのワンタイム＝安全）
- フレームはHDLC準拠（フラグ `0x7E` で囲む）→ ヘッダ → データ → **FCS**
