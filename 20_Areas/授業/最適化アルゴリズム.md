# 最適化アルゴリズム

> 線形計画の双対性から、整数計画とその厳密解法（分枝限定法・切除平面法）、各種の定式化例までをまとめる。双対性は [[SVMとカーネル法]] の双対問題ともつながる。

## 双対問題

### 双対問題とは

ある工場が製品 A・B を作って利益を最大化する、という問題を考える（**主問題**）。

- 製品Aの生産量を $x_1$、製品Bの生産量を $x_2$
- 資源1(材料)は A に1個・B に2個必要で、在庫は16／資源2(労働)は A に2個・B に1個必要で、在庫は14
- 利益は A が20、B が30

$$\max z = 20x_1 + 30x_2 \quad \text{s.t.}\quad x_1 + 2x_2 \le 16,\ 2x_1 + x_2 \le 14,\ x_1,x_2 \ge 0$$

ここで視点を変え、**「この工場の資源をぜんぶ買い取りたい人」**を想像する。資源1・2に値段 $y_1, y_2$ をつけて買おうとし、**支払総額を最小にしたい**（$\min\ 16y_1 + 14y_2$）。ただし工場が「売るより自分で作ったほうが得」と思わない値段でないと売ってもらえない＝製品A・Bを作って得られる利益以上の価値を資源の値段で保証する必要がある。

- A 1個分の資源価値：$1\cdot y_1 + 2\cdot y_2 \ge 20$
- B 1個分の資源価値：$2\cdot y_1 + 1\cdot y_2 \ge 30$
- 値段は負にできない：$y_1, y_2 \ge 0$

これが**双対問題**。

### 答えが一致する（双対性）

- **主問題の最大値も、双対問題の最小値も、ぴったり 260 で一致**する。これが双対性。
- 「利益を最大化した値」と「資源を最小コストで評価した値」が一致するのは偶然ではない。

### 一般形での対応関係

$$\textbf{主問題(P)}:\quad \max_{x}\ c^T x \quad \text{s.t.}\ Ax \le b,\ x \ge 0$$
$$\textbf{双対問題(D)}:\quad \min_{y}\ b^T y \quad \text{s.t.}\ A^T y \ge c,\ y \ge 0$$

|主問題(最大化)|双対問題(最小化)|
|---|---|
|変数 $x$（$n$個）|制約 $A^Ty \ge c$（$n$本）|
|制約 $Ax \le b$（$m$本）|変数 $y$（$m$個）|
|目的関数の係数 $c$|制約の右辺 $c$|
|制約の右辺 $b$|目的関数の係数 $b$|
|係数行列 $A$|転置 $A^T$|

### 弱双対定理（ラグランジュ緩和で導く）

発想は「主問題の目的関数の**上界**を作り、その上界を一番きつく（最小に）する」。$y \ge 0$ を使ってラグランジュ関数を作る：

$$L(x, y) = c^T x - y^T(Ax - b)$$

制約 $Ax \le b$ より $y \ge 0$ のもとで $-y^T(Ax-b) \ge 0$、したがって $L(x, y) \ge c^T x$（常に目的関数以上＝上界）。次に $L$ を $x$ について最大化：

$$\max_x L(x,y) = \max_x \big[(c^T - y^T A)x + y^T b\big]$$

$x \ge 0$ でこの値が有限にとどまるには $x$ の係数が正であってはならず、$c^T - y^T A \le 0 \Longleftrightarrow A^T y \ge c$（これが双対問題の制約！）。このとき第1項は $x=0$ で最大となり $\max_x L(x,y) = b^T y$。つなげると：

$$\max_x c^T x = c^T x^* \le b^T y^* = \min_y b^T y \qquad \textbf{(弱双対定理)}$$

「主問題の任意の実行可能解の値 ≤ 双対問題の任意の実行可能解の値」を保証する。重要な副産物として、**主問題のラグランジュ乗数 $y$ が、そのまま双対問題の設計変数になる**。

### 強双対定理

線形計画では等号まで到達する：

$$\max_x c^T x = c^T x^* = b^T y^* = \min_y b^T y \qquad \textbf{(強双対定理)}$$

スラック変数で標準形にした $x$ を基底変数 $x_D$ と非基底変数 $x_I$ に分け、$A=[A_D\ A_I]$, $c=[c_D\ c_I]$ と分ける。$A_D x_D + A_I x_I = b$ から $x_D = A_D^{-1}(b - A_I x_I)$、目的関数に代入すると：

$$z = c_D^T A_D^{-1} b + \big(c_I^T - c_D^T A_D^{-1} A_I\big)x_I$$

最適基底解では $x_I^* = 0$。最適性条件（非基底変数を増やしても $z$ が増えてはいけない）は被約費用が非正：$c_I^T - c_D^T A_D^{-1} A_I \le 0$。ここで $y^T = c_D^T A_D^{-1}$ と置くと $A^T y \ge c$ となり $y$ は双対の実行可能解。さらに $b^T y = c_D^T x_D^* = c^T x^*$ なので、この $y$ が双対の最適解 $y^*$。よって $c^T x^* = b^T y^*$。

### 相補性定理

$x^*$ が主問題の最適解、$y^*$ が双対問題の最適解であるための**必要十分条件**：

$$(A^T y^* - c)^T x^* = 0 \quad \text{かつ} \quad (Ax^* - b)^T y^* = 0$$

実行可能性から各項は符号がそろう（片方が非負・片方が非正）ので、**項ごとに0**になる。制約ごとに：

- $y_i^* > 0$（影の価格が正）$\Rightarrow$ 主問題の第 $i$ 制約は**アクティブ**（資源を使い切っている）
- 第 $i$ 制約が**非アクティブ**（資源が余っている）$\Rightarrow y_i^* = 0$（余っている資源に値段はつかない）

「余った資源の影の価格はゼロ」「価格がつく資源は使い切っている」という経済的に自然な対応。スラック変数とラグランジュ乗数のどちらか一方が必ずゼロ＝**KKT条件**そのもの。

### 一般の最適化問題への拡張

$\min_x f(x)$ s.t. $g_i(x) \le 0$ でも同じ枠組み。ラグランジュ関数 $L(x,\lambda) = f(x) + \sum_i \lambda_i g_i(x)\ (\lambda_i \ge 0)$ を作り、$q(\lambda) = \min_x L(x,\lambda)$ とおくと $q(\lambda) \le f(x^*) \le f(x)$、$\max_\lambda q(\lambda) = f(x^*) = \min_x f(x)$。

**双対問題による解き方の手順**：

1. ラグランジュ乗数 $\lambda$ を導入してラグランジュ関数を定義
2. $L(x,\lambda)$ を $x$ で最小化して $q(\lambda)$ を求める（$\partial L/\partial x=0$ を代入して $x$ を消去）
3. $\lambda_i \ge 0$ のもとで $q(\lambda)$ を最大化

複雑な変数 $x$ を消去して $\lambda$ だけの簡単な問題に落とせるのが双対化のうれしさ。

## 整数計画問題と分枝限定法

### 整数計画問題とは

線形計画問題に「整数」という条件を足したもの。線形計画問題 $\max_x c^Tx$ s.t. $a_i^Tx \le a_i$（目的も制約も一次式）に、**$x \in \mathbb{Z}^n$** を加えたものが整数計画問題。

- さらに整数を $0$/$1$ に限定したものを**0-1整数計画問題**と呼び、「$i$番目を選ぶ($1$)か選ばない($0$)か」を表す。ナップザックやTSPなど組合せ最適化の多くがこの形。

### なぜ整数計画は難しいのか

- 線形計画なら**シンプレックス法**で効率的に解け、最適解は必ず多角形の「頂点」に来る。
- ところが整数を要求すると、LPの最適頂点はたいてい整数座標でなく、近くの格子点のどれが最良かは簡単に分からない。全格子点を調べると変数増加で指数爆発。

### 線形計画緩和

- 難しくしている整数制約を一旦外したLPを、元のIPの**緩和問題**と呼ぶ。領域は元より必ず広い。
- **性質1**：緩和問題の最適値は元の問題の最適値の**上界**（楽観的見積もり）。
- **性質2**：緩和問題の最適解がたまたま整数なら、それはそのまま元の問題の最適解。

### 分枝限定法

「全格子点は無理だが、調べなくていい部分は早めに切り捨てて効率よく全探索する」考え方。

```
1. 問題リストに初期問題を入れる
2. 実行可能解をひとつ求め、暫定解とする
3. while (問題リストが空でない) {
   3-1. リストから問題をひとつ取り出し、緩和解を求める
   3-2. if (緩和解が暫定解より悪い) continue   // 見込みなし、切り捨て
   3-3. if (緩和解が整数解) 暫定解を入替えて continue  // 解が確定
   3-4. 緩和解の近傍に候補を探し、暫定解より良ければ入替える
   3-5. 変数ひとつを切り上げ／切り捨てして問題を2つに分け、リストに入れる
}
4. 暫定解を最適値として出力
```

- **3-2（限定操作）**：緩和解＝楽観的見積もりですら暫定解より悪いなら、その部分問題を細かく調べても暫定解を超えないので切り捨てる。指数的探索を大幅に省ける。
- **3-3（確定）**：緩和解がたまたま整数なら性質2より最適。
- **3-4（近傍探索）**：暫定解が良くなるほど3-2でより多く枝を切れる。
- **3-5（分枝）**：緩和解が $x_1=2.4$ なら、$x_1 \le 2$ と $x_1 \ge 3$ の2問に分ける。両方合わせれば候補を漏らさない。

## 切除平面法

### 凸包が分かれば線形計画で解ける

- すべての実行可能な整数解を含む最小の凸集合を**凸包**と呼ぶ。凸包の不等式表現が手に入れば頂点はすべて整数になり、**ただのLPとしてシンプレックス法で解けてしまう**。
- だが凸包を直接求めるのは一般に困難。そこで**毎回ちょっとずつ凸包に近づける**：緩和解が整数でないとき、すべての整数解は満たすが今の非整数緩和解だけは満たさない不等式（**切除平面**）を追加する。「すべての実行可能解が満たす不等式」を**妥当不等式**と言う。

### Chvátal-Gomory不等式（妥当不等式の作り方）

制約 $\sum_j a_{ij}x_j \le b_i$ に対し、

1. **重み付けして足す**：各制約に $u_i \ge 0$ をかけて足す → $\sum_j(\sum_i u_i a_{ij})x_j \le \sum_i u_i b_i$（妥当不等式）。
2. **左辺の係数を切り捨てる**：$x_j \ge 0$ なので係数を $\lfloor\cdot\rfloor$ で小さくしても妥当。
3. **右辺を切り捨てる**：左辺は整数値をとるので、$\sum_j \lfloor\sum_i u_i a_{ij}\rfloor x_j \le \lfloor\sum_i u_i b_i\rfloor$。

整数だからこそ右辺を切り捨てられる、という整数性の活用がポイント。元のLP領域では成り立たない場合があるので切除平面に使える。実際の計算ではシンプレックスの表（基底）から自動的に切除平面を作る（ゴモリーの小数切除平面法）。

## いろいろな整数計画問題（定式化の例）

- **ナップザック問題**：容量 $C$ に重さ $w_j$・価値 $p_j$ の荷物を詰め価値最大化。$\max \sum_j p_j x_j$ s.t. $\sum_j w_j x_j \le C,\ x_j\in\{0,1\}$。0-1変数の最も基本的な例。
- **文書要約**（ナップザックの一種）：文 $j$ の長さ $l_j$・重要度 $p_j=\sum_i w_i a_{ij}$、長さ上限 $L$ 以内で重要度最大化。重さ→長さ、価値→重要度と読み替えるとナップザックそのもの。同じ数学構造が違う応用に使える好例。
- **ビンパッキング問題**：品物を容量 $C$ のビンに詰め、使うビン数を最小化。$\sum_j w_j x_{ij} \le C y_i$（ビン未使用なら右辺0）と $\sum_i x_{ij}=1$（各品物はちょうど1つのビン）が肝。
- **割当問題**：子供に果物を配り満足度合計最大化。「各子供はちょうど1つ」「各果物は在庫まで」の2方向制約。
- **1機械スケジューリング**：納期遅れ最小化。**Big-M法**で「if-then」を線形化：$s_i + p_i \le s_j + M(1-x_{ij})$（$x_{ij}=1$ なら順序制約が効き、$0$ なら $M$ で実質無効）。
- **巡回セールスマン問題（TSP）**：各都市の出入りを1本ずつにし、**部分巡回路除去制約** $\sum_{(i,j)\in E(S)} x_{ij} \le |S|-1$ でsubtourを禁止する。これが定式化の難所。

## 分枝限定法の一般論

### 組合せ最適化と列挙木

- 組合せ最適化＝目的関数 $f(x)$ を離散集合 $x\in S$ から最小（最大）化する問題。素朴な解法は**全列挙**で、$x_1,x_2,\dots$ と枝分かれさせると葉が許容解になる**列挙木**ができる。各枝は部分問題への分解。分枝限定法は木を作りつつ**調べる必要のない部分木を打ち切る**。

### 終端（限定）の2条件

- **下界値による限定**：部分問題 $P_i$ の緩和 $P_i'$ の最適値 $g(P_i)$（楽観的見積もり、$g(P_i)\le f(P_i)$）について、
	- $P_i'$ に許容解がなければ $P_i$ にもない
	- $P_i'$ の最適解が $P_i$ で実行可能ならそれが最適
	- 暫定値 $z$ について $g(P_i) \ge z$（最小化なら逆向き）なら、その先に解はないので終端
- **優越関係による限定**：$f(P_i) \ge f(P_j)$（$P_j$ が必ず良い）なら「$P_j$ は $P_i$ に優越」。優越する $P_j$ があれば $P_i$ は終端できる。

### 形式的な処理手順

$A$＝アクティブリスト、$N$＝生成済み部分問題、$z$＝暫定値、$O$＝暫定解、$s(A)$＝探索関数として：

```
1. A = {P0}, N = {P0}, z = ∞, O = ∅
2. A が空でない間:
   - P_i = s(A) で1つ選ぶ
   - u(P_i) < z なら z と O を更新
   - 緩和問題 P'_i に許容解がなければ終端
   - g(P_i) = u(P_i) なら（下界と上界が一致＝確定）終端
   - g(P_i) ≥ z なら（下界値による限定）終端
   - 優越する P_j があれば（優越関係による限定）終端
   - どれにも当てはまらなければ子節点を作り A と N に追加（分枝）
   - A から P_i を除く
3. 最適値は z、最適解は O
```

### 探索順序の選び方

- **盲目的探索**：深さ優先（早く葉まで潜って暫定解を得る）／幅優先。
- **ヒューリスティック探索**：最良下界探索（best-bound、下界値が最も良いノードを優先。$A^*$ 探索がこの仲間）。
- 探索順序で暫定解が早く良くなれば限定操作がよく効き、無駄な探索を減らせる。

## ネットワーク最適化

### グラフとネットワーク

- **グラフ** $G=(V,E)$ とは、頂点の集合 $V$ と、それらを結ぶ辺（edge / arc）の集合 $E$ からなる構造である。
- 辺に向きがあるものを**有向グラフ**、ないものを**無向グラフ**と呼ぶ。
- **ネットワーク**は、このグラフの頂点や辺に「値」を付加したものである。
- 本章で扱う**フローネットワーク**は、有向グラフの各辺 $(u,v)$ に**容量** $c_{uv}$を与えたものである。
- さらに次の2つの特別な頂点を持つ。
	- **始点** $s$：フローが生じる頂点
	- **終点** $t$：フローが消える頂点
- **フロー** $\boldsymbol{x}=(x_{uv})$：各辺 $(u,v)$ を流れる流量の状態である。
- **ネットワークの流量** $f$：$s$ から $t$ まで全体でどれだけ流れているかを表す。

$$ f=\sum_{v} x_{sv}=\sum_{u} x_{ut} $$

### ネットワーク最適化の2大問題

|問題|目的|一言で言うと|
|---|---|---|
|**最大流問題**|総流量 $f$ を最大化|どれだけ多く流せるか|
|**最小費用流問題**|需要・供給を満たしつつ総費用を最小化|どれだけ安く流せるか|

### 最大流問題の定式化

$$
\begin{aligned}
\max_{\boldsymbol{x}}\quad & f \\
\text{s.t.}\quad
& 0 \le x_{uv} \le c_{uv}, && (u,v)\in E \quad\text{（容量条件）}\\
& \sum_{u} x_{uk} = \sum_{v} x_{kv}, && k\in V\setminus\{s,t\} \quad\text{（流量保存条件）}
\end{aligned}
$$

1. **容量条件** $0\le x_{uv}\le c_{uv}$：どの辺も容量を超えて流せず、逆流もしない。
2. **流量保存条件**：始点・終点**以外**のすべての頂点 $k$ では、流入量の総和と流出量の総和が等しい。

- これらをすべて満たす $\boldsymbol{x}$ を**実行可能解**と呼ぶ。

> **重要な視点**：最大流問題は線形計画問題（LP）の特殊ケースなので、原理的にはシンプレックス法で解ける。しかしネットワーク構造を活かした**専用アルゴリズム**を使うと、より簡単かつ効率的に解ける。それが増加路アルゴリズムである。

### 残余ネットワークと増加路アルゴリズム

#### 残余ネットワークの考え方

- あるフロー $\boldsymbol{x}$ が与えられたとき、「ここからさらにどう流量を変化させられるか」を表したのが**残余ネットワーク**である。
- 各辺 $(u,v)$（流量 $x_{uv}$、容量 $c_{uv}$）に対し、2種類の残余辺を作る。

| 残余辺 | 向き | 残余容量 | 意味 |
|---|---|---|---|
| 順方向 $u\to v$ | 元の辺と同じ | $c_{uv}-x_{uv}$ | あと**増やせる**量 |
| 逆方向 $v\to u$ | 元の辺と逆 | $x_{uv}$ | **戻せる**量 |

#### 増加路

- **残余ネットワーク上で $s$ から $t$ へ至るパス**を**増加路**と呼ぶ。
- 増加路が**ある**場合、そのパスに沿って流量を増やせる。増やせる量 $\Delta$ は、パス上の残余容量の**最小値**である。
- 増加路が**ない**場合、現在の流量が最大である。

増加路に沿って更新するとき、各辺が**順方向なら $+\Delta$**、**逆方向なら $-\Delta$** だけ元のフローを変化させる。逆方向の辺を使うのは「いったん流した分を別ルートに振り替える」操作に対応し、これによって貪欲法では到達できない最適解にたどり着ける。

#### 増加路アルゴリズム

```text
すべての辺のフローを 0 とする（初期実行可能解）
繰り返し {
    残余ネットワークを作成する
    増加路がなければ、現在の流量を最大流量として終了
    増加路をひとつ選び、その最小残余容量 Δ を求める
    増加路上の各辺について、
        順方向なら元の辺の流量に +Δ、逆方向なら −Δ
}
```

### カットと最大フロー最小カット定理

#### カットの定義

- **カット** $(S,T)$ とは、頂点集合 $V$ を「始点 $s$ を含む集合 $S$」と「終点 $t$ を含む集合 $T=V\setminus S$」に二分割したものである。
- **カットの容量** $C(S,T)$：$S$ から $T$ へ向かう辺の容量の総和である。

$$ C(S,T)=\sum_{u\in S,\ v\in T} c_{uv} $$

- **カットセットの流量**：$S\to T$ の流量から $T\to S$ の流量を引いたものである。

$$ \sum_{u\in S,\ v\in T} x_{uv}-\sum_{u\in T,\ v\in S} x_{uv} $$

#### 3つの性質

**性質1**

- カットセットの流量はネットワークの流量 $f$ に等しい。
- 任意のカットについて成り立つ。

$$ f=\sum_{u\in S,\ v\in T} x_{uv}-\sum_{u\in T,\ v\in S} x_{uv} $$

直観的には「$S$ という袋の中身を見ると、湧き出しているのは $s$ だけ。だから袋から正味で出ていく流量 $=$ $s$ が送り出す流量 $=f$」ということである。形式的には、流量保存条件を $S$ の全頂点について足し合わせると中間頂点の項が打ち消し合い、$s$ の項 $f$ だけが残る。

**性質2：ネットワークの流量はカットの容量以下。** $f\le C(S,T)$（**弱双対性**）。

性質1から出発して評価する。

$$
f=\underbrace{\sum_{u\in S,v\in T} x_{uv}}_{\le c_{uv}}
-\underbrace{\sum_{u\in T,v\in S} x_{uv}}_{\ge 0}
\le
\sum_{u\in S,v\in T} c_{uv}=C(S,T)
$$

第1項は容量条件 $x_{uv}\le c_{uv}$ で上から、第2項は非負性 $x_{uv}\ge 0$ で引き算を最も小さく（$0$ に）評価した。これが**任意の**フローと**任意の**カットの組で成り立つのがポイントである。

**性質3：** 性質2より、どんなカットの容量も流量の上界を与える。したがって**流量とカット容量が等しいフロー・カットの組を見つければ、それぞれ最大フロー・最小カットである**ことが保証される（弱双対のギャップが $0$ になったので、両方とも最適）。

#### 最大フロー最小カット定理

> **定理**：ネットワークの流量の最大値は、そのネットワークのカットの容量の最小値に一致する。

$$\max_{\boldsymbol{x}} f = \min_{(S,T)} C(S,T)$$

#### 増加路アルゴリズムがこれを与える証明

アルゴリズムが終了した（増加路がない）状態を考える。

1. 残余ネットワークで $s$ から**たどれる**頂点の集合を $S$、たどれない頂点の集合を $T=V\setminus S$ とする。増加路がない以上 $t\in T$ なので、これは正当なカットである。
2. **$S$ から $T$ への順方向の残余辺がない**。もしあれば $S$ をさらに広げられて矛盾する。順方向残余容量 $c_{uv}-x_{uv}=0$、つまり対応する辺は**容量を使い切っている**。

$$ \sum_{u\in S,\ v\in T} x_{uv}=\sum_{u\in S,\ v\in T} c_{uv} $$

3. **$S$ から $T$ への逆方向の残余辺もない**。これは $T\to S$ の元の辺で**流量を戻せない**＝流量が $0$ であることを意味する。

$$ \sum_{u\in T,\ v\in S} x_{uv}=0 $$

4. 性質1に代入すると、次の式になる。

$$
f=\sum_{u\in S,v\in T} x_{uv}-\sum_{u\in T,v\in S} x_{uv}
=\sum_{u\in S,v\in T} c_{uv}-0=C(S,T)
$$

流量とカット容量が一致したので、性質3より**このフローは最大フロー、このカットは最小カット**である。

つまり増加路アルゴリズムは、最大フローと同時に、それを「証明する」最小カットも自動的に作る。最小カットは「ネットワークの真のボトルネック」を教えてくれる、というのが実務的な意味である。

### 最小費用流問題と負閉路消去アルゴリズム

#### 定式化

- 最小費用流問題では、各辺に容量 $c_{uv}$ に加えて**単位費用** $e_{uv}$を与え、各頂点 $u$ に **需給** $b_u$ を与える。
- $b_u>0$：供給点
- $b_u<0$：需要点
- $b_u=0$：通過点

$$
\begin{aligned}
\min_{\boldsymbol{x}}\quad & f=\sum_{(u,v)\in E} e_{uv}x_{uv} \\
\text{s.t.}\quad
& 0\le x_{uv}\le c_{uv}, && (u,v)\in E \quad\text{（容量条件）}\\
& \sum_{v} x_{kv}-\sum_{u} x_{uk}=b_k, && k\in V \quad\text{（需要・供給条件）}
\end{aligned}
$$

- 最大流との違いは、目的が**総費用の最小化**である点である。
- 流量保存条件が「各頂点での正味流出 $=$ その点の供給量 $b_k$」という形になる。

#### 負閉路消去アルゴリズム

- 最小費用流では、増加路の代わりに**残余ネットワーク上の閉路**に注目する。
- 閉路に沿って流量を $1$ 動かしたときの**費用の総和**を計算し、以下を判定する。
	- 費用が**正**の閉路：その向きに流すと費用が増える。
	- 費用が**負**の閉路：その向きに流すと費用が減る。

```text
適当な実行可能解を求める
繰り返し {
    残余ネットワークを作成する
    費用が負の閉路がなければ、現在の費用を最小費用として終了
    費用が負の閉路をひとつ選び、最小残余容量 Δ を求める
    閉路上の各辺について、順方向なら +Δ、逆方向なら −Δ
}
```

| | 最大流問題 | 最小費用流問題 |
|---|---|---|
| 探すもの | 増加路 | 費用が負の閉路 |
| 終了条件 | 増加路がない | 負閉路がない |
| 双対の主役 | 最小カット | ポテンシャル |
