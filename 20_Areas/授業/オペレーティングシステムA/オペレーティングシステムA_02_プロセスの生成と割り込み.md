# オペレーティングシステムA：プロセスの生成と割り込み

- 親ノート: [[オペレーティングシステムA]]
- 担当: 内田（UNIX・システムコール）

## fork()

- プロセス生成の中心は fork() である
- fork() を呼び出したプロセス（親プロセス）は新たなプロセス（子プロセス）を生成する
- fork() は親プロセスのアドレス空間を子プロセスに複製し、両プロセスは fork 文の直後の命令から実行を継続する
- 返り値は次のとおり
  - **子プロセスには 0** を返す
  - **親プロセスには子プロセスの PID** を返す
  - **エラー時には -1** を返す
- fork() が実行されると子プロセスは親プロセスのアドレス空間の複製を受け取る
  - fork 前に親の PID を代入しておいた変数は子プロセスにもそのままコピーされる
  - 子側ではその値が getpid() の返り値（＝子自身の PID）と一致しない
- つまり変数の中身までまるごとコピーされるが、コピー後はそれぞれ独立したプロセスとして動く

## fork()のユースケースとexec()

- fork() の重要な使い道として、1つのプロセスで異なるプログラムを実行したい場合がある
- これはシェルでよく用いられる
- この場合、子プロセスは fork() の直後に exec() を実行する
- シェルはまず fork() で自分の子プロセスを作り、その子プロセスが exec() で目的のプログラム（たとえば `ls`）に「中身を入れ替わる」
- 親であるシェル自身はそのまま残り、子の終了を待つ
- `fork → exec → exit`、親は `wait` で待つ、という流れが標準パターンである

### wait() / waitpid()

- **親が子を放置するとどうなるか**
  - 子プロセスで `sleep` が呼ばれていると、子が終了する前に親が先に終了してしまい、シェルのプロンプトが先に表示されてしまうことがある
- そのため、親プロセスが子プロセスを生成するアプリケーションでは、子がいつどのように終了したかを監視することが有用である
- その監視のための仕組みが `wait` である
- 親プロセスは `wait()` あるいは `waitpid()` を実行することで、子プロセスが終了するまで自身の実行をブロックできる
  - `wait()` … 子プロセスのうちの1つが終了するまで親を休止させる
  - `waitpid()` … 指定した特定の子プロセスが終了するまで親を休止させる
- 「どれか1つの子を待つ」か「特定の子を待つ」かの違いである

## 割り込み

### シグナルとは何か

- プロセスに対して「何が起きたのか、何をしてほしいのか」を通知する仕組みである
- 各シグナルには名前がついており、整数値、もしくは `<signal.h>` で定義されたシグナル名 `SIGxxxx` で指定される

### シグナルの種類とデフォルトの動作

- シグナルには種類ごとに「何も設定しなければこうなる」というデフォルトの動作が決まっている

|シグナル|説明|デフォルトの動作|
|---|---|---|
|SIGFPE|浮動小数点例外（例：0による除算）が起きた|コアダンプ終了|
|SIGSEGV|セグメンテーション違反（不正なメモリアクセス）が起きた|コアダンプ終了|
|SIGINT|キーボードからの割り込み（Ctrl-C）が届いた|終了|
|SIGTSTP|キーボードからの一時中断（Ctrl-Z）が届いた|一時中断|
|SIGKILL|プロセスの強制終了|終了|

### signal()

- デフォルト動作は signal() 関数を使って変更できる
- signal() はシグナル受信時の動作を設定する関数で、指定した番号のシグナルにシグナルハンドラーを設定する

```c
void (*signal(int sig, void (*handler)(int)))(int);
```

- ハンドラーの指定方法は3種類ある
  - **SIG_IGN** … そのシグナルを無視する
  - **SIG_DFL** … デフォルトの動作を行う（元に戻す）
  - **ユーザが指定した関数** … そのシグナルを受け取ったときにその関数が呼び出される
- 動作の流れが重要である
  - シグナル `sig` を受け取ると関数 `handler` が呼び出され、`handler` の引数には届いたシグナルの番号が渡される
  - `handler` が呼び出されるときは、それまで実行していたプログラムの実行はいったん一時中断され、`handler` から戻ると実行を再開する
- ここが「割り込み」という言葉の核心である
  - **通常の処理を走らせている最中にシグナルが届くと、処理が一時的に脇にそれてハンドラが実行され、ハンドラが終わると中断したところから元の処理に戻る**——これがまさに割り込みの動作である

### kill()

- ここまでは「受け取る側」の話だったが、シグナルを「送る側」のための関数が kill() である

```c
int kill(pid_t pid, int sig);
```

- kill() は、`pid` で指定された PID を持つプロセスに、`sig` で指定されたシグナルを送出する
- 名前は「kill（殺す）」だが、必ずしも終了させるとは限らない点に注意する
  - SIGKILL を送れば強制終了になる
  - SIGTSTP を送れば一時中断になる
  - ユーザ定義のハンドラが設定されていればその処理が走る
- kill() は「任意のシグナルを任意のプロセスに送り込む汎用的な手段」である
- 第1回で学んだ PID が、ここで「どのプロセスに送るか」の宛先として効いてくる
