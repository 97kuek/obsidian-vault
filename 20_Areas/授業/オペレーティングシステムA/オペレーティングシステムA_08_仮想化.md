# オペレーティングシステムA：仮想化

- 親ノート: [[オペレーティングシステムA]]
- 担当: 中島（OS内部・仮想化）

## 仮想化の定義

- OS・計算機分野における仮想化とは、**物理資源や下位の仕組みを抽象化し、利用者へ別の論理的な資源として見せること**である
- 例として、1台の物理計算機を複数の仮想マシンとして見せたり、1つの物理ネットワークを複数の論理ネットワークへ分離したりする
- 仮想化は必ずしも現実を忠実にシミュレーションすることではない。重要なのは、上位層へ必要なインターフェースを提供し、物理的な構成との差を隠すことである
- VRやARも「計算機が作る仮想的な表現」という広い発想では関連するが、OSの資源仮想化とは目的と技術層が異なる

## コンピュータ内部の仮想化

- **プロセッサの仮想化**
  - 1台の物理マシン上に**仮想マシンモニタ**を置き、その上に複数の OS を「あたかも専用の CPU（vCPU）を持っているかのように」走らせる
  - 物理 CPU は数個でも、各 OS には専用 CPU があるように見せかける
- **ネットワークの仮想化**
  - 1つの物理ネットワークの上に、「財務部用の仮想ネットワーク」「エンジニアリング部用の仮想ネットワーク」を**仮想化レイヤ**で切り出す
  - 物理的には同じ配線でも、論理的には独立したネットワークに見える
- **サーバレスコンピューティング**
  - イベントなどを契機に、基盤側が必要な実行環境を割り当てて関数やアプリケーションを実行する
  - サーバが物理的に存在しないわけではなく、利用者からサーバ管理を隠す**運用上の抽象化・サービスモデル**である
  - 実装にはコンテナやMicroVMなどの仮想化技術が使われることがあるが、サーバレス自体を仮想化方式の一種と同一視しない

## 仮想マシンとコンテナ

|方式|構造|特徴|
|---|---|---|
|**仮想マシン**|ハイパーバイザが仮想ハードウェアを提供し、各VMがゲストOSのカーネルを持つ|異なるOSを動かせ、分離境界が強い。一方でゲストOS分の資源が必要|
|**コンテナ**|ホストOSのカーネルを共有し、namespaceやcgroupなどでプロセス・資源を分離する|起動が速く軽量だが、分離は共有カーネルの設計と設定に依存する|

- どちらも目的は共通で、**「隔離されたアプリケーション実行環境」を提供する**ことである
- 「VMは常に安全、コンテナは安全でない」という単純な二分ではない。実際の安全性はハイパーバイザ／カーネルの実装、設定、権限、運用に左右される
- さらに発展形として **Amazon Firecracker（MicroVM）** がある
  - 「コンテナの軽さ」と「仮想マシンの強い分離」を両立させようとする技術である
  - KVM の上で極小の仮想マシンを多数走らせる

## Software Defined XXX

- ここで一段抽象度が上がる
- **SDN（Software Defined Network）** は、ネットワーク機器を「データを流す部分（Data Plane）」と「制御する部分（Control Plane）」に分離し、制御をソフトウェア（SDN Controller）に集約する考え方である
- 講義は「仮想化の概念の素晴らしさ：Software Defined XXX の世界」でこう主張する
  - 仮想化すれば**現実を簡単に改変できる**——現実と異なる状況をオンデマンドで作り出せる（仮想マシン、仮想ネットワーク、仮想ストレージ、仮想世界…）
  - **複数の仮想世界を同時に作れる**——世界ごとに好きな言語・好きな OS・好きなミドルウェアを選べて、しかも互いにセキュアに分離できる
- 「XXX」の部分には何でも入る、というのがミソである
  - Software Defined Network, Storage, …… そして最終的に **Software Defined Society（ソフトウェア駆動社会）** へとつながっていく
- ただし、`Software Defined X` は単にXを仮想化するという意味に限らない。**制御方針をハードウェアの個別設定から分離し、ソフトウェアからプログラム可能にする**点が中心である
