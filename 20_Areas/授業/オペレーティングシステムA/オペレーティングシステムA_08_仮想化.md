# オペレーティングシステムA：仮想化

- 親ノート: [[オペレーティングシステムA]]
- 担当: 中島（OS内部・仮想化）

## 仮想化の定義

> 「何かを、コンピュータが生成した現実のシミュレーションに変換すること」

- VR（仮想現実）や AR（拡張現実）も同じ仲間である
- 要は**本物ではないけれど、本物のように振る舞うものをコンピュータで作り出す**のが仮想化だ、という捉え方である

## コンピュータ内部の仮想化

- **プロセッサの仮想化**
  - 1台の物理マシン上に**仮想マシンモニタ**を置き、その上に複数の OS を「あたかも専用の CPU（vCPU）を持っているかのように」走らせる
  - 物理 CPU は数個でも、各 OS には専用 CPU があるように見せかける
- **ネットワークの仮想化**
  - 1つの物理ネットワークの上に、「財務部用の仮想ネットワーク」「エンジニアリング部用の仮想ネットワーク」を**仮想化レイヤ**で切り出す
  - 物理的には同じ配線でも、論理的には独立したネットワークに見える
- **サーバレスコンピューティング**
  - イベントが発生すると、システムが**自動的にプロセスを生成して、与えられた関数を実行**してくれる
  - 利用者は「サーバ」を意識せず、処理したい関数だけ書けばよい——これも「サーバという存在を仮想化して消した」と見なせる

## 仮想マシンとコンテナ

|方式|構造|特徴|
|---|---|---|
|**ハイパーバイザ型**|ハードウェア → ハイパーバイザ → 各仮想サーバにOS（カーネル）丸ごと|分離は強固だが重い|
|**コンテナ型**|ハードウェア → OS（1つ） → コンテナ管理ソフト → 各コンテナ|OSを共有するので軽量・高速|

- どちらも目的は共通で、**「隔離されたアプリケーション実行環境」を提供する**ことである
- さらに発展形として **Amazon Firecracker（MicroVM）** がある
  - 「コンテナの軽さ」と「仮想マシンの強い分離」を両立させようとする技術である
  - KVM の上で極小の仮想マシンを多数走らせる

## Software Defined XXX

- ここで一段抽象度が上がる
- **SDN（Software Defined Network）** は、ネットワーク機器を「データを流す部分（Data Plane）」と「制御する部分（Control Plane）」に分離し、制御をソフトウェア（SDN Controller）に集約する考え方である
- 講義は「仮想化の概念の素晴らしさ：Software Defined XXX の世界」でこう主張する
  - 仮想化すれば**現実を簡単に改変できる**——現実と異なる状況をオンデマンドで作り出せる（仮想マシン、仮想ネットワーク、仮想ストレージ、仮想世界…）
  - **複数の仮想世界を同時に作れる**——世界ごとに好きな言語・好きな OS・好きなミドルウェアを選べて、しかも互いにセキュアに分離できる
- 「XXX」の部分には何でも入る、というのがミソである
  - Software Defined Network, Storage, …… そして最終的に **Software Defined Society（ソフトウェア駆動社会）** へとつながっていく
