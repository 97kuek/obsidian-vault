# オペレーティングシステムA：世界はソフトウェアでできている

- 親ノート: [[オペレーティングシステムA]]
- 担当: 中島（OS内部・仮想化）／講義主題

## 計算が価値を生む

- **計算（computation）そのものが、新たな価値を生み出す**という主張である
  - 計算が**金銭**という対価を提供する（仮想経済・暗号資産・各種プラットフォーム）
  - 計算が**安全**という対価を提供する（自動運転、見守り、防災システムなど）
  - 計算が**楽しみ**という対価を提供する（ゲーム、エンタメ。映画『レディ・プレイヤー1』が引用される）
- つまり、これまでコンピュータは「効率化の道具」だったが、これからは**金銭・安全・楽しみといった人間社会の価値そのものを計算で生成する時代**である

## 実世界仮想化の抽象モデル

1. **オートメーション（自動化）**：効率化の方向。工場の自動化、自動運転など、人間がやっていた作業を計算に置き換える
2. **ゲーミフィケーション**：人間の行動を、ゲームのように設計する。鍵は**「適切なレベルデザイン」**——簡単すぎず難しすぎず、人がつい動きたくなる仕掛けを作って、行動をナビゲートする
3. **仮想経済（シェアリングエコノミー）**：Uber や Airbnb のようなシェアリングエコノミーも「実世界の仮想化」と捉える。仮想化の概念で抽象化し、**計算可能なオペレータ（操作）として取り出す**ことで、さらなる連携や自動化が可能になる、という見方である
4. **デジタルツイン**：実世界の物（工場、都市、人など）を、ソフトウェア上にそっくりな「双子」として再現する。仮想側でシミュレーションしてから現実に反映できる
5. **代替仮想現実**：現実そのものを別の体験に置き換える方向

## 効率から「楽しさ・快適さ」へ

- 代替仮想現実のスライドでは、目指す方向として「より効率的」だけでなく、**より楽しい／より幸せ／より快適／より刺激的**を挙げる

```
デザイン駆動アプローチ
   ↑
AR/VRサービス
ソーシャルミドルウェア
分散基盤ソフトウェア
```

## 実世界の仮想化を3軸で整理

- **オートメーション**：コンテキストアウェアネス、工場の自動化、自動運転、そしてゲーミフィケーションによる人間の行動ナビゲーション
- **フィクション性の埋め込み**：現実空間に虚構を重ねる。パブリックディスプレイやプロジェクションマッピングなど
- **物理世界の拡張**：ユーザー体験のエンゲージメントを高める。ライフログ（生活の記録）や、人を“超人的”にする技術など
