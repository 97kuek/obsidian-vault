# 人工知能：探索による問題解決

- 親ノート: [[人工知能]]

## 探索による問題解決

### 探索という発想

- ロボットが迷路やパズルを解いたり効率的に移動するには、**実際に行動する前に、頭の中（メモリ内）で行動の列を計画する**機能が必要
- これが探索
- 探索には、少なくとも「ある状態でどの行為が可能で、その結果どの状態へ移るか」という**後継状態のモデル**が必要である
- ゴールまでの距離の見積もりは必須ではない。これを持たない場合でもBFSや均一コスト探索は可能であり、持つ場合はヒューリスティック探索で効率化できる
- 計算機はこのモデルを内部に持ち、**実際に移動する代わりに地図上で経路を探す**
- この過程が探索であり、実行前に行動を計画するという意味で「プランニング」の一手法でもある

### 問題の5要素による定式化

1. **初期状態** — エージェントがいる「今」の状態
2. **オペレータ** — 各状態で可能な行為（action）。地下鉄の例なら「隣の駅へ移動」
3. **状態空間** — 初期状態からオペレータの適用で到達できる状態全体の集合。オペレータを順に適用していくと状態空間の中を移動することになり、その軌跡を**経路**という
4. **ゴール検査** — ゴール状態にあるかを判定する
5. **経路コスト** — 経路のコストを返す関数。コストは広義の概念で、経路長・所要時間・乗換回数・運賃やそれらの融合でもよい

- 初期状態からゴールへの**最小コスト経路（オペレータの列）を求める**ことが、探索による問題解決

### 抽象化と性能評価

- 計算機で扱うには**抽象化**が必要
- 状態の抽象化では問題の目的に不要な情報を捨てる
- 地下鉄の例なら「隣の駅の関係」だけが本質的で、「売店がある」「トイレが3つある」は無関係。行為も同様に「駅の移動」だけを残す

- **解が必ず見つかるか（完全性）**
- **良い解か（経路が最適か）**
- **解を得るまでの時間とメモリ量（探索コスト）**

- 全体コストは概念上「**経路コスト＋探索コスト**」だが、両者は単位が違うので単純に足せないこともある

### 代表的な問題例

- **8パズル / 15パズル** — 状態は8個（15個）のタイルと空白の配置。オペレータは「空白を上下左右に動かす（タイルと空白の交換）」。コストは各手1。8パズルの全配置は $9!$ 通りだが、置換の偶奇性により、ある初期状態から到達できるのは半分の $9!/2=181{,}440$ 状態である
- **ハノイの塔** — 状態は各円板の位置のリスト `(x1, x2, x3)`。オペレータは各塔の一番上の円板を別の塔へ移すが、小さい円板の上には乗せられない。ゴールは `(3 3 3)`。この「禁止されるオペレータの条件」を一般のn枚で考える課題が出ている
- **掃除機ロボット（Vacuum world）** — N×Mのマスを動いてゴミを吸う。重要な注意：「右へ移動→吸う→左へ移動」で位置は元に戻っても**状態は異なる**（ゴミが消えたから）。地下鉄の例との本質的な差で、状態を正しく定義する練習になる

- 状態空間は、同じ状態へ複数経路から到達できる**グラフ**である。探索時には経路ごとのノードから探索木が生じるが、訪問済み集合を使うグラフ探索では同一状態の重複展開を抑える

### 情報のない探索

- ここからは**ゴールへの方向を示す情報を持たず、基本的に網羅的に状態を調べる**探索

#### 用語の整備

- **分岐数 b（branching factor）** — 1つのノードから1回のオペレータで到達できる子ノード数の平均
- **探索木の深さ m** — 頂点から最も深いノードまでの枝数。有限とは限らない
- **ゴールまでの深さ d** — 頂点から最も浅いゴールまでの深さ
- **閉ノード / closedリスト** — すべての子ノードの展開が完了したノードとその一覧
- **開ノード / openリスト** — まだ展開し終えていないノードとその一覧

- 評価尺度は **完全性・計算量・空間量（メモリ）・最適性** の4つ

#### 探索の基本パターンの発展

**(a) 素朴なランダム探索**

- 子ノードを作り、ランダムに1つ選んで進む
- メモリは小さいがゴールにたどり着く保証がなく、同じ状態を覚えないので**ループに陥る**

**(b) Closedリストの導入**

- 訪問済み状態を履歴として保持し、二度行かないようにする
- 探索木が有限なら必ず終了するが、**解は得られないことが多い**

**(c) Openリストの導入**

- これ以上展開できない地点に着いたとき**戻れるように**、途中で展開したノードを保持する
- これで**探索木が有限なら必ず終了し、解が存在すれば見つけられる**状態に到達する
- Openリストの各要素は `(n p)` のように**ノードと親の対**で持ち、ゴール発見時に親をたどれば経路が復元できる

#### 幅優先探索（BFS）

- 各深さのノードをすべて展開してから次の深さへ
- Openリストから「直前ステップで追加された要素全部」を選ぶ
- **解があれば必ず見つかり、各手のコストが一定なら最適**
- しかし次の深さを展開するために、ある深さの全ノードを覚えておく必要があり、メモリが指数的に増える
- 深さdまでに覚えるノード数は

$$1 + b + b^2 + \cdots + b^d = O(b^d)$$

#### 均一コスト探索

- 各手のコストが異なる場合、BFSは最良解を見逃しうるので、Openリストに**累積コスト** `c' = c + c(n,n')` を持たせ、**最小コストの要素を取り出す**ように一般化する
- 各ステップコストが正の下限を持つなどの条件下で完全であり、ゴールを優先度付きキューから取り出した時点で最適解が確定する
- ただし計算量・メモリはBFSと基本的に同じで、大きな問題には使いにくい
- ゴールを**生成した時点**では、より低コストの経路が未展開の可能性がある。ゴールが累積コスト最小のノードとしてキューから取り出された時点で終了する

#### 深さ優先探索（DFS）

- 最も深いノードを1つ展開し、行き止まったら**バックトラック**して次に深いノードへ
- 実装は**Openリストをスタックに変える**だけ

1. 木が無限の深さを持つと**停止しない可能性**がある
2. メモリは少ない（探索済みの枝を忘れられる）—— 深さmまで各階層で平均b個覚えるので **O(bm)**
3. BFSより効率的なことが多い（BFSは長さdの経路を調べる前に長さd−1を全探索する必要があるため）
4. **最適解が求まるとは限らない**
5. 3と4は子ノードを探索する順序に依存する

**深さ制限探索（Depth-Limited）** — DFSに深さ上限 limit を設け、無限に潜るのを防ぐ。問題は「ゴールまでの深さdが事前にわからない」こと。limit < d だと解が見つからず、limit が大きすぎると深いゴールを先に拾って最適性を失う

**反復深化探索（IDS）** — 深さ制限探索を limit=0,1,2,… と増やしながら繰り返す。これが情報のない探索の決定版

- 分岐数が有限なら完全であり、各手のコストが同一なら最浅ゴールを見つけるため最適である
- **メモリは深さ優先と同じく小さい O(bd)**（DFSの利点）
- 計算量は $O(b) + O(b^2) + \cdots + O(b^d) = O(b^d)$ で、オーダーとしては深さ優先と同等かそれ以下

一見「浅い層を何度も探索し直すのは無駄」に見えるが、計算量のオーダーで見ると繰り返し分の寄与は小さく、これは**完全性・最適性・メモリ効率を同時に得るための代償**である。逆に「無駄を省こうと状態を記憶するとメモリが爆発」してBFSに逆戻りする、という対比が効いている。大きな探索空間でゴールまでの経路長が不明なときの定番手法

- **双方向探索（付録扱い）**
  - ゴールが既知の場合、初期状態からの前向き探索とゴールからの後ろ向き探索を同時に進める。
  - 両者が出会った時点で経路を合成する。
  - 探索範囲を大幅に狭められ、計算量を節約できる。
  - 後ろ向き探索では、ある子ノードに至る親をすべて知る必要がある。
  - チェスのようにゴール状態が膨大にある問題には適用しにくい。

#### 情報のない探索の総まとめ表

l はlimit、d はゴール深さ。

||幅優先|均一コスト|深さ優先|深さ制限|反復深化|双方向|
|---|---|---|---|---|---|---|
|計算量|O(bᵈ)|コスト分布に依存|O(bᵐ)|O(bˡ)|O(bᵈ)|O(b^(d/2))|
|空間計算量|O(bᵈ)|展開ノード数に比例|O(bm)|O(bl)|O(bd)|O(b^(d/2))|
|最適性|単位コストならYes|条件下でYes|No|一般にNo|単位コストならYes|条件下でYes|
|完全性|有限分岐ならYes|正のコスト下限などが必要|有限木でなければNo|l≥d ならYes|有限分岐ならYes|逆向き遷移生成などが必要|

均一コスト探索の代表的な上界は、最適解コストを $C^*$、最小ステップコストを $\varepsilon>0$ とすると $O\!\left(b^{1+\lfloor C^*/\varepsilon\rfloor}\right)$ で表される。

### ヒューリスティック探索

#### ヒューリスティックとは

- 「heuristic（発見的）」はこの分野では「**必ず正解になるわけではないが、その代わりに効率的に正解（に近い解）を得る**」ことを指す
- 現代的解釈では**平均的な問題は効率的になるが、最悪ケースの性能は必ずしも改善しない**関数
- 地下鉄の例なら「目的地が西なら西へ進む路線を優先する」のように、正しいとは限らないが探索を導く指針になる
- 知識を適用する場所は、**子ノードを展開した後、次にどれをたどるか選ぶところ**
- この順序づけを与える評価関数を**ヒューリスティック関数**と呼ぶ

#### 最良優先探索

- 評価関数 eval-fn の値が最良のノードを優先的に展開する戦略
- アルゴリズムは均一コスト探索とほぼ同じで、**Openリストを eval-fn の値で並べ替えて先頭を取り出す**だけ
- これが以降のヒューリスティック探索すべての**テンプレート（基本形）**になる
- 名前は「最良優先」だが、評価関数が本当に最良のノードを当てられる保証はない（当てられるなら探索は不要）。「最良と推定される／多くの場合ほぼ最良」を選ぶイメージ

#### 欲張り探索（greedy）

- 各ノードからゴールまでのコストを見積もるヒューリスティック関数

$$h(n) = \text{ノード } n \text{ からゴールまでの最短経路の見積り}\quad(\text{ゴールなら } h(n)=0)$$

を eval-fn に使った最良優先探索。「ゴールに近そうなノードから展開する」だけ

- **最適解は必ずしも見つからない**。地下鉄の例で h(n) を「駅nと目的地の地図上の直線距離」とすると、新宿から明治神宮前へ向かう際、直線距離だけを頼りに進むと遠回りになり、最短経路はすぐには見つからない
- 迷路の例では、スタート近くで誤った選択をすると、それを覆すにはその下の探索木をすべて検証し尽くして失敗を確認しないと戻れない、という**探索初期の選択の重要性**が説明されている

#### A*探索

- greedyは効率的だが最適でも完全でもない
- 均一コストは完全で最適だが効率が悪い
- **そこで両者を合わせる**のがA*

- $g(n)$ = 出発点からノードnまでの経路コスト（探索しながら計算できる）
- $h(n)$ = nからゴールまでの最短経路の見積もりコスト

$$f(n) = g(n) + h(n)$$

f(n) は「nを経由する最短経路の見積もりコスト」で、これを評価関数に使う

**完全性・最適性が成り立つための条件**が決定的に重要。h(n) が**許容的（admissible）**、すなわち

$$h(n) \le \text{実際のコスト}$$

（実際より小さめに見積もる＝楽観的）であるとき、f(n) もこの性質を受け継ぎ、nを経由する経路の真のコストを超えない。この許容的なfを使った最良優先探索が**A***で、**完全かつ最適**

**最適性の証明（背理法）** — Gを最適ゴール、ルートからGへの真のコストを f* とする。最適でないゴール G₂（g(G₂) > f*）をA*が先に選んだと仮定する。このときGへの最適経路上には、まだ展開されていない縁のノード n が存在する

- h が許容的だから **f* ≥ f(n)**
- n がG₂より先に展開されなかったから **f(n) ≥ f(G₂)**
- G₂はゴールだから **h(G₂)=0**、よって f(G₂) = g(G₂)

これら3式から **f* ≥ g(G₂)**。これは g(G₂) > f* に矛盾。よってA*が最適でないゴールを先に選ぶことはない

**アルゴリズムの注意点** — 子ノード n' を追加するとき、(i) すでにOpenリストに同じ状態がより高コストで入っていれば置き換える、(ii) Closedリストにある状態でも、より低コストの経路が見つかれば**Closedから外してOpenに戻す**。この再オープン処理がA*の正しさを支える

- 良いhは展開ノード数を大幅に減らせるが、一般の問題で最悪時の指数的増加が消えるとは限らない。さらに、h自体の計算コストも考慮する必要がある
- h(n)=0 も許容的で、このときA*は均一コスト探索に一致する

#### ヒューリスティック関数の評価

8パズルで2つの許容的ヒューリスティックを比較する。

- **h₁ = ゴールの位置にないタイルの数**
- **h₂ = 各タイルのゴール位置までのマンハッタン距離の和**

講義の例題（初期状態とゴール）では

- **h₁ = 7**（ブロック6以外の7枚が違う位置）
- **h₂ = 4+3+3+1+4+0+3+3 = 21**

**なぜ両方とも許容的か。** h₁は、位置の合っていないタイルは最低1ステップ動かす必要があり、1回に1枚しか動かせないので過大評価しない。h₂も、どの動作も1回でゴールに高々1だけ近づくだけなので過大評価しない

**優位性（dominance）。** 真のコストを h* とすると

$$h_1 \le h_2 \le h^*$$

が成り立ち、h₂のほうが真の値に近い。このとき「**h₂はh₁より優位（dominate）**」という。一般に、より大きい（より真値に近い）許容的ヒューリスティックほど無駄な展開が減って効率的。ただし**ヒューリスティックの計算量が十分小さい**ことが前提

#### 有効分岐数と実測

- 探索効率を1つの数で表す指標が**有効分岐数 b***
- A*が探索したノード総数を $N$、解の深さを $d$ としたとき、深さ $d$ で $N+1$ 個のノードを持つ「均一な木」の分岐数を b* とする

$$N+1 = 1 + b^* + (b^*)^2 + \cdots + (b^*)^d$$

- 例えば深さ5の解を52ノードで見つけたなら b* ≈ 1.9。**b*が1に近いほど効率的**で、b*=1なら一切無駄なく一直線にゴールへ到達したことを意味する

#### 良いヒューリスティックの作り方 — 弱条件問題

- $h$ の作成は難しいが、有力なヒントが**弱条件問題**
- 元の問題の制約を緩めると、解の見積もりが容易になる

- **h₁** は「タイルを外して正しい場所へ直接はめ込み直してよい」と緩めたパズルの解
- **h₂** は「他のタイルの上を通って移動してよい」と緩めたパズルの解
- **地下鉄の直線距離**は「障害物もレールもなく直線移動できる」と緩めた問題

- 緩めた問題は元の問題より簡単に解けるので、その解は元の最短手数を超えず、自動的に許容的になる
- 複数の弱条件問題から $h_1, \dots, h_m$ を作ったときは、以下の式によりすべて**許容的なまま最も真値に近いものを使える**

$$h(n) = \max(h_1(n), \dots, h_m(n))$$

### 高速・実時間手法と反復改良

- これまでの手法は計算量に加え**メモリ使用量**が問題であった
- そこで限られたメモリで動く改良型と、解だけがわかればよい問題向けの手法を扱う

#### 無矛盾なヒューリスティック

- IDA*の前提として導入される
- 許容的なhが**無矛盾**であるとは、任意のノードnとその子n'について三角不等式

$$h(n) \le c(n, n') + h(n')$$

が成り立つこと。「移動コスト以上にh(n')が小さくならない」という、より強い条件

#### IDA*（反復深化A*）

- 反復深化は完全・最適でメモリ効率が良い
- そこで**反復深化を行うが、深さではなく f(n)=g(n)+h(n) の値で探索を打ち切る**のがIDA*
- 各回は深さ優先で進めるが、fの値が境界値 limit を超えたノードはスタックに積まず捨てる。1回終わると limit を上げて再スタートする

- **メモリは極端に小さい**（展開途中で limit 以下のノードだけをスタックに積むだけ）
- A*より再展開が増える場合がある一方、必要メモリは探索深さにほぼ比例する
- 有限分岐、正のステップコスト、許容的なヒューリスティックなどの条件下で完全かつ最適である
- 欠点：limitを変えるたびに**全部忘れる**ので無駄な繰り返しがある（IDSと同じ欠点だがメモリは少ない）。複雑な領域ではこの繰り返しが効率を下げる

#### 反復改良アルゴリズム

- 問題によっては**経路はどうでもよく、解そのものだけわかればよい**ことがある
- 任意の状態から始め、質を少しずつ改善することを繰り返してゴールを得る

**山登り法**

- 現在状態に可能なオペレーションを施し、最も「良い」状態へ移る
- `value`を高さに例えると山を登るイメージ
- コストを下げる局所探索と勾配降下法は発想が近い。ただし、勾配降下法は微分可能な連続空間で勾配を使う特別な手法であり、離散的な山登り法と同一ではない
- 欠点
	- **局所最大/最小** — 局所的なピークに達すると抜け出せず、谷の向こうの高い山へ行けない
	- **高原** — 平らな高い場所でランダムに動き回ってしまう
	- **峰** — 尾根には達するが山頂まで緩やかで`value`がなかなか向上しない

**焼きなまし法**

- 局所最大を抜け出すため、**悪くなる方向にもある確率で動く**手法
- 最大化問題で $\Delta= \mathrm{value(next)}-\mathrm{value(current)}$ とすると、$\Delta\ge 0$ なら受理し、$\Delta<0$ でも確率 $\exp(\Delta/T)$ で受理する
- $T$ はスケジュール関数で時間とともに徐々に下げる
- $T$ が大きいほど動き回る確率が高く、$T=0$ でランダム移動が止まる
- 金属を徐々に冷やしてきれいな固体を作る「焼きなまし」になぞらえた命名
- 状態空間は連続でも離散でもよい。重要なのは近傍状態の定義と温度スケジュールである
- 理論上の収束保証には非常にゆっくりした冷却が必要であり、実用上の有限時間スケジュールでは最適解を保証しない
