# Topic 3：ゲーム・マルチエージェント

**問題**

- 下のゲーム木をアルファベータ法を用いて探索し、選択された状態に○印、枝刈りされた部分に×印をつけよ。
- 木は4階層。頂点が MAX、その下が MIN（3ノード）、その下が MAX（6ノード）、最下段が葉（12個）である。
- 葉の値は左から順に **6, 1, 2, 3, 5, 4, 7, 1, 2, 4, 5, 2**。

**解答**

まず枝刈りを無視して各ノードの値を確定させ、次に左から評価したときにどこが刈られるかを追う。下の図が最終的な答えである（緑＝選択された手＝○、赤の×＝枝刈り）。評価の流れを順に追うと次のとおりである。

- まず各 MAX ノード（下段の○）が葉のペアの最大値を取る。左から **6, 3, 5, ≥7, 4, （×）** となる。
- 次に各 MIN ノード（中段の□）が子の MAX の最小値を取る。左から **3, 5, ≤4** となる。
- 頂点の MAX はこれらの最大値 **max(3, 5, ≤4) = 5** を取る。よって**根の値は5**、選ばれる手は**真ん中の枝（MIN2）**である。
- 枝刈りは左から評価したときに次の2か所で起こる。
    - **βカット（真ん中の枝）**: MAX3 が先に 5 に確定し、親の MIN2 は上限 β=5 を持つ。次の MAX4 で葉 7 を見た瞬間、その値は 7 以上に確定する。β=5 を超えるので MIN2 はこの枝を選ばない。よって MAX4 の2つ目の葉「1」は**調べる必要がなく×**。
    - **αカット（右の枝）**: 真ん中の枝まで評価して根は α=5 を確保している。右の MIN3 を調べ始め、最初の子 MAX5 が 4 に確定すると、MIN3 は 4 以下に確定する。根はすでに 5 を持つので 4 以下の枝は選ばない。よって MIN3 の残り（MAX6 とその葉「5」「2」）は**まるごと×**。
- 結局、探索しなくてよい葉は **3つ**（位置8の「1」、位置11の「5」、位置12の「2」）である。
- 選択された手（○）の筋道は **根 → MIN2 → MAX3 → 葉5** となる。

## ミニマックス法の前提

### 問題

> ミニマックス法では、自分の手番のノードで最大値、相手の手番のノードで最小値を取る。なぜ相手のノードで「最小」を取るのか、その前提を説明せよ。

### 解答

- ミニマックス法は「**相手の最良の手は、自分にとって最悪の手である**」という前提を置いている。
- 自分は利得を最大化したい。相手は自分とは逆に、こちらの評価値を最小化するように動くと仮定する。
- したがって相手の手番のノードでは、相手が「自分にとって最も評価値が低い子」を選ぶと考え、最小値（min）を伝播させる。
- これは「相手が最善を尽くしても、少なくともこれだけは保証できる」という**最悪ケースの保証**を求める考え方であり、後のゼロサムゲームの「ミニマックス戦略」と同じ発想である。
- 注意点として、これは相手が完全に合理的に最善手を打つという仮定であり、相手がミスをする場合は最善とは限らない。

## アルファベータの順序依存

### 問題

 > アルファベータ枝刈りで得られる「最終的な手」はミニマックス法と完全に一致するのに、なぜ「子ノードの評価順序」が重要だと言われるのか。順序が良い場合・悪い場合で何が変わるのかを説明せよ。

### 解答

- 枝刈りの結果（選ばれる手・根の値）は順序によらず常に正しく、ミニマックス法と一致する。**変わるのは「枝刈りできる量」＝計算効率**である。
- αは「MAX側が確保した下限」、βは「MIN側が許す上限」である。良い手（評価値の高い枝）を先に調べると α が早く大きくなり、後続の枝で α≧β が成立しやすくなって、多くの枝を刈れる。
- 逆に悪い手から調べると α・β の幅がなかなか狭まらず、刈れる枝が減る。スライド29の例では、葉の並びを反転させると**βカットができなくなった**。
- 最良順で並べられれば、探索すべきノード数は最悪時のおよそ平方根のオーダーまで減る（＝実質的に2倍の深さまで読める）。
- このため実装では「有望そうな手を先に並べる」工夫（ムーブオーダリング）が効率を大きく左右する。

## 衝突の定義と時間発展グラフ

**問題**

> Cooperative A* における「衝突」とは具体的にどの2種類の状況を指すか。また、時間発展グラフで「同時刻間の逆向きエッジ」を禁止領域とするのはなぜか。

**解答**

- 衝突は次の2種類である。
    - **ノード衝突**：同じノード（マス）に複数のエージェントが同時に存在すること。
    - **エッジ衝突（すれ違い）**：同じエッジ上で2つのエージェントが互いに逆向きに移動して交差すること。
- 時間発展グラフでは、ノードを時刻ごとにコピーし、既存経路が使うノードを禁止領域にすることで**ノード衝突**を防ぐ。
- しかしそれだけでは、エージェント A が i→j、エージェント B が j→i と同時刻に動く「すれ違い」を防げない。両者とも相手の「出発ノード」には入っていないため、ノード禁止だけではすり抜けてしまう。
- そこで、時刻 t−1 から t の間でエッジ i→j が使われていれば、**同時刻の逆向きエッジ j→i も禁止**にする。これでエッジ衝突を防げる。
- 性質として、この方法では最短経路は必ず見つかるが、時間方向にグラフが膨らむため探索時間は増大しうる（ただし現在の計算機なら実用範囲）。

## LRA* の弱点

### 問題

> Local Repair A*（LRA*）はどんな環境で有効で、どんな環境で問題が起きるか。問題が起きる理由とともに説明せよ。

### 解答

- LRA* は「隣のエージェントだけを障害物とみなして A* で経路を作り、ぶつかりそうなら止まって作り直す」という局所的・近視眼的な手法である。
- **有効な環境**：平面のような広い空間を、少数のエージェントで動かす場合。迂回路が多いので、止まって作り直してもすぐ別の道が見つかり、効率的である。
- **問題が起きる環境**：
    - エージェント数が多く混雑する領域。他のエージェントに囲まれて脱出できなくなる。
    - 迷路のように経路が限られた環境。迂回路がない（または大きく遠回り）ため、**デッドロック**（互いに道を譲れず全員が動けなくなる状態）に陥りやすい。
    - 環境の端（隅）で混雑が起きると動けなくなる。
- 根本原因は、各エージェントが**他者全体の経路を考慮せず、目先の隣だけを見て**判断するため、全体としての調整が取れない点にある。Cooperative A* は既存経路全体を考慮することでこの弱点を補う。

## 支配戦略と支配戦略均衡

### 問題

> 次の利得行列（各セルは (P₁の利得, P₂の利得)）について、P₁・P₂それぞれの支配戦略を求め、支配戦略均衡が何かを答えよ。

```
            P2: X        P2: Y
  P1: A    (3, 3)      (5, 1)
  P1: B    (1, 5)      (2, 2)
```

### 解答

- **P₁の支配戦略を調べる**
    - 相手が X のとき：A=3 ＞ B=1 → A が良い。
    - 相手が Y のとき：A=5 ＞ B=2 → A が良い。
    - どちらの場合も A の方が常に大きいので、**A は P₁の強支配戦略**である。
- **P₂の支配戦略を調べる**
    - 相手が A のとき：X=3 ＞ Y=1 → X が良い。
    - 相手が B のとき：X=5 ＞ Y=2 → X が良い。
    - どちらも X が常に大きいので、**X は P₂の強支配戦略**である。
- 両者が支配戦略を持ち、それを取り続けるので、**支配戦略均衡は (A, X)＝(3, 3)** である。
- ここでは均衡 (3,3) が他のどの組合せより両者にとって悪くないため、ジレンマは生じていない

## 囚人のジレンマ

### 問題

> 次の利得行列（各セルは2人の懲役年数。利得なので符号は負）について
> (1) 各プレイヤーの支配戦略
> (2) 均衡点
> (3) なぜ「ジレンマ」と呼ばれるかを答えよ。
> S₁＝黙秘（協調 C）、S₂＝自白（裏切り D）。

```
              P2: S1(黙秘)   P2: S2(自白)
  P1: S1(黙秘)  (-2, -2)      (-10, 0)
  P1: S2(自白)  (0, -10)      (-5, -5)
```

### 解答

- **(1) 支配戦略**（P₁の利得＝左の数字で比較）。
    - 相手が S₁(黙秘) のとき：S₁=−2 ＜ S₂=0 → S₂(自白) が良い。
    - 相手が S₂(自白) のとき：S₁=−10 ＜ S₂=−5 → S₂(自白) が良い。
    - どちらの場合も自白の方が大きいので、**S₂(自白) が P₁の支配戦略**。対称なので P₂も同様に **S₂(自白) が支配戦略**。
- **(2) 均衡点**：両者が支配戦略 S₂ を取るので、均衡点は **(S₂, S₂)＝(自白, 自白)＝(−5, −5)**（2人とも懲役5年）。
- **(3) なぜジレンマか**
    - 各自が合理的（自分の利得を最大化）に動くと (自白, 自白)＝(−5, −5) に落ち着く。
    - しかし2人とも黙秘した (S₁, S₁)＝(−2, −2)（懲役2年）の方が、両者にとって明らかに良い。
    - つまり**個々の合理的選択の結果が、全体として望ましくない結果を生む**。これがジレンマである。

## ゼロサムゲームのミニマックス戦略と均衡点

### 問題

> 次はゼロサムゲームで、行列は P₁ の利得を表す（P₂ の利得はその符号反転）
> P₁ の maximin と P₂ の minimax を計算し、均衡点（鞍点）が存在するか判定せよ
> 存在するならどの戦略の組か答えよ。

```
            C1   C2   C3
  R1        4    3    7
  R2        6    5    8
  R3        2    1    9
```

### 解答

- **P₁ の maximin を求める**
    - 各行の最小値：R1 → min(4,3,7)=3、R2 → min(6,5,8)=5、R3 → min(2,1,9)=1
    - maximin ＝ max(3, 5, 1) ＝ **5**
- **P₂ の minimax を求める**
    - 各列の最大値：C1 → max(4,6,2)=6、C2 → max(3,5,1)=5、C3 → max(7,8,9)=9。
    - minimax ＝ min(6, 5, 9) ＝ **5**（列 C2）。
- **判定**：一般に maximin ≤ minimax が成り立つが、ここでは **maximin ＝ minimax ＝ 5** で一致する。よって**均衡点（鞍点）が存在する**。
- 均衡点は **(R2, C2)＝5** である。この値は「R2 の行の最小値」かつ「C2 の列の最大値」になっている（鞍点の条件）。
- この点では戦略が安定する。P₁ は相手が C2 を取る限り R2 から動くと損をし、P₂ も R2 に対して C2 から動くと損をする。よって互いに戦略を変えるインセンティブがない安定点である。
