## 北村_BERT
### BERTとは何か
- 2018年にGoogleが発表したNLPモデル
- 前後の文脈を両方見て単語の意味を理解する
### GPT・ELMoとの違い
- Transformerエンコーダのみ
- GPTは「次の単語を予測する」モデルなので、処理時に未来の単語を見ることが出来ない
- BERTはエンコーダのみを使うので、最初から全単語同時に見ることが出来る
### Self-Attention
- BERTの場合、12または16分並列でMulti-Head-Attentionする
- GPTはCasual Maskで未来を隠すが、BERTはエンコーダなので未来を隠す必要はない。パディングマスク（意味のない穴埋めトークンを無視する）だけ使う
### Embeddingsの3種構造
- BERTへの入力は3種類の埋め込みの和
- [CLS]は文全体の要約として使われる特殊トークン、[SEP]は文の区切り
```text

[CLS] my dog is cute [SEP] he likes playing [SEP]

  ↓       ↓               ↓

トークン埋め込み（単語の意味）

セグメント埋め込み（文Aか文Bか）

位置埋め込み（何番目の単語か）

```
### 事前学習
- ラベル無しデータだけで行なえる
- Wikipediaや書籍のテキストををそのまま使えるので、人手でアノテーションする必要がない
```text
入力: [CLS] こたつで [MASK] を 食べる [SEP] こたつで テレビを みる [SEP]
         ↑                                    ↑
       NSPの判定に使う                    MLMの穴埋め予測
```
#### MLM（Masked Language Modeling）
- BERTで双方向の文脈を学習させるには、単語を隠して前後両方から予測させるという穴埋め形式が自然な解決策
```text
選ばれた15%のトークンに対して：
  80%  → [MASK] トークンに置き換える
           例: みかん → [MASK]
  10%  → 語彙からランダムな別の単語に置き換える
           例: みかん → りんご
  10%  → そのまま変えない
           例: みかん → みかん
```
- 事前学習中は[MASK]が頻繁に登場するが、ファインチューニングや実際の推論時には[MASK]は一切登場しない
- 「[MASK]を見たときだけ一生懸命考える」というクセがついてしまうと、実データに対する汎化性能が下がる
- 10%をランダムな単語にすることで「どのトークンも正しいとは限らない」という状況を作り、モデルが常に全トークンに注意を払うようになる
- さらに10%をそのままにすることで、実際の単語に対する表現力も保つ
**クロスエントロピーの基本形**
$$H(p, q) = -\sum_{v \in V} p(v) \log q(v)$$
- $p$：正解の分布（正解単語だけ確率1、他は0のワンホット）
- $q$：モデルの予測分布（各単語の確率）
- $V$：語彙全体
- 正解が「みかん」なら $p(\text{みかん}) = 1$ で他は0なので、この式は実質 $-\log q(\text{みかん})$ だけが残る
**マスク1個あたりの損失**
$$\mathcal{L}_{MLM}^{(i)} = -\log P(w_i^* \mid w_1, \ldots, w_{i-1}, w_{i+1}, \ldots, w_K)$$
- $w_i^*$：位置 $i$ の正解単語
- 条件部分：位置 $i$ **以外**の全トークン（前後両方）
- 「前後の文脈全部を見たうえで、正解単語の予測確率がどれくらい高いか」をログで測ったもの
- 確率が高いほど損失は小さくなる
**全マスク位置での損失**
$$\mathcal{L}_{MLM} = -\sum_{i \in M} \log P(w_i^* \mid w_1, \ldots, w_{i-1}, w_{i+1}, \ldots, w_K)$$
- マスクされた位置の集合 $M$ 全体で損失を足し合わせる
#### NSP（Next Sentence Prediction）
- MLMだけでは単語レベルの文脈は学べるが、「2つの文がどんな関係にあるか」という**文間の関係性**は学習できない
- 質疑応答（質問と回答の関係）や含意判定（2文の論理関係）などのタスクには文間の理解が必要なので、NSPが導入
**タスクの形式**
- 学習データから文ペアを作成
```
50%の確率で：実際に連続する2文（IsNext）
  [CLS] 犬が公園で遊んでいる [SEP] 彼はとても楽しそうだ [SEP]
  → ラベル: true
50%の確率で：無関係な2文（NotNext）
  [CLS] 犬が公園で遊んでいる [SEP] 経済成長率が発表された [SEP]
  → ラベル: false
```
- `[CLS]`トークンに文全体の情報が集約されるので、これを使って二値分類を行う
**NSPの損失関数**
- `[CLS]`の出力から分類用ベクトルを作る
$$h_{pool} = \tanh(W_{pool} \cdot h_{cls})$$
- $h_{cls}$：`[CLS]`トークンの最終出力ベクトル（D次元）
- $W_{pool}$：プーリング用の重み行列
- tanh：値を $-1$ ～ $1$ の範囲に収める活性化関数
- この変換で「文ペア全体の意味をまとめた」ベクトルができる
**IsNext/NotNextの確率へ**
$$P(y \mid w_1, \ldots, w_K) = \text{softmax}(W_{nsp} \cdot h_{pool})$$
- $W_{nsp} \in \mathbb{R}^{2 \times D}$：2クラス分類用の重み行列
- 出力は「IsNextである確率」と「NotNextである確率」の2値
**NSPの損失**
$$\mathcal{L}_{NSP} = -\log P(y^* \mid w_1, w_2, \ldots, w_K)$$
- 正解ラベル $y^*$（trueまたはfalse）に対応する確率の対数をとったもの
- 正解クラスの予測確率が高いほど損失が小さくなる
#### 2タスクを同時に学習する
- 最終的な損失はMLMとNSPの和で計算される
$$\mathcal{L} = \mathcal{L}_{MLM} + \mathcal{L}_{NSP}$$
- 1つのミニバッチで、MLMの穴埋めとNSPの次文判定を同時にこなす
- バックプロパゲーションでBERT全体（エンコーダ部分＋両タスクのヘッド）が更新される
```
1バッチの処理イメージ：
  入力: [CLS] こたつで [MASK] を 食べる [SEP] こたつで テレビを セミ [SEP]
                ↑穴埋め予測                              ↑ランダム単語
  MLM損失: [MASK]→「みかん」、「セミ」→「見る」をどれだけ正しく予測できたか
  NSP損失: この2文は連続しているか（true）をどれだけ正しく判定できたか
  L = L_MLM + L_NSP を最小化するようにパラメータ更新
```
## 越智_CLIPCLAP
### CLIPとは何か
- BERTが「テキストだけ」を扱うモデルだったのに対し、CLIPは**テキストと画像を同時に扱う**モデル
- 2021年にOpenAIが発表
- 「インターネット上に無数ある画像とその説明文のペアを使って、両者の対応関係を学習する」というもの
### 従来手法の問題点
- あらかじめ決めたカテゴリ（「猫」「犬」「車」など）しか扱えなかった
- 新しいカテゴリを認識させるには、追加のラベル付きデータが必要で、非常に手間がかかる
### 対照学習（Contrastive Learning）
- CLIPの学習方法は**対照学習**と呼ばれる
```
バッチサイズN=4の場合、4枚の画像と4つのテキストが入力される
画像:  [犬の写真] [猫の写真] [車の写真] [飛行機の写真]
テキスト: ["Pepper the aussie pup"] ["A cute cat"] ["A red car"] ["A plane"]
N×N = 4×4 = 16 通りの組み合わせが生まれる
→ 正しいペア(対角線): 似ているベクトルになるよう引き寄せる
→ 間違いペア(非対角): 遠ざけるよう学習
```
- 画像エンコーダとテキストエンコーダを同時に学習
- 「正しいペアのベクトルはコサイン類似度が高く、間違いペアは低くなる」ように訓練する
### モデルの構造
#### テキストエンコーダ
- TransformerベースのGPTに似たモデル
- 特徴的なのは`[EOS]`トークン（文末）の出力をテキスト全体の表現として使う点で、BERTの`[CLS]`トークンと役割は似ています。
- masked self-attentionを使っているので、GPTと同じく左から右へ処理
- BERTと異なり双方向ではない
```
入力: [SOS] Pepper the aussie pup [EOS]
                                      ↑
                             この出力ベクトルをテキスト表現として使う
```
#### 画像エンコーダ
**a) ResNet-50ベース**（比較用のベースライン）
- 従来の畳み込みニューラルネットワーク
- 最後のGlobal Average Pooling層を、Transformerの仕組みを使ったAttention Poolingに置き換えることで精度を向上
**b) Vision Transformer（ViT）ベース**（提案モデル）
- 画像をパッチ（小さいブロック）に分割してTransformerに通す方法
- 大規模になるほどResNetより効率が良く、CLIPの本命アーキテクチャ
### ゼロショット分類の仕組み
- 一度も見たことのないカテゴリでも、テキストで指定するだけで分類可能
```
【事前準備】ラベルから分類器を作る
  plane → "A photo of a plane."
  car   → "A photo of a car."
  dog   → "A photo of a dog."
      ↓ テキストエンコーダで変換
  各クラスのテキストベクトルを作っておく
【推論時】
  1. 入力画像を画像エンコーダでベクトル化
  2. 全クラスのテキストベクトルとコサイン類似度を計算
  3. 最も近いクラスを予測ラベルとして出力
```
### プロンプトエンジニアリング
- ラベルが単語一語だと問題が起きる
- "crane"は機械（クレーン）なのか鳥（ツル）なのか判断できない
### CLIPの性能と特徴
#### ゼロショット vs 教師あり
- ゼロショットにもかかわらず、教師あり学習の ResNet50（完全にラベル付きデータで訓練済み）と比べて16データセットで勝った
- 一般的なタスクではゼロショットCLIPが互角以上
#### ロバスト性が高い
- 学習データと分布が変わる（写真→スケッチ、標準→ノイズありなど）と従来モデルは大きく性能が落ちるが、CLIPはその落ち幅が緩やか
- ImageNet-Adversarialでは+51.2%、ImageNet-Aでは+74.4%という大差でResNet101を上回った
- これは自然言語で「一般的な概念」を学習しているため、特定のデータセットに過適合しにくいから
#### 弱点
- 専門性が高いタスク
- 抽象的タスク
- 専門ドメインの概念はWebデータにあまり含まれていないため、学習データ不足になるのが原因
#### ゼロショット vs Few-Shot の逆転現象
興味深いことに、CLIPへ少数のラベルデータを追加して単純に線形分類器を学習させると、ゼロショット時より**性能が下がる**ケースがあります。これはゼロショットで獲得した豊富な事前知識を、少数の例からの学習では十分に活用できないことを示しています。
### CLAPとは何か
- CLAPはCLIPの「音声版」
- 画像の代わりに**音声**とテキストを対照学習で紐づけ
- 発想は同じだが、音声特有の問題がある
### CLAPが解決する音声特有の問題
#### 可変長入力の処理

- 音声は画像と違って長さがバラバラ
**短い音声（T ≤ 10秒）の場合：**
```
3秒の音声 → [3秒][3秒][3秒][1秒の無音] と繰り返して10秒にする
```
**長い音声（T > 10秒）の場合：**
```
音声全体をダウンサンプリング → グローバル情報（全体の流れ）
前・中・後ろから10秒ずつ切り出す → ローカル情報（局所の詳細）
↓
Attention Feature Fusionで最適な融合比率を学習
```
グローバル（全体像）とローカル（部分の詳細）を合わせることで、長い音声でも情報を落とさずに扱えます。
#### ラベルしかないデータへの対応（Keyword-to-Caption）
- 音声データの多くはテキストキャプションではなく、タグやラベルしか持っていない（例：「犬の鳴き声」だけ）
- CLAPは言語モデル（T5）を使ってラベルをキャプションに変換
```
「犬の鳴き声」 → T5 → 「犬が公園で吠えている音」
```
### CLAPの損失関数
CLIPと同じ対称的なクロスエントロピー損失です。
$$L = \frac{1}{2N} \sum_{i=1}^{N} \left( \log \frac{\exp(E_i^a \cdot E_i^t / \tau)}{\sum_j \exp(E_i^a \cdot E_j^t / \tau)} + \log \frac{\exp(E_i^t \cdot E_i^a / \tau)}{\sum_j \exp(E_i^t \cdot E_j^a / \tau)} \right)$$
- 左の項が「この音声に対して、正しいテキストを選べるか」
- 右の項が「このテキストに対して、正しい音声を選べるか」
- 両方向から同時に学習しているので**対称**と呼ばれます。$\tau$（温度パラメータ）は類似度スコアのスケールを調整する学習可能なパラメータ
### 応用例
#### DALL-E2（CLIPの応用）
- CLIPとDiffusionモデルを組み合わせた画像生成モデル
```
テキスト → [CLIPテキストエンコーダ] → テキスト埋め込み
                                          ↓ Prior（Diffusion）
                                     CLIP画像埋め込みを生成
                                          ↓ Decoder（Diffusion）
                                       画像を生成
```
- CLIPが「このテキストに対応する画像はどんな特徴を持つか」をすでに知っているため、それを使って高品質な画像生成がで可能
#### Text2FX（CLAPの応用）
- CLAPとDDSP（微分可能デジタル信号処理）を組み合わせて、自然言語でオーディオエフェクトを制御
```
"This sound is bright" というプロンプトを与えるだけで
→ CLAPの埋め込み空間でイテレーションを繰り返し
→ EQの18個のパラメータを自動調整
→ 音が「明るく」なるよう変化
```
- 新しいモデルを訓練する必要がなく、プロンプト1つで音色を操作できる点が革新的です。
### 共通の課題
- 専門領域（医療・衛星）はWebデータに少ない → 性能低下
- 細粒度認識の弱さ → 「犬」は分かるが「柴犬とポメラニアンの区別」は苦手
- 真の意味理解ではなく、共起パターンを学習している → 背景に依存した誤認識が起きうる
- Webデータ由来なので社会的偏見を内包する可能性がある
- テキストエンコーダにCLIP Transformerを使うと過学習が強い
  → RoBERTaのほうが汎化しやすい
