## なぜ線形回帰では分類できないのか

たとえば「メールがスパムかどうか」を予測したいとする。出力は 0（スパムでない）か 1（スパム）の二値。

線形回帰 $y = w^\top x + b$ をそのまま使うと、出力が 0〜1 の範囲に収まらず、1.5 や −0.3 のような値が出てしまう。確率として解釈できない。そこで、出力を「0〜1 の範囲に押し込める関数」が必要になる。
## シグモイド関数の導入

その役割を果たすのが **シグモイド関数**（ロジスティック関数）。
$$\sigma(z) = \frac{1}{1 + e^{-z}}$$
- 入力 $z$ が任意の実数でも、出力は必ず $(0, 1)$ の範囲に収まる
- $z = 0$ のとき $\sigma(0) = 0.5$
- $z \to +\infty$ で $\sigma(z) \to 1$、$z \to -\infty$ で $\sigma(z) \to 0$
- 微分が $\sigma'(z) = \sigma(z)(1 - \sigma(z))$ と綺麗な形になる（学習時に便利）

ロジスティック回帰は、線形結合 $z = w^\top x + b$ をシグモイド関数に通して、出力を確率として解釈する。
$$P(y=1 \mid x) = \sigma(w^\top x + b) = \frac{1}{1 + e^{-(w^\top x + b)}}$$
## 損失関数：交差エントロピー

- パラメータ $w, b$ をどう学習するか
- 線形回帰のように二乗誤差を使うと、シグモイド関数の非線形性のせいで損失関数が非凸になり、最適化が難しくなる
- 代わりに使うのが**交差エントロピー損失**
$$L_i = -\bigl[y_i \log \hat{y}_i + (1-y_i) \log(1-\hat{y}_i)\bigr]$$
-  $\hat{y}_i = \sigma(w^\top x_i + b)$ は予測確率
- データセット全体では、全サンプルの平均をとる
$$L(w, b) = -\frac{1}{N}\sum_{i=1}^{N}\bigl[y_i \log \hat{y}_i + (1-y_i)\log(1-\hat{y}_i)\bigr]$$
- この損失関数は $w, b$ に関して**凸関数**になるため、勾配降下法で大域的最適解に収束できる
## 勾配降下法によるパラメータ更新

損失を最小化するために、各パラメータの勾配を計算して更新する
まず $z_i = w^\top x_i + b$ として、損失の勾配を連鎖律で求めると、
$$\frac{\partial L_i}{\partial w} = (\hat{y}_i - y_i), x_i, \qquad \frac{\partial L_i}{\partial b} = \hat{y}_i - y_i$$
ここで $\hat{y}_i - y_i$ は「予測確率と正解ラベルの差」、つまり誤差を表す。

更新式は次のようになる（$\alpha$ は学習率）。

$$w \leftarrow w - \alpha \frac{1}{N}\sum_{i=1}^N (\hat{y}_i - y_i), x_i$$

$$b \leftarrow b - \alpha \frac{1}{N}\sum_{i=1}^N (\hat{y}_i - y_i)$$
## 決定境界

- 学習後、新しい入力 $x$ に対して $\hat{y} = \sigma(w^\top x + b)$ を計算し、閾値（通常 0.5）と比較して分類
- $\hat{y} = 0.5$ となるのは $w^\top x + b = 0$ のときで、これが**決定境界**
- 入力が2次元なら直線、3次元なら平面、一般には超平面
- ロジスティック回帰は本質的には線形分類器
## 正則化

- 過学習を防ぐために、損失関数に正則化項を加えることが一般的。
- **L2 正則化（Ridge）：** $L + \frac{\lambda}{2}|w|^2$ — 重みを全体的に小さくする
- **L1 正則化（Lasso）：** $L + \lambda|w|_1$ — 一部の重みをちょうど 0 にして、特徴量選択の効果がある
## 多クラスへの拡張

- 二値分類を多クラス（$K$ クラス）に拡張するには、シグモイドの代わりに**ソフトマックス関数**を使用
$$P(y=k \mid x) = \frac{e^{w_k^\top x + b_k}}{\sum_{j=1}^{K} e^{w_j^\top x + b_j}}$$
- 各クラスに対して線形結合を計算し、全クラスの確率の合計が 1 になるように正規化
- これは**ソフトマックス回帰**（多項ロジスティック回帰）と呼ばれ、ニューラルネットワークの出力層でも広く使われている。
## まとめ

全体の流れを整理すると、「入力に対する線形結合 → シグモイドで確率に変換 → 交差エントロピーで損失を計算 → 勾配降下法でパラメータを更新」というのがロジスティック回帰の学習パイプライン。
シンプルでありながら、確率的な解釈ができ、損失が凸で最適化しやすく、勾配の形も綺麗という、非常にバランスの良いモデル。ニューラルネットワークの1層分（活性化関数がシグモイドの場合）と見なすこともでき、深層学習の入り口としても重要な位置づけにある。
