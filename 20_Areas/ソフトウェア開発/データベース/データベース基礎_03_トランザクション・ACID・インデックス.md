# トランザクションとは

- たとえば銀行振込を考える

```txt
Aさんの残高を1000円減らす
Bさんの残高を1000円増やす
```

- この2つは必ずセットで成功しないといけない

もし、

```txt
Aさんの残高だけ減った
Bさんの残高は増えていない
```

- そこでトランザクションを使う
- 複数の処理を「全部成功」か「全部失敗」のどちらかにする仕組み

```txt
予約を作成する
空室数を減らす
支払い情報を作る
```

- この途中で失敗したら、全部なかったことに戻す必要がある
- これを **ロールバック** と言う

---

# ACID特性

トランザクションには **ACID** という性質があります。

|性質|意味|
|---|---|
|Atomicity|原子性。全部成功か全部失敗|
|Consistency|一貫性。DBのルールを壊さない|
|Isolation|独立性。同時実行しても干渉しにくい|
|Durability|永続性。確定したデータは消えない|

初心者のうちは、まずはこれだけ覚えれば大丈夫です。

> トランザクションは、複数のDB操作を安全にまとめる仕組み。

# インデックスとは

インデックスは、検索を速くするための仕組みです。

本の索引をイメージするとわかりやすいです。

本で「トランザクション」という単語を探すとき、最初から全ページを読むのは大変です。  
でも索引があれば、何ページに載っているかすぐわかります。

DBのインデックスも同じです。

たとえば、

```sql
SELECT *
FROM customers
WHERE email = 'yamada@example.com';
```

という検索をよくするなら、`email` にインデックスを貼ると速くなります。

```sql
CREATE INDEX idx_customers_email
ON customers(email);
```

ただし、インデックスを貼れば貼るほどよいわけではありません。

インデックスが多すぎると、

```txt
データ追加が遅くなる
更新が遅くなる
DB容量を使う
```

というデメリットもあります。

基本は、

> よく検索条件に使う列にインデックスを貼る

と考えるとよいです。
