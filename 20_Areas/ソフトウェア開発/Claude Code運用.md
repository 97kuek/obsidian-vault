# Claude Code運用

> Claude Code を効率よく運用するためのノウハウ。①大規模コードベースでの「ハーネス設計」、②トークン消費の節約、の2軸でまとめる。

## 大規模コードベース運用

> 大方針: **モデルの賢さに頼るのではなく、Claudeが迷わず動ける環境（ハーネス）を人間が設計すること。**

### 大規模コードベースで苦戦する理由と前提

- **探索方式の違い**: RAG型（事前インデックス）とは異なり、Claude Codeはライブなコードを直接探索（Agent型）するため、情報が古いという問題は起きない。
- **迷子問題**: 巨大なリポジトリに丸投げすると、関連ファイルに辿り着く前にコンテキスト（記憶容量）を使い切ってしまう。
- **環境への依存**: アウトプットの質は「モデルの性能」以上に、周囲に構築された「ハーネス（環境設定）」に大きく左右される。

### ハーネス（拡張機能）の正しい使い分け

Claudeがコードを読むための補助線となる各機能の役割は以下の通り。

- **CLAUDE.md（常時ロード）**
    - 毎回読み込まれるため**薄く書く**のが鉄則。
    - 全体像、ディレクトリの役割、致命的な注意点（ガッチャ）、社内用語のみに留める。
- **Hooks（自動実行スクリプト）**
    - Claudeに口頭で頼むより、lintやフォーマッタを**決定的に強制実行**させるために使う。
    - セッションの振り返りと学びの自動追記にも有効。
- **Skills（オンデマンド知識）**
    - 特定タスク（決済、セキュリティ等）の時だけ呼び出される専門知識。
    - 「何でもCLAUDE.mdに書く」のを防ぎ、コンテキストを節約する。
- **Plugins（設定のパッケージ化）**
    - 出来上がった設定をチーム全体に配布し、一部の人だけが使いこなす「属人化」を防ぐ。
- **LSP（言語サーバー連携）**
    - テキスト検索（grep）のノイズをなくし、**シンボル単位（関数や型）で正確にジャンプ**させる。大規模環境では最も費用対効果が高い。
- **MCPサーバー（外部ツール連携）**
    - 社内APIやJira等と繋ぐ機能。ただし、基本のハーネスが整っていない初期段階で増やすべきではない。
- **サブエージェント（並列処理）**
    - 巨大な変更の際、「探索して要約するだけの専用エージェント」を作り、親エージェントのコンテキスト消費を抑える。

### 大規模導入を成功させる3つのベストプラクティス

- **パターン1：コードベースを読みやすくする**
    - CLAUDE.mdはルートに全体像、詳細はサブディレクトリに分けて階層化する。
    - ルートではなく、**タスク対象のサブディレクトリでClaudeを起動**する。
    - `.ignore`や`permissions.deny`でノイズ（自動生成ファイルなど）を隠す。
    - 複雑なリポジトリなら、目次となる`MAP.md`を作る。
- **パターン2：モデルの進化に合わせて設定を更新する**
    - 「1ファイルずつ編集して」といった**過去のモデル用の制限が、最新モデルの足を引っ張る**ことがある。
    - 設定（CLAUDE.mdやHooksなど）は3〜6ヶ月に一度見直す。
- **パターン3：オーナーシップを明確にする**
    - ボトムアップ任せにせず、専任チームや責任者（DRI）を決める。
    - セキュリティやガバナンスを含めたルール作りを早期に行う。

### 個人開発・小規模チームへの応用ポイント

- CLAUDE.mdには何でも書かず、必要最小限にする。
- lint/formatはプロンプトではなくHooksで機械的に強制する。
- 大きめのタスクは、サブエージェントに「探索・要約」を任せてから編集に入る。

## トークン節約の実践

### トークン消費を抑える10の実践テクニック

- **タスクの区切りで `/clear` する**
    - 別の作業に移る際はセッションをリセットし、無関係な履歴によるコスト増を防ぐ。
    - ちょっとした質問は履歴に残らない `/btw` を活用する。
- **コンパクション（圧縮）をカスタマイズする**
    - `/compact` を実行する際（または `CLAUDE.md` 内）で、「テスト結果を優先して残す」など重要な情報が消えないように指示する。
- **用途に合わせてモデルを使い分ける**
    - 基本は「Sonnet 4.6」、複雑な設計やデバッグは「Opus 4.6」、サブエージェントなどの単純作業は「Haiku 4.5」と使い分ける。
- **探索・調査はサブエージェントに任せる**
    - 大量のファイルを読み込む作業は別プロセスに委任し、メインの会話には「要約」だけを返させてコンテキストを節約する。
- **`CLAUDE.md` は200行以内にスリム化する**
    - 毎回ロードされるため最小限にし、特定タスクの手順（デプロイやPRレビューなど）は「Skills」に逃がす。
- **`.claudeignore` で不要ファイルを弾く**
    - トークンを爆食いするロックファイルやビルド成果物（`node_modules/` など）を初期スキャンから除外する。
- **MCPサーバーのオーバーヘッドを管理する**
    - 使っていない連携ツールは無効化する。GitHubやAWSなどはMCP経由よりCLIツールを直接叩く方がコスト効率が良い。
- **拡張思考（Extended Thinking）のレベルを下げる**
    - 簡単なタスクは `/effort low` に設定し、課金対象となる思考トークンを節約する。
- **Hooks で巨大な出力を前処理する**
    - 数万行のログを直接読ませず、事前にスクリプト等でエラー部分だけを抽出・フィルタリングしてから渡す。
- **プロンプトを具体的に書く**
    - 曖昧な指示は無駄なファイル探索を誘発するため、「どのファイルの、どの関数を、どうするか」を明確にする。

### コスト管理・状況確認コマンド

| コマンド | 用途 |
|---|---|
| `/context` | **現在のコンテキスト消費の内訳を表示。**（何が重いのか一目でわかるため特に有用） |
| `/cost` | 現在のセッションのトークン使用量・金額を表示。 |
| `/stats` | 利用パターンを表示（Pro / Maxプラン向け）。 |
