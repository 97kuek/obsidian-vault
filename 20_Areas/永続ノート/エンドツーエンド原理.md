# エンドツーエンド原理

賢い処理は**端末**が担い、ネットワーク内部は「パケットを転送するだけ」の単純機能に徹する、というインターネットの設計原理。

## 核心的な主張

- ネットワークに高度な機能を詰め込まず、**端末側のソフトウェアに複雑さを寄せる**。
- 中核（ネットワーク）は "dumb"（賢くない）に保つ。やることは宛先へのパケット転送だけ。
- 信頼性（再送・順序保証）やセキュリティ（暗号化）といった付加機能は、通信の両端＝端末が責任を持つ。

## 詳細・補足

- IP は転送のみ。失われたパケットの回復（再送・順序整序）は端末の **TCP** が担う。
- **E2E暗号**もこの延長：途中の ISP やキャリアが覗いても中身は読めない。賢い処理を端から端で完結させる発想。
- なぜ中核を単純にするのか：
	- 多様な物理層・アプリ層技術を一つのIP上に統合できる（砂時計モデルのくびれ）。
	- 中核がシンプルなほど拡張・スケールしやすい。
	- 全経路の信頼性は各リンクでは保証できない → 経路全体を見る端末が担うのが理にかなう。
