# 階層的委任による分散管理


名前空間やアドレス空間を**階層化**し、各階層の管理を下位組織へ**委任**することで、スケーラビリティと耐障害性（SPOF回避）を同時に満たす設計原理。

## 核心的な主張

- 一箇所で全部を管理すると、規模拡大で破綻し、そこが止まると全体が止まる（**単一障害点 / SPOF**）。
- 名前空間を階層に区切り、各組織が**自分の担当範囲だけ**を管理する。
- これにより ① 衝突回避は同一階層内だけ考えればよくなり、② 管理が分散して中央集権を避けられる。＝**分割統治**。

## 詳細・補足

- 同じ構造が複数の場所に現れる：
	- **DNS**：root → TLD(.jp) → 2LD(ac.jp) → 組織。各権威サーバが下位を委任。
	- **IPアドレス**：ICANN → RIR → NIR(JPNIC) → 組織。
	- **AS**：経路制御ポリシーを決める単位として番号を委任。
- 一意性（重複してはいけない）と非中央集権（一極集中してはいけない）という相反要求を、階層委任で両立させる。
- 対照的な失敗例が初期の **HOSTSファイル集中管理**：ホスト増加で破綻し、管理停止が通信停止に直結した。

## 他の概念との関係

- 関連: [[エンドツーエンド原理]]（中核を単純・分散に保つ思想として共通）
- 対比: 集中管理（HOSTSファイル／公衆交換電話網）
- 応用: 大規模システムの権限委任・名前空間設計一般
