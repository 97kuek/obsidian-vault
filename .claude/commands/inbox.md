`00_Inbox/` に溜まった殴り書きノートを、Claudeが整理する。**人間は「Inboxに書く」だけ、種別・置き場所・テンプレ・タグ・日付は考えない**——その後始末をこのコマンドが引き受ける。日々〜気が向いたときに実行する。

**原則：**
- **このコマンドは整理案を「提案」し、ユーザーの承認を得てから実行する。**
- **人間が書いた本文は消さない。** 整理＝構造化・移動・命名であって、要約して捨てることではない（明らかな重複や本人が消してと言った場合を除く）。
- 破壊的操作（移動・上書き）の前に作業前コミットを提案する。
- 迷ったら勝手に決めず、その1枚について短く確認する。

---

## 1. 事前準備

`VAULT_INDEX.md` と `CLAUDE.md`（ファイル命名規則・フロントマター規約）を読む。`99_Templates/` の各テンプレートが「整形の型」になる。

```bash
cd "$CLAUDE_PROJECT_DIR" 2>/dev/null || cd .
echo "=== 00_Inbox の中身 ==="
find ./00_Inbox -name '*.md' ! -name 'はじめに.md' 2>/dev/null
```

Inboxが空なら「Inboxはクリーンです」と報告して終了。

---

## 2. 各ノートを読んで種別判定

`00_Inbox/` の各ノートを読み、内容から種別を判定する。判断材料はタイトル・本文・既存のタグ・文体。

| 種別の見分け | 判定の手がかり | 行き先 |
|---|---|---|
| 授業・勉強会メモ | 講義名・日付・聴講メモ的な箇条書き | 既存の科目ノート（例 `人工知能.md`）へ**追記**、または `20_Areas/該当/` に新規 |
| 論文メモ | 論文タイトル・著者・手法の記録 | `30_Resources/該当トピック/略称_メモ.md`（新規） |
| 実験・作業ログ | 実験条件・コマンド・結果・日付 | `10_Projects/該当プロジェクト/YYYYMMDD_タイトル.md` |
| 永続ノート（概念） | 「自分の理解」を1概念で書いている | `20_Areas/永続ノート/概念名.md`、`【MOC】永続ノート` にリンク |
| プロジェクトノート | 特定プロジェクトの設計・まとめ | `10_Projects/該当/タイトル.md` |
| 雑メモ・思いつき | 断片・ToDo・未整理 | 育てば永続/プロジェクトへ昇格、不要なら破棄、保留はInbox据え置きを提案 |
| MOC / レビュー / ドラフト | それぞれの体裁 | 規約どおりの場所 |

判定が曖昧な1枚は、選択肢を添えてユーザーに確認する。

---

## 3. 整理案を提示

```
## Inbox 整理案（YYYY-MM-DD）

| # | 元ファイル | 判定 | 整形後 | 行き先 | タグ | リンク/MOC | 処理 |
|---|---|---|---|---|---|---|---|
| 1 | 思いついたAttention.md | 永続ノート | アテンション機構.md | 20_Areas/永続ノート/ | permanent | [[Transformer_メモ]]/【MOC】永続ノート | 新規作成して移動 |
| 2 | 6/13人工知能メモ.md | 授業メモ | （追記） | 20_Areas/授業/人工知能.md | - | - | 既存ノートへ追記 |
| 3 | あとで調べる.md | 雑メモ | - | 00_Inbox/ | inbox | - | 据え置き（情報不足） |
```

各行で「新規作成／既存へ追記／据え置き／破棄」を明示する。**命名は `CLAUDE.md` の規則に従う**（知識ノート＝日付なし、ログ＝`YYYYMMDD_`、論文＝`_メモ`）。

---

## 4. 承認後に実行

ユーザーが承認した項目だけ、次をセットで行う。

1. （破壊的操作があるなら）作業前コミットを提案・実行
2. 本文を該当テンプレートの型に**整形**（フロントマター付与・見出し整理。本文は残す）
3. 規約どおりの**名前で正しいフォルダへ移動**（`git mv`、または新規作成＋元Inboxファイル削除）
4. **タグ付与**（`CLAUDE.md` のタグ体系に従う／小文字英語・日本語）
5. **リンク・MOC・VAULT_INDEX を更新**（永続ノートは `【MOC】永続ノート`、論文は `【MOC】30_Resources` 等）
6. 仕上げに切れリンクを確認（`/vault-review` の手順1）

---

## 5. サマリー出力

```
## Inbox 整理 完了（YYYY-MM-DD）
- 処理: N件（新規 a / 追記 b / 据え置き c / 破棄 d）
- 移動先の内訳: …
- 更新したMOC / VAULT_INDEX: …
- 残ったInbox: M件（理由）
```

整理しきれなかったものは理由を添えてInboxに残す。深い棚卸し（重複・統合）は `/vault-gc`、定期点検は `/vault-review`。
