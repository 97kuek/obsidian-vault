# 人工知能

## 知能（Intelligence）とは何か

- 辞書的には「外界の状況の知覚をもとに認識・理解を行い、得られた知識・情報を使って問題を適切に処理し、適応する能力」
- 哲学辞典では「**学習能力・抽象思考能力・環境適応能力**」の3つに整理される
- **何ができれば「知能がある」と言えるのか**が中心的な問いである。
- 定義を並べても、判定基準ははっきりしない
### 「機械は考えるか」という思考実験

- 知能の本質を考えるため、機械と生物の運動を比較する。

- 機械は**飛ぶ**か → Yes（ただし鳥や昆虫のように羽ばたきはしない）
- 機械は**泳ぐ**か → No（船は航行するが「泳ぐ」とは言わない）
- 機械は**走る**か → Yes
- 機械は**駈ける（gallop）**か → No

- 機械は同じ「移動」という結果を達成しても、**生物と同じ方法を使っているわけではない**。
- ここから「機械が考えるとは何か」という核心的な問いが生じる。

- この問いには、次の2つの論点がある。
  - **Deep BlueやAlphaGoの勝利**：人間に勝つ結果を出せても、そこに知性があると言えるか。
  - **サール（Searle）の「中国語の部屋」**：規則に従って正しい中国語の応答を返せても、部屋の中の人が中国語を理解しているとは限らない。
- 中国語の部屋は、**記号操作ができることと理解していることは別である**と主張する思考実験である。
- したがって、結果を出せることだけで知性の存在を判断できるとは限らない。

- 知能の判定基準として、次の2方向が対比される。
  - **人間を基準にする**：人間の発想や判断を基準とするが、誤りや文化依存性をどこまで模倣するかが問題となる。
  - **合理性を基準にする**：可能な限り広く深く推論することを基準とする。評価基準は明確だが、必ずしも人間的ではない。
- この対立が、AI研究の4象限を構成する一つの軸となる。

## AI研究の4象限

- AI研究を**2つの軸で4つに分類**する重要な枠組みである。

||**人間を基準**(誤りも真似るかも)|**合理性を基準**(可能な範囲で最善解を求める理想像)|
|---|---|---|
|**行動を基準**(外から見た振る舞い)|**人間のように行動する**|**合理的に行動する**|
|**思考を基準**(内部の思考方法)|**人間のように考える**|**合理的に考える**|

縦軸が「**思考か行動か(内部のプロセスか、外から見える振る舞いか)**」、横軸が「**人間を基準にするか、合理性を基準にするか**」である。各象限を見ていく。

**① 人間のように行動する**：「人間が行う場合には知能を必要とする機能を達成する機械を創る技術」。評価方法は**チューリングテスト**(人間と区別がつかない応答ができれば合格。emacsの Eliza が例として挙げられている)。

**② 合理的に行動する**：「知的行動の自動化に関する計算機科学の一分野」。これが**「合理的エージェント」アプローチ**で、知識や信念から可能な限りの合理的結論を得る。

**③ 人間のように考える**：「計算機が考えるようにする。真の意味で心を持った機械を造る、刺激的で新しい試み」。これが**認知科学的アプローチ**(人間の記憶や思考の過程を考える)。スライドではこの象限が青枠で強調され、「機械で意識や人間性(humanity)を実現したい、科学的に解明したい」という究極の目標を持つ領域とされている。

**④ 合理的に考える**：「計算機モデルを用いた心の機能の研究/認識・推論・行為を可能にする計算の研究」。**形式的な推論や思考の規則=論理学**のアプローチ。

重要な注意点が2つある。**4枠どれもAIの研究**であること(AIは知性を広く科学的に解明する目的を持つため)。そして、**人間も時に合理的・時に非合理的、時に論理的・時に直感的**であり、これらの枠はきれいに分かれているわけではない、ということである。

### 強いAIと弱いAI

- **強いAI**：心(意識や自我)を実現する。アルゴリズムで心が実現可能だという立場。**サール自身はこれに否定的**
- **弱いAI**：機械は心・意識を持たない。そこまでは追求しない(できない)。

## 合理的エージェントアプローチと限定合理性

### 合理的エージェントとは

- 保持している情報(知識・時間・使える資源)に基づき、常に最善の決定を下すもの
- この意味で「知的」
- ここでポイントは「**調べれば分かるのに『知らなかったことにしよう』は非合理**」という点
- **合理性には、必要な知識を取得するという行為も含まれる**

### 限定合理性

- 可能な限り、限りある範囲での合理性
- 完全な合理性は無理だが、与えられた資源の範囲内で最善を尽くす
- これがこのアプローチの現実的な立場
- **人間の思考過程を真似ても真似なくてもよい**

## 現代のAIの定義


1. 「環境を知覚し、**成功の可能性を最大化する**行動をとるシステム」——しかし"成功"は曖昧。
2. 「**期待効用(expected utility)を最大化する**システム。ただし効用(目的)は人間の設計者が与える」——しかし多くの場合、効用は固定できず、設計者が与えきれない。
3. 「**人間の真の目的が不明確な場合でも、その人間の代わりに動作するシステム。したがって、何を最大化すべきかを学習し、目的が不明確なままでも適切に機能する必要がある**」

この3段階目は、近年のAIの考え方を象徴している。「目的を与えられて最適化する機械」から「**目的そのものが不確実で、それを学習しながら適切に振る舞う機械**」へと定義が深化しているのである。

知的に行動するエージェントの条件として、最終的に4つが挙げられる：**①ゴールと状況に適切に行動する、②環境やゴールの変化に柔軟に対応する(適応)、③経験から学ぶ(学習)、④知覚的・計算的限界の中で適切に選択する(限定合理性)**。

## エージェントとは何か

- ラテン語の **agere(行動する)** から **Agent(行動する者、動作主)** が来ている
- つまりエージェントは本質的に「**行動する主体**」
- エージェントとは、**センサー**を通して環境を認知し、**エフェクター**により環境に対して自律的・合理的に行為を行う**実体(entity)**。
- ここで「実体」には、機械・ロボット・人間のような**物理的実体だけでなく、仮想的なソフトウェア・プログラムも含まれる**
- エージェントは**センサーで環境を知覚し → 内部で判断し → エフェクタ(アクチュエータ)で環境に働きかける**。
- そしてその行為が環境を変え、変わった環境をまた知覚する……という**ループ**を回す

### エージェントの具体例

- **ロボット**：認知=超音波・接触・距離計・カメラ、行為=移動・方向転換
- **ソフトウェアエージェント**：認知=プログラムへの入力(数値・文字・音・画像)、行為=プログラムからの出力
- **組織の代理(delegates)**：Microsoftや日本政府の代理として動く専門家(教師・医者・トレーダー・旅行代理店…)。あるいはサーモスタット、飛行機の制御プログラム、Google Car、Mars rover など
- **生き物**：犬・魚・鳥・昆虫・細菌・ウイルス(物理学でよく扱う対象)

### 合理性・自律性・創発

- エージェントには、次の3つの重要な性質がある。
- **合理性**
  - 分かっていることから予測できる範囲で、最大限の成功を目指す性質である。
  - 知り得ない事態による失敗は、不合理とは言えない。
  - 理想合理的エージェントは、知覚の履歴と保有する知識を用いて、最大の性能を引き出す動作を選ぶ。
  - 性能を測るには**性能基準（performance measure）**が必要である。
  - 性能基準には、ゴール達成・速度・使用資源・環境への影響などがあり、問題ごとに異なる。
- **自律性（autonomy）**
  - 行為を環境や自らの経験に基づいて決定できる性質である。
  - 外部から事前に与えられた命令だけに従う場合、自律的とは言えない。
  - 経験がない段階で動くための事前知識は必要だが、環境変化への適応も求められる。
- **創発（emergence）**
  - 個々のエージェントが単純な動作しか行わなくても、全体として知的な行動や現象が現れることである。
  - 例として、渡り鳥のV字編隊やイワシの群れがある。
  - 各個体の自律的な判断から、集団全体の秩序が形成される。

なお歴史的補足として、「エージェント」という語は1988年頃からAI分野で使われ始め、1992-3年頃から「自律した計算主体・プログラム」として計算機科学やソフトウェア工学に広く普及した、と述べられている。

## エージェントが備えるべき能力

「自律的・知的に振る舞うには、何をプログラムとして内部に実装すればいいか」を整理する。図に沿って**4つの要素**が挙げられる。

**① 認知・判断能力**：可能な行動のモデル、環境の認知とその内部表現(モデル)、自分の行為が環境に与える影響のモデル。 → 課題：**計算機内に環境を表現(モデル化)する必要がある**。

**② ゴール・優先順位・重要さ**：希望する状態とその重要さ。これと①を合わせて、希望状態に達しうる行為を選択する。 → 課題：**希望状態を与えるか学習する必要があり、そこへ達する行為の計画=プランニング(planning)が必要**。

**③ 知識と経験(メモリ)**：行為と結果の知識、過去の行為と環境変化の記憶。これにより似た状況で即座に対応できる。 → 課題：**「選択可能な行為と想定結果」(行動モデル)や「望ましい結果の記録」が必要**。

**④ 環境の観測・刺激**：現在の状況の把握、行為結果の確認、想定外の結果や回避すべき状態の認識。 → 課題：**環境の観測と内部モデルとの照合が必要**。

そしてスライドは「**これらの能力をどうやって実現するか……**」と問いかけて次へ続く。実はこの①〜④の「課題」が、この後の講義トピック(探索・プランニング・機械学習・知識表現など)に対応しているのである。導入講義が後の全トピックの「地図」になっている構造が見える。

## エージェントのアーキテクチャ

「内部をどう設計するか」の代表的な4つのアーキテクチャが、**単純なものから高度なものへと積み上がる形**で説明される。これは重要なので順に見る。なお、スライド左上の▲マークは「少し進んだ内容/飛ばすこともある」を示す。

### ① 単純反射エージェント(Simple reflex agents)

**知覚と行為の対応表(lookup table)や if-then 形式のルール(プロダクションルール)**だけで行動を選ぶ。**状態(メモリ)を持たない**——過去の環境・知覚・行動を一切記録しない。動物の反射的な単純行動に対応する。

擬似コードはこうである：

```
Function simple-reflex-agent(percept) returns action
  static: rules  // 条件-行動ルールの集合
  state  ← Interpret-Input(percept)   // 知覚を解釈
  rule   ← Rule-Match(state, rules)   // 合致するルールを探す
  action ← Rule-Action-Part(rule)     // ルールの行動部を取り出す
  return action
```

**問題点**が明確に列挙されている：(a)対応表が巨大になる(チェスは約 10¹²⁰ の状態がある)、(b)知覚への対応しかできず「知覚→行動→新状態→行動」と先を読めない(新状態を知覚しないと次に動けない)、(c)環境変化に対して固定的、(d)学習でテーブルを書き換えるには情報が少ない、(e)**過去を覚えるメモリがないのでループが起こりうる**。

### ② 記憶を持つ反射エージェント(Agents with memory)

- 単純反射エージェントに**内部メモリ**を加えたものである。
- 現在の環境の内部モデルや、過去に知覚した内容を記憶する。
- 内部モデルとは、エージェントが持つ環境の表現である。
- 外部から与えるモデルには、次の2つがある。
  - 自分の行為が環境をどのように変えるかを表す**状態遷移モデル**
  - 自分の行為と独立に環境がどのように変化するかを表す**確率的モデル**
- これにより、環境変化に内部モデルを追随させ、少し先の行為まで推論できる。

### ③ ゴール主導エージェント(Agents with goals)

②に**ゴール(エージェントが望む状態)**を加えたもの。単に状態をメモリに持つだけでなくゴールを持つことで、「どの状態に変わるのがより適切か」を判断できる。**ある状態で可能な行為が複数あるとき、そのうち適切なものを選べる**ようになる(それまでは状態に対し基本的に単一行動が対応)。例：「授業に遅刻しそうなのにラーメンは食べに行かない(ゴール次第だが)」。

この性質は、①②の**反射的(reactive)**に対して**熟考的(deliberative)**と呼ばれる。さらに内部モデルに基づき数手先の状態を考え、ゴールへ達する道筋を作る。この「**ゴールに向けた行為の列をプラン(plan)**」、それを生成する過程を「**プランニング(planning)**」と呼ぶ。

### ④ 効用に基づくエージェント(Utility-based agents)

②に**効用関数(utility function)** `U: State → 数値` を加えたもの。各状態の「望ましさの度合い」を数値で表す。ゴールの達成/未達成という二択だけでなく、**効率・安全性・速度・成功確率**なども数値として考慮できる。

- 効用関数が必要な理由は、ゴールを達成する方法が複数あり、評価が単一指標とは限らないためである。
- 迅速性・成功確率・安全性などは、同時に最大化できない場合がある。
- 効用関数を用いると、各指標の重要度を数値で表して行為を選択できる。
- ゴールへ近づくこと自体も効用関数に含められる。
- したがって、効用ベースのエージェントはゴール主導のエージェントを包含する、より一般的な形式である。

この①→②→③→④という積み上げの構造を押さえておくと、後のトピック(探索・強化学習など)で「このアルゴリズムはどのレベルのエージェントの話か」が見通せるようになる。

## 環境の性質による分類

**エージェントの能力は、置かれた環境の性質に左右される**。環境を分類する4つの軸が示される。

**① アクセス可能性(accessible / inaccessible, observable)**：行為決定に必要な情報をセンサーで**完全に取得できる**なら「アクセス可能(観測可能)」。部分的にしか取れない場合もある(partially observable)。

**② 決定的・非決定的(Deterministic / Nondeterministic)**：次の状態が**現在の状態と行為によって完全に決まる**なら「決定的」。アクセス可能かつ決定的な環境なら、エージェントは不確実性を考えなくてよい。逆に、本当は非決定的でも取得できない情報があると決定的に見えることがある(その場合でも非決定的として扱う)。

**③ 静的・動的(Static / Dynamic)**：エージェントが行為しなければ変化しないのが「静的環境」(扱いやすい)。「動的環境」では、情報を取得せずに熟考していると結果が無意味になることもある。

**④ 離散的・連続的(Discrete / Continuous)**：知覚や行為が**離散値**で表せれば離散的(例：チェス)、**連続値**なら連続的(例：自動運転の速度やハンドル角)。両者の融合もあり、連続値も計算機内部では細かく分けて離散的に扱うことが多い。

これらの軸は、後の探索アルゴリズムや強化学習で「どんな問題設定か」を規定する重要な前提になる。

## AIの歴史

### 前史

- 学問としての問いは2000年以上前(哲学)
- 計算機の誕生としては、Zuse の Z3(1941、リレー式2進計算機)、Wilkes らの EDSAC(1949、最初のプログラム格納式電子計算機、真空管式)
- AIに直結する萌芽として、**McCulloch & Pitts の神経細胞の数理モデル(1943、ニューラルネットの始まり)**、Shannon と Turing のチェスプログラム(1950, 1953)、Minsky と Edmonds のニューラルネット(1951、パーセプトロン)
- ただしこの後しばらくブランクがある

### 第1次ブーム(1956〜60年代後半)

- 基本原理創出の時代**Artificial Intelligience**という言葉は、1956年のダートマス会議で John McCarthy が初めて使った
- これがAIの公式な誕生
- この時期は、一般問題解決器、定理証明、ゲーム理論、論理学などの基礎が築かれた
- McCarthy は MIT で **LISP言語**を作り、Shakey ロボット(’69-71頃、最初の自走式AI制御ロボット、視覚あり)も登場する

### 第2次ブーム：知識の時代とAI産業化

- 「知識こそ力」の時代
- **エキスパートシステム**——特定問題領域に絞ってAIを実用化する試み
- 専門知識を**プロダクションルール(if-then)、フレーム、セマンティックネットワーク、論理式**で表現
- 医療診断の **MYCIN**(血液感染症)、分子構造解析の **Dendral** が代表例
- 専門家にインタビューして知識を計算機に表現する「**知識エンジニア**」という職業も生まれた。
- 商用では DEC の構成助言システム **R1**(年間4千万ドルの経費節減)が成功し、日本の**第五世代プロジェクト(1982-1992)**、米国の MCC、LISP専用マシンなどが登場、1988年には20億ドル市場に成長した

### 1980年代後半〜90年代：転換

- ニューラルネットの復活、遺伝的アルゴリズム、Bio-inspired アルゴリズムが登場(85年〜)
- 一方で「**単にルールを蓄積するだけでは限界**」が見え、専門家からの知識獲得も難しい(**知識獲得ボトルネック**)という問題が明らかになる
- 90年代からは実世界事例ベースの研究へ
- NNなどの機械学習が音声処理・画像処理で有用となり、実環境とのインタラクション、**自律エージェント、マルチエージェント**へ関心が移る

### 2000年〜現在：第3次ブーム

- **機械学習、特に統計に基づく学習が普及**
- インターネットやIoTによる**ビッグデータ**の存在が背景
- ビッグデータに基づく回帰分析・クラスタリング・NN
- これが「今(第3次ブーム)」
- 実世界・ネットワークを介したエージェント間の相互作用と調整のための社会学習、複雑系・複雑ネットワーク、マルチエージェントシステム研究へと広がっている

## 探索による問題解決

### 探索という発想

- ロボットが迷路やパズルを解いたり効率的に移動するには、**実際に行動する前に、頭の中（メモリ内）で行動の列を計画する**機能が必要
- これが探索
- もし地図も路線図も何もなければ、各駅に着いて初めて隣の駅がわかるだけで、その先は不明なのでランダムに進んで運任せにするしかない
- つまり**初期状態・ゴール・行為（次の駅へ移動する）だけでは問題は解けない**
- 必要なのは「環境のモデル」、すなわち地図・路線図と、隣接駅から目的地までの距離(駅数)といった情報
- 計算機はこのモデルを内部に持ち、**実際に移動する代わりに地図上で経路を探す**
- この過程が探索であり、実行前に行動を計画するという意味で「プランニング」の一手法でもある

### 問題の5要素による定式化

1. **初期状態** — エージェントがいる「今」の状態
2. **オペレータ** — 各状態で可能な行為（action）。地下鉄の例なら「隣の駅へ移動」
3. **状態空間** — 初期状態からオペレータの適用で到達できる状態全体の集合。オペレータを順に適用していくと状態空間の中を移動することになり、その軌跡を**経路**という
4. **ゴール検査** — ゴール状態にあるかを判定する
5. **経路コスト** — 経路のコストを返す関数。コストは広義の概念で、経路長・所要時間・乗換回数・運賃やそれらの融合でもよい

- 初期状態からゴールへの**最小コスト経路（オペレータの列）を求める**ことが、探索による問題解決

### 抽象化と性能評価

- 計算機で扱うには**抽象化**が必要
- 状態の抽象化では問題の目的に不要な情報を捨てる
- 地下鉄の例なら「隣の駅の関係」だけが本質的で、「売店がある」「トイレが3つある」は無関係。行為も同様に「駅の移動」だけを残す

- **解が必ず見つかるか（完全性）**
- **良い解か（経路が最適か）**
- **解を得るまでの時間とメモリ量（探索コスト）**

- 全体コストは概念上「**経路コスト＋探索コスト**」だが、両者は単位が違うので単純に足せないこともある

### 代表的な問題例

- **8パズル / 15パズル** — 状態は8個（15個）のタイルと空白の配置。オペレータは「空白を上下左右に動かす（タイルと空白の交換）」。コストは各手1。状態数は 9! = 362,880
- **ハノイの塔** — 状態は各円板の位置のリスト `(x1, x2, x3)`。オペレータは各塔の一番上の円板を別の塔へ移すが、小さい円板の上には乗せられない。ゴールは `(3 3 3)`。この「禁止されるオペレータの条件」を一般のn枚で考える課題が出ている
- **掃除機ロボット（Vacuum world）** — N×Mのマスを動いてゴミを吸う。重要な注意：「右へ移動→吸う→左へ移動」で位置は元に戻っても**状態は異なる**（ゴミが消えたから）。地下鉄の例との本質的な差で、状態を正しく定義する練習になる

- 状態遷移は本来、同じ状態に戻り得るので**グラフ構造**だが、たどった状態を記憶して重複を防げば**木構造**として扱える

### 情報のない探索

- ここからは**ゴールへの方向を示す情報を持たず、基本的に網羅的に状態を調べる**探索

#### 用語の整備

- **分岐数 b（branching factor）** — 1つのノードから1回のオペレータで到達できる子ノード数の平均
- **探索木の深さ m** — 頂点から最も深いノードまでの枝数。有限とは限らない
- **ゴールまでの深さ d** — 頂点から最も浅いゴールまでの深さ
- **閉ノード / closedリスト** — すべての子ノードの展開が完了したノードとその一覧
- **開ノード / openリスト** — まだ展開し終えていないノードとその一覧

- 評価尺度は **完全性・計算量・空間量（メモリ）・最適性** の4つ

#### 探索の基本パターンの発展

**(a) 素朴なランダム探索**

- 子ノードを作り、ランダムに1つ選んで進む
- メモリは小さいがゴールにたどり着く保証がなく、同じ状態を覚えないので**ループに陥る**

**(b) Closedリストの導入**

- 訪問済み状態を履歴として保持し、二度行かないようにする
- 探索木が有限なら必ず終了するが、**解は得られないことが多い**

**(c) Openリストの導入**

- これ以上展開できない地点に着いたとき**戻れるように**、途中で展開したノードを保持する
- これで**探索木が有限なら必ず終了し、解が存在すれば見つけられる**状態に到達する
- Openリストの各要素は `(n p)` のように**ノードと親の対**で持ち、ゴール発見時に親をたどれば経路が復元できる

#### 幅優先探索（BFS）

- 各深さのノードをすべて展開してから次の深さへ
- Openリストから「直前ステップで追加された要素全部」を選ぶ
- **解があれば必ず見つかり、各手のコストが一定なら最適**
- しかし次の深さを展開するために、ある深さの全ノードを覚えておく必要があり、メモリが指数的に増える
- 深さdまでに覚えるノード数は

$$1 + b + b^2 + \cdots + b^d = O(b^d)$$

#### 均一コスト探索

- 各手のコストが異なる場合、BFSは最良解を見逃しうるので、Openリストに**累積コスト** `c' = c + c(n,n')` を持たせ、**最小コストの要素を取り出す**ように一般化する
- 完全かつ最適
- ただし計算量・メモリはBFSと基本的に同じで、大きな問題には使いにくい
- 重要な注意点として、「ここで解を見つけても、もっと低コストの解があるかもしれないので、全ノードのコストがその解以上になるまで探索を続ける」必要がある

#### 深さ優先探索（DFS）

- 最も深いノードを1つ展開し、行き止まったら**バックトラック**して次に深いノードへ
- 実装は**Openリストをスタックに変える**だけ

1. 木が無限の深さを持つと**停止しない可能性**がある
2. メモリは少ない（探索済みの枝を忘れられる）—— 深さmまで各階層で平均b個覚えるので **O(bm)**
3. BFSより効率的なことが多い（BFSは長さdの経路を調べる前に長さd−1を全探索する必要があるため）
4. **最適解が求まるとは限らない**
5. 3と4は子ノードを探索する順序に依存する

**深さ制限探索（Depth-Limited）** — DFSに深さ上限 limit を設け、無限に潜るのを防ぐ。問題は「ゴールまでの深さdが事前にわからない」こと。limit < d だと解が見つからず、limit が大きすぎると深いゴールを先に拾って最適性を失う

**反復深化探索（IDS）** — 深さ制限探索を limit=0,1,2,… と増やしながら繰り返す。これが情報のない探索の決定版

- **完全かつ最適**（BFSの利点）
- **メモリは深さ優先と同じく小さい O(bd)**（DFSの利点）
- 計算量は $O(b) + O(b^2) + \cdots + O(b^d) = O(b^d)$ で、オーダーとしては深さ優先と同等かそれ以下

一見「浅い層を何度も探索し直すのは無駄」に見えるが、計算量のオーダーで見ると繰り返し分の寄与は小さく、これは**完全性・最適性・メモリ効率を同時に得るための代償**である。逆に「無駄を省こうと状態を記憶するとメモリが爆発」してBFSに逆戻りする、という対比が効いている。大きな探索空間でゴールまでの経路長が不明なときの定番手法

- **双方向探索（付録扱い）**
  - ゴールが既知の場合、初期状態からの前向き探索とゴールからの後ろ向き探索を同時に進める。
  - 両者が出会った時点で経路を合成する。
  - 探索範囲を大幅に狭められ、計算量を節約できる。
  - 後ろ向き探索では、ある子ノードに至る親をすべて知る必要がある。
  - チェスのようにゴール状態が膨大にある問題には適用しにくい。

#### 情報のない探索の総まとめ表

l はlimit、d はゴール深さ。

||幅優先|均一コスト|深さ優先|深さ制限|反復深化|双方向|
|---|---|---|---|---|---|---|
|計算量|O(bᵈ)|O(bᵈ)|O(bᵐ)|O(bˡ)|O(bᵈ)|O(b^(d/2))|
|空間計算量|O(bᵈ)|O(bᵈ)|O(bm)|O(bl)|O(bd)|O(b^(d/2))|
|最適性|Yes|Yes|No|Yes/No*|Yes|Yes|
|完全性|Yes|Yes|No|l≥d なら Yes|Yes|Yes|

*深さ制限の最適性は「l≥d かつ、解を見つけても同じ深さまで全部調べるなら Yes」

### ヒューリスティック探索

#### ヒューリスティックとは

- 「heuristic（発見的）」はこの分野では「**必ず正解になるわけではないが、その代わりに効率的に正解（に近い解）を得る**」ことを指す
- 現代的解釈では**平均的な問題は効率的になるが、最悪ケースの性能は必ずしも改善しない**関数
- 地下鉄の例なら「目的地が西なら西へ進む路線を優先する」のように、正しいとは限らないが探索を導く指針になる
- 知識を適用する場所は、**子ノードを展開した後、次にどれをたどるか選ぶところ**
- この順序づけを与える評価関数を**ヒューリスティック関数**と呼ぶ

#### 最良優先探索

- 評価関数 eval-fn の値が最良のノードを優先的に展開する戦略
- アルゴリズムは均一コスト探索とほぼ同じで、**Openリストを eval-fn の値で並べ替えて先頭を取り出す**だけ
- これが以降のヒューリスティック探索すべての**テンプレート（基本形）**になる
- 名前は「最良優先」だが、評価関数が本当に最良のノードを当てられる保証はない（当てられるなら探索は不要）。「最良と推定される／多くの場合ほぼ最良」を選ぶイメージ

#### 欲張り探索（greedy）

- 各ノードからゴールまでのコストを見積もるヒューリスティック関数

$$h(n) = \text{ノード } n \text{ からゴールまでの最短経路の見積り}\quad(\text{ゴールなら } h(n)=0)$$

を eval-fn に使った最良優先探索。「ゴールに近そうなノードから展開する」だけ

- **最適解は必ずしも見つからない**。地下鉄の例で h(n) を「駅nと目的地の地図上の直線距離」とすると、新宿から明治神宮前へ向かう際、直線距離だけを頼りに進むと遠回りになり、最短経路はすぐには見つからない
- 迷路の例では、スタート近くで誤った選択をすると、それを覆すにはその下の探索木をすべて検証し尽くして失敗を確認しないと戻れない、という**探索初期の選択の重要性**が説明されている

#### A*探索

- greedyは効率的だが最適でも完全でもない
- 均一コストは完全で最適だが効率が悪い
- **そこで両者を合わせる**のがA*

- $g(n)$ = 出発点からノードnまでの経路コスト（探索しながら計算できる）
- $h(n)$ = nからゴールまでの最短経路の見積もりコスト

$$f(n) = g(n) + h(n)$$

f(n) は「nを経由する最短経路の見積もりコスト」で、これを評価関数に使う

**完全性・最適性が成り立つための条件**が決定的に重要。h(n) が**許容的（admissible）**、すなわち

$$h(n) \le \text{実際のコスト}$$

（実際より小さめに見積もる＝楽観的）であるとき、f(n) もこの性質を受け継ぎ、nを経由する経路の真のコストを超えない。この許容的なfを使った最良優先探索が**A***で、**完全かつ最適**

**最適性の証明（背理法）** — Gを最適ゴール、ルートからGへの真のコストを f* とする。最適でないゴール G₂（g(G₂) > f*）をA*が先に選んだと仮定する。このときGへの最適経路上には、まだ展開されていない縁のノード n が存在する

- h が許容的だから **f* ≥ f(n)**
- n がG₂より先に展開されなかったから **f(n) ≥ f(G₂)**
- G₂はゴールだから **h(G₂)=0**、よって f(G₂) = g(G₂)

これら3式から **f* ≥ g(G₂)**。これは g(G₂) > f* に矛盾。よってA*が最適でないゴールを先に選ぶことはない

**アルゴリズムの注意点** — 子ノード n' を追加するとき、(i) すでにOpenリストに同じ状態がより高コストで入っていれば置き換える、(ii) Closedリストにある状態でも、より低コストの経路が見つかれば**Closedから外してOpenに戻す**。この再オープン処理がA*の正しさを支える

- 計算量は、hが真の値に近ければ指数オーダーにはならない。が、**hをうまく定義できるか**が問題になる
- h(n)=0 も許容的で、このときA*は均一コスト探索に一致する

#### ヒューリスティック関数の評価

8パズルで2つの許容的ヒューリスティックを比較する。

- **h₁ = ゴールの位置にないタイルの数**
- **h₂ = 各タイルのゴール位置までのマンハッタン距離の和**

講義の例題（初期状態とゴール）では

- **h₁ = 7**（ブロック6以外の7枚が違う位置）
- **h₂ = 4+3+3+1+4+0+3+3 = 21**

**なぜ両方とも許容的か。** h₁は、位置の合っていないタイルは最低1ステップ動かす必要があり、1回に1枚しか動かせないので過大評価しない。h₂も、どの動作も1回でゴールに高々1だけ近づくだけなので過大評価しない

**優位性（dominance）。** 真のコストを h* とすると

$$h_1 \le h_2 \le h^*$$

が成り立ち、h₂のほうが真の値に近い。このとき「**h₂はh₁より優位（dominate）**」という。一般に、より大きい（より真値に近い）許容的ヒューリスティックほど無駄な展開が減って効率的。ただし**ヒューリスティックの計算量が十分小さい**ことが前提

#### 有効分岐数と実測

- 探索効率を1つの数で表す指標が**有効分岐数 b***
- A*が探索したノード総数を $N$、解の深さを $d$ としたとき、深さ $d$ で $N+1$ 個のノードを持つ「均一な木」の分岐数を b* とする

$$N+1 = 1 + b^* + (b^*)^2 + \cdots + (b^*)^d$$

- 例えば深さ5の解を52ノードで見つけたなら b* ≈ 1.9。**b*が1に近いほど効率的**で、b*=1なら一切無駄なく一直線にゴールへ到達したことを意味する

#### 良いヒューリスティックの作り方 — 弱条件問題

- $h$ の作成は難しいが、有力なヒントが**弱条件問題**
- 元の問題の制約を緩めると、解の見積もりが容易になる

- **h₁** は「タイルを外して正しい場所へ直接はめ込み直してよい」と緩めたパズルの解
- **h₂** は「他のタイルの上を通って移動してよい」と緩めたパズルの解
- **地下鉄の直線距離**は「障害物もレールもなく直線移動できる」と緩めた問題

- 緩めた問題は元の問題より簡単に解けるので、その解は元の最短手数を超えず、自動的に許容的になる
- 複数の弱条件問題から $h_1, \dots, h_m$ を作ったときは、以下の式によりすべて**許容的なまま最も真値に近いものを使える**

$$h(n) = \max(h_1(n), \dots, h_m(n))$$

### 高速・実時間手法と反復改良

- これまでの手法は計算量に加え**メモリ使用量**が問題であった
- そこで限られたメモリで動く改良型と、解だけがわかればよい問題向けの手法を扱う

#### 無矛盾なヒューリスティック

- IDA*の前提として導入される
- 許容的なhが**無矛盾**であるとは、任意のノードnとその子n'について三角不等式

$$h(n) \le c(n, n') + h(n')$$

が成り立つこと。「移動コスト以上にh(n')が小さくならない」という、より強い条件

#### IDA*（反復深化A*）

- 反復深化は完全・最適でメモリ効率が良い
- そこで**反復深化を行うが、深さではなく f(n)=g(n)+h(n) の値で探索を打ち切る**のがIDA*
- 各回は深さ優先で進めるが、fの値が境界値 limit を超えたノードはスタックに積まず捨てる。1回終わると limit を上げて再スタートする

- **メモリは極端に小さい**（展開途中で limit 以下のノードだけをスタックに積むだけ）
- **計算量はA*と理論的に同程度**
- **完全かつ最適**
- 欠点：limitを変えるたびに**全部忘れる**ので無駄な繰り返しがある（IDSと同じ欠点だがメモリは少ない）。複雑な領域ではこの繰り返しが効率を下げる

#### 反復改良アルゴリズム

- 問題によっては**経路はどうでもよく、解そのものだけわかればよい**ことがある
- 任意の状態から始め、質を少しずつ改善することを繰り返してゴールを得る

**山登り法**

- 現在状態に可能なオペレーションを施し、最も「良い」状態へ移る
- `value`を高さに例えると山を登るイメージ
- コストで表すなら低い谷を目指すので勾配降下法とも呼ばれる
- 欠点
	- **局所最大/最小** — 局所的なピークに達すると抜け出せず、谷の向こうの高い山へ行けない
	- **高原** — 平らな高い場所でランダムに動き回ってしまう
	- **峰** — 尾根には達するが山頂まで緩やかで`value`がなかなか向上しない

**焼きなまし法**

- 局所最大を抜け出すため、**悪くなる方向にもある確率で動く**手法
- ランダムに選んだ次状態が今より良ければ無条件に進み、悪い場合も $\Delta E$ と温度 $T$ に依存する確率 $\exp(\Delta E/T)$ で進む
- $T$ はスケジュール関数で時間とともに徐々に下げる
- $T$ が大きいほど動き回る確率が高く、$T=0$ でランダム移動が止まる
- 金属を徐々に冷やしてきれいな固体を作る「焼きなまし」になぞらえた命名
- このアルゴリズムは**問題の結果が連続的に変化すること**を仮定
- 囲碁・将棋のように一マスの違いで評価が激変する問題には本来そぐわず、それでもまず適用して後で学習により改善する戦略もあるが、自動運転など失敗の許されない領域では注意が必要

## 複数エージェント系の入門
### なぜAIでゲームの研究がなされるのか

- ルールが比較的単純で、**ゲームの状態を正確に表現できる**こと
- **敵対エージェントが自分を妨害する**という状況の良いモデルになること
- このため最近では、より広く **Adversarial search（敵対的探索）** とも呼ばれる。応用先はサイバーセキュリティ、軍事、金融取引、スポーツなど多岐にわたる。
- チェスや将棋である程度強いエージェントが作れれば、「そこに知能が存在する」ことを示せる、と考えられたこと。
### ゲームの難しさ

- 相手が何をするか分からない**不確実性**を含む
- **組合せが膨大**
- **時間制限**があり、効率の悪さは致命的
- 勝敗が明確で、わずかな差が勝ち負けという極端な結果になる

そこで重要なのが、
- **探索木を制限する（枝刈り、悪手は考えない）** こと
- **相手の手を読む（推測する）** こと

### ゲームを探索問題として定式化する

- **初期状態** … 駒の最初の配置など
- **オペレーター** … プレーヤーがルールに従って行える指し手
- **終端テスト** … ゲームが終了した状態かどうかの判定
- **効用関数／利得関数** … 結果を数値化する関数。例えば勝ち=1、負け=−1、引き分け=0。

### ミニマックス法の考え方

- ゲーム木では、自分と相手が交互に手を打つ
- 相手の最良の手は、自分にとって最悪の手である→双対問題
- 自分の番のノードでは評価値が**最大になる手を選び、相手の番のノードでは評価値が最小**になると考える
- この前提のもと、ゲーム木の**葉から下から上へ評価値を伝播させて手を決める方法**が ミニマックス法
- 評価値は、勝ち = +∞、負け = −∞、引き分け = 0 のように割り当てる
#### アルゴリズム

1. すべての筋道で終端状態に至る完全な探索木を生成する。
2. 各終端状態に効用関数を適用して評価する。
3. 終端状態の効用から、一つ上のノードの効用を計算する（自分の番なら max、相手の番なら min）。
4. これを繰り返し、初期状態（木の頂点）に達するまで計算する。
5. 最大の効用となる手を選択する。
#### ミニマックス法の問題点と対策

**問題点**

- **最後まで読みきれないと評価できない**
- 三目並べならマスが9個しかないので9階層で読みきれるが、一般のゲームではそうはいかない
- **複雑になると計算量が急激に増大する**

**対策**

- **終端まで読まずに深さ N で探索を打ち切り、その状態を評価関数で評価する**方法
- チェスでは実際に評価関数があり、駒の種類ごとの点数や配置の良し悪しを合計して有利・不利を判定
- 完全に正しくはないが、かなりの指針になる。
- ただし「次にクイーンを取られる」「ポーンが成る」など、1手先で評価が劇的に変わることもあり、**どの深さまで読むかを決めるのは難しい**問題

### アルファベータ枝刈り

- ミニマックス法は無駄も多く探索する
- **アルファベータ枝刈り** は、深さを限定するのではなく、**最終結果に関係ない枝の探索を打ち切る**ことで効率を上げる手法
- ここで2つの変数を導入する
	- **$\alpha$** … MAX側（自分）が**少なくともこれだけは確保できる**という下限値
	- **$\beta$** … MIN側（相手）が**これ以上は許さない**という上限値

**$\beta$カット**

- まず左の MAX ノードが葉 {3, 5} を見て **5** に確定
- その左の MIN ノードは「5 以下にしたい」ので、上限 $β = 5$ を持つ
- 次に隣の MAX ノードの葉を見ると最初が **6**
- MAX ノードは最大を取るので、この時点でその値は **6 以上**が確定
- ところが親の MIN は 5 を超える値を選ぶはずがない（6 以上なら 5 より悪いので、相手＝MIN はこちらを選ばない）
- **この MAX ノードの残りの葉「?」は調べる必要がない**
- これが $β$ カット
- 直感的には「**どうせ 5 を超える手を相手は許してくれない**ので、それ以上は期待せず無視する」という考え方

**αカット** 

- 左半分の結果からルートの MAX は「**少なくとも 5 は確保できる**」ことが分かり、下限 $α = 5$ を持つ
- 次に右側の MIN ノードを調べ始めると、その最初の子（MAX）が **2** に確定
- MIN ノードは最小を取るので、この時点でその値は **2 以下**が確定
- しかしルートの MAX はすでに 5 を確保しているので、2 以下の枝を選ぶことは絶対にないよって**右の MIN ノードの残り（「?」2つ）はまるごと調べなくてよい**
- これが $\alpha$ カット

### 枝刈りは「子の並び順」に依存する

- 重要な注意点として、アルファベータ枝刈りの効率は**子ノードを評価する順番に依存**
- 良い手（評価値の高い枝）を先に調べれば早く $\alpha$ が大きくなり、たくさん刈れる
- 逆に、上の例で葉の並びを反転させると、同じ木でも**β カットができなくなる**ことがある
- 実装上は「有望そうな手を先に並べる」工夫（ムーブオーダリング）が効率を大きく左右する

### マルチエージェント経路探索

- 複数のロボットが環境内で**衝突せずに**それぞれの目的地へ動く問題を考える
- これまでの A* 探索はエージェントが1つだけの前提だったが、今度は複数いる
- ただしゲーム木と違い、**他のロボットは敵ではなく、わざと邪魔をしてくるわけではない**点がポイント（協調的な状況）
#### Local Repair A*（LRA*）

- **Step 1**: 環境内で**隣にいるエージェントだけ**を障害物とみなし、目的地までの経路を A* で作る。それ以外のエージェントは無視する。作れなければ少し待ってから Step 1 を繰り返す。
- **Step 2**: 作った経路で進む。目的地に着いたら終了。移動中に他のエージェントが進行方向の隣にいてぶつかりそうになったら止まる。
- **Step 3**: 止まったところで（少し待つこともある）、Step 1 に戻って経路を作り直す。

- 平面のような広い空間を**少数のエージェントで動かすときは効率的**
- 一方、エージェント数が多く混雑する領域では、他のエージェントに囲まれてなかなか脱出できない
- 経路が限られる迷路のような環境では**デッドロック**に陥ることもある
- 広いグリッドなら迂回路が多いので問題になりにくいが、**迷路だと迂回路がない／大きく遠回りになり、環境の隅で混雑して動けなくなる**
#### Cooperative A*（CoopA*）

- すでに複数のエージェントの経路が作成済みで、それらと衝突せずに目的地へ至る最短経路を新たに生成する」という状況

まず環境をグラフ G = (V, E) と考える。V はノードの集合、E はノードを結ぶエッジ（路）である。グリッドも迷路もグラフで容易に表現できる。ここで**衝突**とは、「同じノードに複数のエージェントがいる」または「同じエッジで交差する（すれ違う）」ことを指し、これを防ぐ。

鍵となる工夫が **時間発展した拡張グラフ（time-expanded graph）** である。

- ノードを**時刻ごとにコピー**し（t = 0, 1, 2, …）、各ノードを「隣接ノード」および「次の時刻の自分自身」とつなぐ（＝その場で待つ動作も表現できる）。
- 探索は**必ず1ステップずつ時間軸方向に進む**制約付きで行う。
- すでにある他エージェントの経路を、このグラフ上で**禁止領域**として記入する。さらに、時刻 t−1 と t の間でエッジ i→j が使われているなら、**同じ時刻間の逆向きエッジ j→i も禁止**にする（これで正面衝突＝すれ違いを防ぐ）。

アルゴリズムは次の通りである。

- **Step 1**: G の時間発展グラフを作り、既存経路のノードと（逆向きを含む）エッジを禁止領域とする。
- **Step 2**: 新しいエージェントの開始点を t = 0 に置き、A* 探索で目的地まで移動する。移動方向は矢印の向きのみ、禁止領域のノードは障害物、エッジは無いものとして扱う。ヒューリスティック関数は直線距離などとしておく。

**性質**: 最短経路は見つかるが、時間発展グラフは大きくなり、探索時間が増えることもある（ただし現在の計算機なら実用範囲内で経路生成は可能）。

### ゲーム理論入門

- 2つのエージェントが独立にある戦略をとり、**相手の戦略が自分の結果にも影響する**という、より一般的な相互作用を数学的に扱う

#### なぜ「戦略」を考えるのか

- 単独なら「何を目指すか」は明確だが、相手がいると話が変わる
- 各行動をうまく選ぶこと以上に、**そもそも何を目指して行動すべきか**という判断が根本的な課題
- しかも実社会の関係は敵対だけではない
	- エージェント間の **win-win（協力・協調）** の関係
	- 双方が最善手を選ぶと対立するが、どちらかが2番目に良い手を取れば両者がそこそこ利益を得られる、という**調整**の関係
### 利得行列

- プレイヤー $P_1, P_2$ がそれぞれ戦略$S_1, S_2$ を持つとき、戦略の組合せによって各プレイヤーが得る利得を並べた行列を **利得行列** と呼ぶ
- 各プレイヤーは**一度だけ同時に**戦略を選び、対応する利得を得る
- 利得は大きいほど嬉しく、収入のような直接的利益に限らず「こうなってほしい」という好ましさを表す広い概念
- 各プレイヤーの利得成分を取り出した行列を$M_1,M_2$とするとき、$M_1=M_2^T$が成り立つゲームを **対称ゲーム**、それ以外を **非対称ゲーム** と呼ぶ
- 講義の **新聞社の例**（対称ゲームの例）では、P₁ から見ると相手が S₁ でも S₂ でも、自分は S₁ の方が利得が高くなる結果として両者とも S₁ を選ぶ
- これは実はミニマックス法と同じ考え方
- なお利得行列の数値そのものより、**値の相対的な大小関係が重要**である点に注意

### 合理性・支配戦略・支配戦略均衡

- **合理性** … 各プレイヤーは自分の利得を最大にするよう努力する
- **支配戦略** … 相手がどんな戦略をとっても、他の戦略より利得が高くなる戦略。常に利得が大きいなら**強支配戦略**、等号を含む場合は**弱支配戦略**と呼ぶ
- **支配戦略均衡** … 両者が支配戦略を持ち、それを取り続ける組合せ。

### 反復支配戦略

- 両者が支配戦略を持つとは限らない
- 以下の例を考える

> 大きい豚BPと小さい豚SPが檻に入っている。どちらかの豚がボタンを押すと、少し離れたところにある餌箱に餌が出てくる。小さい豚がボタンを押すと、餌箱に着く前に大きな豚がほとんどを食べてしまう。大きな豚が押すと小さな豚も半分食べることが出来る。ただ、ボタンを押し餌箱に戻るのに2のエネルギーを使う

- この例では、

- 大きい豚には支配戦略が**ない**
- 小さい豚には「待つ」が支配戦略となる（自分が押しに行っても、餌箱に着く前に大きい豚に食べられてしまうため）。

- その結果、小豚は待ち続け、大豚は仕方なく押しに行く
- このように、**一方に支配戦略があり、その結果として他方の戦略も決まる**場合を **反復支配戦略**、その組合せを **反復支配戦略均衡** と呼ぶ
- 相手の合理性を仮定している点に注意が必要

### ゼロサムゲームとミニマックス戦略・均衡点

- 戦略は2種類とは限らないので、一般には m×n の利得行列になる
- 一方の利得が他方の損失そのもの（aᵢⱼ = −bᵢⱼ）になるゲームを **ゼロサムゲーム** と呼ぶ

（反復）支配戦略均衡が存在しないとき、ゼロサムゲームでは「**最悪の状態を避ける**」という戦略が考えられる。すなわち、相手が自分にとって最悪（最小値）の手を打つと想定し、**その最小値が最大になる**ような戦略を選ぶ。これを **ミニマックス戦略** と呼ぶ。

- 一般に $\mathrm{maximin} \leq \mathrm{minimax}$ が成り立つ。
- 両者が一致する戦略の組を、ゼロサムゲームの**均衡点（equilibrium point）**と呼ぶ。
- 均衡点では、一方が単独で戦略を変更すると損をするため、戦略が安定する。
- したがって、均衡点は**安定点**である。
- 作為的な操作を行うインセンティブが生じにくく、信頼性やセキュリティの面でも重要な性質である。

### 囚人のジレンマ

- **安定点が常に望ましいとは限らない**これを示すのが **囚人のジレンマ** 
- 検事が共犯と思われる囚人A・Bに司法取引を持ちかる（2人は別室で相談できない）。
	- 2人とも黙秘 → 2人とも懲役2年
	- 1人だけ自白（共犯を認める）→ 自白した側は釈放（0年）、黙秘した側は懲役10年
	- 2人とも自白 → 2人とも懲役5年
- ここで黙秘 = **協調戦略（C, cooperate）**、自白 = **裏切り戦略（D, defect）** に対応する
- 各自が合理的に「相手がどちらでも自白した方が得」と考えるため、均衡点は (自白, 自白) = (D, D) になる
- しかしこれは2人とも懲役5年で、本当は (黙秘, 黙秘) = (C, C) の懲役2年の方が望ましい 
- **個々の合理性が社会的に望ましくない結果を生む**

### 繰り返し囚人のジレンマと協調の創発

- 一回限りなら裏切りが最適だが、**同じ相手と何度も繰り返す**ことが分かっていれば、協調が生まれる可能性がある
- 協調を生みやすい代表的アルゴリズムが **しっぺ返し戦略** 
- 協調する相手とは協調し、非協調的な相手との協調は無駄と考える（相手より利得が多いとよいのではなく、**自分の利得を上げること**が目的）

### 合理性と均衡点をめぐる考察

**合理性について**

- 合理的戦略は各自の評価尺度を最大化する行為であり、利己的とは限らない
- 「人を助けるのが最善」ならそういう利得行列を与えればよい
- 利他的状況を利得とみなす能力を人間はどう獲得するのかは研究テーマ
- 「社会性」という知的行動の発現に必要

**均衡点**

- 安定点であり信頼性・セキュリティ上重要だが、望ましい行為とは限らない（ジレンマ）

**エージェントと人間**

- 人間も広くはエージェントとしてモデル化できる
- プログラムが合理性のみを追求したとき、目の前の人間が協調相手として相応しいか判断する時が来るかもしれない
- 一方で人間は常に合理的とは限らない
- **では、なぜ人は合理的でないことがあるのか、真の合理性とは何か**

## 機械学習の分類と決定木

### 機械学習の3分類

- 教師あり学習
	- 訓練データに対する正解が与えられる
	- データと正解/不正解の関係を学習する
	- 決定木はここに含まれる

- 強化学習
	- 出力に対して報酬が与えられ、これを大きくする方向に学習
	- 教師信号と違い、報酬は「正解/不正解」を確定できない点が重要
	- 「今より大きな報酬がまだあるかもしれない」ため、得た報酬が最大とは判断できない
	- 報酬は**行動の最後**にまとめて得られ、一手一手の正解情報はない

- 教師なし学習
	- 教師信号も報酬も存在しない
	- データだけが与えられ、クラスタリング・次元圧縮・概念学習が主目的

### 学習システムを設計するときの3つの基準

- 強化対象
	- 知識量の増加・応答速度の向上・知識記述のコンパクトさ・正答率の向上、のどれを伸ばすか
	- 余談として「キャッシュは学習か?」が触れられている。広義には学習と言えるが、本講義では**整理した知識として蓄え、推論を短縮・効率化すること**を学習と呼び、単なる一時記憶（キャッシュ）は学習に含めない、と区別している
- 外界からの情報
	- 教師あり学習でも、教師主導型・学習者主導型・インタラクティブ型など与えられ方に種類がある
- 事前知識との関係
	- **知識調節**：既存の知識だけでは説明できない部分を補う形で学習する
	- **知識同化**：新しく学んだことを、既存知識と**矛盾しない形**に調整して保存すること

### 学習のアプローチ

- **記号主義（シンボリズム）**：述語論理・推論規則など記号に基づく
- **統計的学習**：大量データから分類・傾向・相関を抽出（回帰分析、推定、仮説検定、クラスタリングなど）決定木などがこれに該当
- **コネクショニズム**：神経回路網（ニューラルネット）。ニューロンの結合とその強さを変えて学習する。深層学習がここに属し、通常は大量の教師データが必須

### 決定木とは

- 概念を区別するための分類木のこと
- 分類に有効な概念を見つけ出す作業でもあり、概念学習の一部

### 「よい決定木」とは

- 判断基準は **オッカムの剃刀**
- 同じ分類精度なら、**より単純（＝深さが浅い）な木のほうがよい**
- 各分割でなるべく**均一な部分集合**に分けられる属性を選べばよい

- 正5・負5の10事例を考える
- **左の分割**：「正2・負0」と「正3・負5」に分かれる → 片方が正だけになり、部分的にきれいに分類できている
- **右の分割**：「正2・負2」と「正3・負3」に分かれる → どちらも正負が混ざったまま。ばらつきが変わらず**情報が得られていない**
- この「ばらつきの減り具合」を定量化する道具が **エントロピー** 

### エントロピーの定義と意味

- 観測データの集合を $S$、要素数を $n$、ある情報が $m$ 種類の値をとり各値のデータ数を $C_i$ とする
- 出現確率 $p_i = C_i/n$ を使って、エントロピーは次のように定義される

$$E(S) = \sum_{i=1}^{m} \bigl(-p_i \times \log_2 p_i\bigr)$$

- $E(S) = 1$（2分類の場合の最大値）：分類されておらずバラバラの状態
- $E(S) = 0$：きれいに分類されてばらつきがない状態

> **エントロピーを最も大きく下げる情報（属性）に基づいて要素を分類すれば、決定木を最も単純にできる。このアルゴリズムが ID3 である。**

- エントロピーの**差分を「情報量」と呼び、これが大きい属性ほどよい分割になる

### 情報量の計算例

#### 例1：先ほどの正5・負5の分割

- **分類前**：$E(S) = -\frac{5}{10}\log_2\frac{5}{10} - \frac{5}{10}\log_2\frac{5}{10} = 1$

- **左の分割**（正2負0 が2個、正3負5 が8個）
- $E(S_1) = 0$（正だけなのでばらつきゼロ）
- $E(S_2) = -\frac{3}{8}\log_2\frac{3}{8} - \frac{5}{8}\log_2\frac{5}{8} = 0.9544$
- 部分集合の大きさで重み付け：$\frac{2}{10}\times 0 + \frac{8}{10}\times 0.9544 = 0.7635$
- **情報量 = 1 − 0.7635 = 0.2365**

- **右の分割**（正2負2、正3負3）
- どちらも $E = 1$ → 重み付けしても 1
- **情報量 = 1 − 1 = 0**（何も得られていない）

- 数値が「左の分割のほうがよい」という直感を裏付けている
- ここで重要なのは、各部分集合のエントロピーを**そのまま足すのではなく、部分集合の要素数の割合で重み付けして足す**点
#### 例2：阪神ファンを判定する決定木（11人のデータ）

- 属性は「性別・出身・学部・趣味」
- 出身地で分けた場合を計算する

- **分類前**：ファン8人・非ファン3人なので $E(S) = -\frac{8}{11}\log_2\frac{8}{11} - \frac{3}{11}\log_2\frac{3}{11} = 0.8453$

- **出身地で分割**
- 関東 {A,B,G,I}：ファン2・非2 → $E = 1.0$
- 中部 {D,E,H,K}：ファン3・非1 → $E = 0.8113$
- 関西 {C,F,J}：全員ファン → $E = 0$
- 重み付け：$\frac{4}{11}(1.0) + \frac{4}{11}(0.8113) + \frac{3}{11}(0) = 0.6037$
- **情報量 = 0.8453 − 0.6037 = 0.2416**

### ID3アルゴリズム

- 観測データ集合 $S$、属性集合 $A$ を入力とする再帰アルゴリズム ID3$(S, A)$ である。
- **Step 1**：ルートノードを作る
- **Step 2**：$S$ の全要素が同じカテゴリなら（分類完了済みなので）そのまま終了
- **Step 3**：$A$ の各属性について情報量を計算し、**最も情報量の高い属性 $c$** を選んでルートノードのテストとする
- **Step 4**：$c$ の値で $S$ を互いに素な部分集合 $S_i$ に分割し、各 $S_i$ について ID3$(S_i, A - \{c\})$ を**再帰的に**実行する。属性集合から使った $c$ を除く点に注意

つまり「**最も情報量の高い属性で分け → 各枝で残り属性に対して同じことを繰り返す**」という貪欲法である。

### ID3の発展形

- **C4.5**：ID3を拡張。たとえば**連続量データ**を扱えるようにした
- **C5.0 / See5**：C4.5を高速化し、初期に商用化したパッケージ。R や SPSS など統計処理パッケージでも利用可能
- ほかに J4.8 など

### CART とジニ係数

- ID3系がエントロピーを使うのに対し、**CART** は**ジニ係数**を使用する

$$G(S) = 1 - \sum_{i=1}^{m} p_i^2$$

- ジニ係数はデータの**不純度**を表す指標
- CARTは不純度を下げる方向に分類を進め、**必ず2分木**を生成する
#### 計算例

> 文系100人(A)・理系100人(B)を英語70点以上/未満で分割
> ① 70点以上（A60・B30の計90人）
> ② 70点未満（A40・B70の計110人）

① 70点以上（A60・B30、計90人）：$G = 1 - (\frac{60}{90})^2 - (\frac{30}{90})^2 = 0.4444$
② 70点未満（A40・B70、計110人）：$G = 1 - (\frac{40}{110})^2 - (\frac{70}{110})^2 = 0.4628$


- 分割前：$G = 1 - (\frac{100}{200})^2 - (\frac{100}{200})^2 = 0.5$
- **減少量** $= 0.5 - \frac{90}{200}(0.4444) - \frac{110}{200}(0.4628) = 0.04548$

## 強化学習

### 多段決定問題という見方

- ある大きな問題 $s$ を、部分問題 $s_1, s_2, \dots, s_m$ に分けて解くことを考える
- 組み合わせの良さを評価する関数を $J$ とすると、$J(s_1,\dots,s_m)$ を最大化する組み合わせを探すのが目標
- 全探索すると、各部分問題の解が平均 $N$ 個あるとき、組み合わせは $N^m$ 通りもあり、**指数関数的に増えてしまう**
- ここで評価関数が次のように「**隣り合う2状態だけの評価値 $h$ の足し算**」で書ける特別な場合を考える

$$J(s_1,\dots,s_m) = \sum_{i} h(s_i, s_{i+1})$$

- このとき、$s_1$ から順に解き、それまでの結果を使い回せば、計算量は $N^m$ から $N^2 m$ 程度まで激減する
- これが動的計画法が効く本質

### 状態遷移としての理解

- 部分問題 $s_1 \to s_2 \to \dots \to s_m$ を「時間とともに移り変わる**状態**」とみなすと、これは状態遷移の列になる
- つまり「初めから部分問題が並んでいる」のではなく、「**解の状態へ向かって遷移を探索しながら到達していく問題**」と捉え直せる

### 評価値・コスト・報酬の関係

- $h$ を**コスト**とみなせば、$J$ を**最小化**する問題
- 評価値とコストが混在するなら、コストを負の数にして全体を**最大化**する問題に統一できる
- 評価値の代わりに、途中や最後に得られる**報酬**を使うこともある
- このとき「累積報酬が大きいほど良い」と考える

### 動的計画法のアルゴリズム


1. **前向き（i = 1 → m）**：各状態 $s_i$ について、そこに至る最適な累積評価値 $F_i(s_i)$ を計算する。

$$F_i(s_i) = \max_{s_{i-1}} \left[ F_{i-1}(s_{i-1}) + h(s_{i-1}, s_i) \right]$$

このとき、最大値を与えた1つ前の状態 $\hat{s}_{i-1}(s_i)$ を記録しておく。

2. **後ろ向き（i = m−1 → 0）**：ゴールから記録をたどって、最適な経路 $(s_0^*, s_1^*, \dots, s_m^*)$ を復元する。

具体例が**最短経路問題**である。スタート $S$ からゴール $G$ への最短経路上のノード $n$ について「部分経路 $S\to n$ と $n\to G$ も、それぞれ最短経路になっている」という性質（最適性の原理）が成り立つので、ダイクストラ法やベルマン・フォード法、ナップザック問題、編集距離などに応用できる。これらは各自で確認するよう指示されている。

### 強化学習の定義

- エージェントが自ら行動し、その結果に応じて報酬を得て（認知して）、最適な行為を獲得（強化）する学習
- エージェントには「どう行動すべきか」という知識は**与えられていない**
- ただし**環境（状態）は認知できる**し、**報酬も認知できる**
- 報酬は行為の直後に得られることもあれば、チェスのように**最後にまとめて**得られることもある
- 学習する対象は、(a)**状態の価値** $V(s)$ か、(b)**各状態での行為の価値** $Q(s,a)$ のどちらか
- ここでの「**価値**」とは、目先の報酬ではなく、**将来を含めて得られるであろう報酬の累積（の期待値）**のこと

### マルコフ決定過程（MDP）

- 状態の集合 $S = \{s_1, \dots, s_n\}$
- 行為の集合 $A = \{a_1, \dots, a_m\}$
- 状態 $s(t)$ で行為 $a(t)$ を取ると、確率的に次状態 $s(t+1)$ へ遷移する。その確率が**状態遷移確率** $Pr(s(t), a(t), s(t+1))$。
- そのとき**報酬** $r(s(t), a(t))$ が得られる。

最重要の性質が**マルコフ性**である。

> 次状態への遷移と報酬が、**今の状態と今の行為だけ**に依存し、それ以前の履歴とは無関係である。

「過去をすべて忘れて、現在の状態さえ分かれば未来が決まる」という性質で、これがあるおかげで「現在の状態に価値を割り当てる」というアプローチが成立する。なお、遷移確率が常に1か0（次状態が一意に決まる）の場合は**決定的（deterministic）なMDP**と呼ぶ。

### 価値関数と政策関数

- **価値関数** $V(s)$：状態 $s$ の価値（その先の累積報酬の期待値）。
- **Q値** $Q(s,a)$：状態 $s$ で行為 $a$ を取ることの価値。
- **政策関数（policy）** $\pi: S \to A$：各状態でどの行為を選ぶかを決める関数。

強化学習とは、突き詰めれば「**高い累積報酬を生む政策関数 $\pi$ を学習すること**」である。

### N本腕バンディット問題

- N台のスロットマシン、ここでは N=10を考える
- 各マシンには報酬の期待値が秘密裏に設定されていて（実験では平均0・分散1の正規分布から決める）
- 引くとその期待値に近い報酬が出る
- エージェントの目標は「**どのマシンが当たりやすいかを見つけ、報酬を最大化する**」こと
- $t$ 回試行して $n$ 番目の腕を $k$ 回選び、報酬 $r_1, \dots, r_k$ を得たなら、その価値を

$$V(s_n) \approx \frac{r_1 + r_2 + \dots + r_k}{k}$$

と推定する（**標本平均法**）
### 探索と活用のジレンマ

$t$ 回学習したあと、$t+1$ 回目にどのマシンを選ぶべきか。ここに根本的な葛藤がある

- **活用**：今までの学習を信じて、最も良さそうな手を選ぶ。
- **探索**：もっと良い手があるかもしれないので、あえて別の手を試す。

このジレンマを回避するために、2つの戦略がある

- **Greedy戦略**：常に、今わかっている中で最善の手だけを選ぶ
- **ε-greedy戦略**：基本はgreedyだが、**確率 $\varepsilon$ でランダムに別の手を試す**（通常 $\varepsilon = 0.01\sim0.1$ 程度）
	- $\varepsilon = 0.1$（大きめ）：最善手を早く見つけるが、**報酬は頭打ち**
	- $\varepsilon = 0.01$（小さめ）：最善手を見つけるのは**遅い**が、**最終的な報酬は大きい**

- 実用的な対応策は「**初めは $\varepsilon$ を大きく、徐々に小さくする**」こと
- ただし $\varepsilon$ を小さくしすぎると、**環境が変化したときに古い学習結果を更新できない**という別の問題が生じる

### 学習環境の分類

- 環境にアクセス可能か
	- 状態を完全に観測できれば話は単純
	- 一部しか見えない場合は実際の状態を推測する必要がある
- 行為が環境を変えられるか
	- **受動的**：環境が確率的に変わり、エージェントはそれを観測して状態の価値を学ぶだけ
	- **能動的**：エージェント自身が行為で状態を変える。価値の高い状態へ自ら向かう。**行為が状態に影響しない受動的環境では、行為の価値であるQ学習は存在しない。**
- 報酬のタイミングと意味
	- 報酬は最終状態だけか、途中でも得られるか
	- 絶対的な尺度（勝敗）か、途中の行為のヒント（好手・悪手）か
	- 途中の報酬は便利な反面、**目先の報酬に惑わされて最終結果を軽視する**危険もある
	- 負の報酬があると、「何もしない方がマシ」と学習してしまうこともある

- この分類のもと、強化学習には2つの本質的特徴がある
- **試行錯誤的探索**（正解は与えられず報酬だけ）と**遅延報酬**（行為の結果は遅れて分かる）

### TD法

#### TD法の考え方

- 状態 $s$ の価値 $V(s)$ とは「$s$ 以降に得られる報酬の総和の期待値」
- 状態 $s$ から $s'$ へ遷移して報酬 $r$ を得たとき、**未来も含めた累積**なので、以下の式が成立する
$$V(s) \approx r + V(s')$$

- **現在の $V(s)$ の値を、少しだけ「$r + V(s')$」に近づける**

$$V(s) \leftarrow V(s) + \alpha\big( r + V(s') - V(s) \big)$$

- ここで $\alpha$（$0 < \alpha \le 1$）は**学習率**で、どれだけ近づけるかの度合い
- なぜ「＝」ではなく「近づける」なのか。それは $r + V(s')$ 自体も**まだ正確な値ではない**（$V(s')$ も学習途中）から
- 少しずつ修正を繰り返すことで、真の値へ収束していく
- 括弧の中の $r + V(s') - V(s)$ を**TD誤差**と呼ぶ
- 「予測 $V(s)$」と「より新しい見積もり $r + V(s')$」のズレであり、これをゼロに近づけるのがTD法の本質

#### TD法の更新式

実際には将来の報酬を割り引いた形を使う（後述）。

$$V[i] \leftarrow V[i] + \alpha\big( r + \gamma V[j] - V[i] \big)$$

ここで $i$ が1つ前の状態、$j$ が現在の状態である。最終状態に到達したら、その先はないので $V[j]=0$ として扱い、$V \leftarrow (1-\alpha)V + \alpha r$ と更新する。

#### 割引報酬

- 行為の列が無限に続く場合、報酬を単純に足すと発散してしまう
- そこで**割引報酬**を導入

$$V(s(t)) = \sum_{n=0}^{\infty} \gamma^n r\big(s(t+n), a(t+n)\big)$$

- ここで $\gamma$（$0 \le \gamma < 1$）が**割引率**

1. **将来の報酬は割り引いて評価する**（早く大きな報酬を得る方が良い）
2. **数学的には、無限系列でも和が収束する**ことを保証

#### 既知環境から未知環境へ

- TD法の優れた点は、**状態遷移行列 $M$ を一切使わない**こと
- 未知環境でもほぼそのまま使える
- 状態 $S$ は実際に動いて観測したものを記憶していけばよく、初めて出会った状態をテーブルに追加して動的に増やすだけである。

能動的環境では、行為 $a$ ごとに遷移確率が変わるので、行為ごとの遷移行列 $M^a$ を観測から学習する。そして「次にどの行為を取るか」を、期待報酬が最大の行為で選ぶ。

$$a^* = \arg\max_{a}\left[ r + \sum_{k} p^a_{i,k} V(s_k) \right]$$

ただしこれだけだと新しい挑戦がなく学習が頭打ちになるので、ここで先述のジレンマ（ε-greedyなど）が効いてくる。

### Q学習

- 従来は「状態の価値 $V(s)$」を学び、価値の高い状態へ移動していた
- Q学習は「ある状態でどの行為が最も価値が高いか」を直接学習
- 状態 $s$ で行為 $a$ を取る価値が **Q値 $Q(s,a)$**

$$V(s) = \max_{a \in A} Q(s, a)$$

- 利点は、**状態遷移モデル $M$ が不要**なこと
- 状態遷移確率の情報まで丸ごとQ値の中に吸収して学習してしまうから

#### 理想のQ値

- 正しい価値を $V^*, Q^*$ とすると、理想のQ値は以下の式で定義できる

$$Q^*(s, a) = r(s, a) + \gamma \sum_{s'} P(s, a, s') V^*(s')$$

- 学習では、現在のQ値をこの理想値に少しずつ近づける

#### アルゴリズム

1. すべての $Q(s,a)$ を初期化（0、または**大きめの値**にするのがコツ。大きめにすると初期に多様な行為を試すよう促せる）
2. 状態 $s$ で、ε-greedyなどで行為 $a$ を選び実行
3. 報酬 $r(s,a)$ を得る
4. 遷移先 $s'$ を観測し、次式でQ値を更新
5. $s \leftarrow s'$ として 2 へ戻る

**Q学習の更新式**

$$Q(s, a) \leftarrow Q(s, a) + \alpha\Big( r(s,a) + \gamma \max_{a'} Q(s', a') - Q(s, a) \Big)$$
#### なぜ max なのか

TD法では次状態の価値 $V(s')$ を使っていたが、Q学習では式(5.0)より $V(s') = \max_{a'} Q(s', a')$ と置き換わる。これが更新式の $\max$ の正体である。**「次の状態では最善の行為を取るはずだ」と仮定して更新する**わけである。この置き換えのおかげで $V(s')$ を別途求める必要がなくなり、行動選択もQ値を比較するだけで済むので、**状態遷移確率が完全に不要**になる。

#### 具体例

> ロボットが前進P・左L・右Rを選んでゴール（報酬20）を目指し、途中の宝箱を通ると報酬1がもらえる設定（$\alpha=0.1, \gamma=0.9$）。現在のQ値は表で与えられる。

スライドの計算例（$s_n$ で P を選び、$r=0$ で $s_{n+1}$ へ遷移、$s_{n+1}$ の最大Q値は $\max\{8,6,5\}=8$）：

$$Q(s_n, P) = 0.9 \times 10 + 0.1(0 + 0.9 \times 8) = 9 + 0.72 = 9.72$$

課題5-4は $Q(s_{n+1}, P)$ の更新後の値を求める問題である。同じ式に当てはめる（$s_{n+1}$ で P を選び、$r=1$ で $s_{n+2}$ へ遷移、$s_{n+2}$ の最大Q値は $\max\{6,8,3\}=8$）：

$$Q(s_{n+1}, P) = 0.9 \times 8 + 0.1(1 + 0.9 \times 8) = 7.2 + 0.1 \times 8.2 = 7.2 + 0.82 = 8.02$$

#### 収束定理

- 十分な回数行動を選び、学習率 $\alpha$ が「$\sum \alpha(t) \to \infty$ かつ $\sum \alpha(t)^2 < \infty$」を満たすとき、Q値は最適値に収束する
- 実用上多い有限長の行為列でもこの定理は成立する

### 発展的な行動選択と手法

- 学習したQ値やV値から政策 $\pi$ を定義でき、その $\pi$ に従って行動し、その経験でまた学習して $\pi$ が更新されるというループが強化学習の動作の核

### ボルツマン選択（ソフトマックス）

- ε-greedyに加わる3つ目の行動選択戦略
- 行為 $a$ を $\exp(Q(s,a)/T)$ に比例する確率で選ぶ

$$P(a_i) = \frac{\exp(Q(s, a_i)/T)}{\sum_{a} \exp(Q(s, a)/T)}$$

- $T$ は**温度**と呼ばれ、時間とともに0に近づけることが多い
- $T=0$ は $\varepsilon=0$（完全greedy）に相当
- 学習が進むと自然に最善手に集中し、無駄な試行が減って効率が上がる
- ただし $\varepsilon > 0$ の場合と違い、環境変化への追従性は失われる

## 教師無し学習

### クラスタリング

- データの集合を、その「類似性」に従ってグループ分けすること
- 分けられた一つ一つのまとまりを**クラスタ**と呼ぶ
- **正解データも報酬もない**
- **概念学習**の一種

#### 特徴ベクトルと特徴空間

- データを区別するには、その特徴を数値化する必要がある
- 特徴は多様なので、ふつう**複数の数値を並べたベクトル**として表す
- これを**特徴ベクトル**または**特徴量**と呼ぶ

### K-means法

- **K個の、距離的に近いグループに分ける**手法

#### アルゴリズム

1. $Ω$からランダムに $K$ 個の点を取り、それを中心 $c_1, \dots, c_K$ とする
2. すべての点 $d \in \Omega$ について、各中心との距離 $|d - c_i|^2$ を計算し、**一番近い中心のインデックス** $i$ をその点に与える（これを $I(d) \in \{1,\dots,K\}$ と書く）
3. 各インデックス $i$ ごとに、そこに割り当てられた点の集合の**重心**を求め、それを新しい中心 $c_i$ とする
4. 2へ戻る。**割り当て（インデックスの付け替え）が変化しなくなるまで**繰り返す

#### 何を最小化しているのか

- このアルゴリズムは、次の評価関数 $J$ を最小化する中心の組とグループ分けを求めている

$$J(c_1, \dots, c_K) = \sum_{i=1}^{K} \sum_{d \in \Omega_i} |d - c_{I(d)}|^2$$

- ここで $\Omega_i = \{d \in \Omega : I(d) = i\}$ は、中心 $c_i$ に割り当てられた点の集合
- つまり $J$ は「**各点とその所属クラスタ中心との距離の2乗の総和**」
- クラスタ内のばらつきの総量を表している
- これを小さくする＝各クラスタがぎゅっとまとまっている、ということ

#### 利点と欠点

**利点**
- 単純で効率がよく分かりやすいこと
- 多次元空間で外れ値（異常値）を見つけやすいこと

**欠点**
- 複数の変数の相関の影響を受けること
- 高次元では点が固まりやすく分割しにくくなること
- 距離ベースなので**超球の形状にしか対応できない**こと
- 結果が**初期値の取り方に依存し、安定しない**こと

### 階層的クラスタリング

- **個々の点から始めて、近いものを順に集めていく**ことでクラスタを作る
- 統合を繰り返すうちに徐々に大きな塊になっていくため、自然に**階層性**が生まれる
- これを**凝集型クラスタリング**、あるいは凝集型階層クラスタリングと呼ぶ

#### アルゴリズム

- 初期状態として、各点をそれ自身だけからなるクラスタとする（$c_i = \{d_i\}$）
- すべてのクラスタ対について距離 $\mathrm{dist}(c_i, c_j)$ を測り、**最も近いペアを統合**する
- クラスタ数が K（あるいは1）になるまで繰り返す

### クラスタ間の距離の定義

- ここで本質的な問いは「**集合と集合の距離**をどう定義するか」
- 点同士の距離なら明らかだが、点の集合同士の距離は決め方が複数ある

- **最短距離法**：2つの集合のうち、最も近い点同士の距離をその集合間の距離とする
- **最長距離法**：逆に、最も遠い点同士の距離を使う
- **群平均法**：2集合の点ペアすべての距離の平均を使う
- **重心法**：各集合の重心を求め、その重心同士の距離を使う
- **Ward法**：集合 C の重心 b からの各点の距離の2乗和を $L(C)$ とおき、統合距離を $\mathrm{dist}(c_i, c_j) = L(c_i \cup c_j) - L(c_i) - L(c_j)$ と定義する

- Ward法の式は「**2つを併合したときに増える内部のばらつきの量**」を距離とみなしている、と読むと分かりやすい
- なお $L(C)$ は全点でなくサンプルで近似する方法もある

### デンドログラム

- 統合の履歴をたどると、**階層的な木構造**が得られる
- これを**デンドログラム**と呼ぶ
- 横軸にデータ、縦軸に統合された高さを取った樹形図
- どの高さで切るかによってクラスタ数を後から選べるのが特徴で

### クラスタリングの応用例

- **画像の減色（色の圧縮）**
	- 画像の各画素は色（RGBを8ビット×3で24ビット）で表せ、各点が3次元ベクトル
	- ある写真にたとえば K=32 として K-means を適用し、得られた32個の重心の色で各画素を塗り直すと、**使われる色が32色に近似され、情報量が大きく下がる**
	- ただし問題点として、3次元空間での色の近さと人間の色覚は必ずしも一致しないこと、人間は顔など特定箇所に自然に注目するため感覚的に不自然に感じることがある

- **文書分類（Bag-of-Words）**
	- 単語の出現数をベクトルにする手法
	- 「全/単語/でも/よい/が/普通/は/着目/する/単語」のように分解し、着目する単語ごとに出現数を数えてベクトル化（例では単語=2回、普通=1回…で `[2,1,0,0,1,0]`）
	- 文書を数百〜数万次元の空間に配置してクラスタリングすれば、文書を分類できる
	- 発展としてN-gramやword2vecが自然言語処理の講義につながる、と案内されている

- **顧客分析**
	- 顧客の購買履歴をベクトル化してクラスタリング
	- 重みを揃えたいなら正規化（例：0〜10へスケール調整）して、購入総額のような大きな値の影響を抑えバランスを取る
	- 同じクラスタの顧客は似た傾向を持つとみなし、「購入の多い月の直前にメールを送る」「似た嗜好の他顧客の行動から購入を推薦する」といった施策につなげられる

### 主成分分析

- これまでの例は次元がとても高そうである一方、使われない変数や、分類にあまり重要でない変数もありそう
- 「**n次元のデータN個を、できるだけデータの区分が保たれるように、より少ない m次元（$m \le n$）で表現し直したい**」というのがPCAの目的（**低次元化**）

#### 基本方針

- データをある軸に射影したとき、**分散が最大になる軸**を第1主成分 $w_1$ とする（分散が大きい＝データが広く散らばる＝区別しやすい）
- 第1主成分と**直交する**軸の中で、射影したときの分散が最大の軸を第2主成分 $w_2$ とする
- 一般に、これまで選んだ主成分すべてに直交し、射影分散が最大となる軸を第k主成分とする
- これを第m主成分まで繰り返す

各軸 $w_i$ の大きさは1（単位ベクトル、$|w_i|^2 = w_i \cdot w_i = 1$）に固定する。これは一般性を失わない約束である。

#### 定式化

各データ $x_i$ をn次元ベクトルとし、N個を縦に並べた行列を $X$ とする。まず各列（各特徴）の標本平均 $\mu_j$ を引いて**平均を0に揃える（中心化する）**操作を行い、$\bar{x}_{ij} \leftarrow x_{ij} - \mu_j$ とした行列を $\hat{X}$ とする。

準備(1)で見たとおり、単位ベクトル $w_1$ への射影は内積なので、点 $\bar{x}_i$ の射影は $p_i = \bar{x}_i \cdot w_1$、まとめて $\mathbf{p} = \hat{X} w_1^{\top}$ と書ける。

中心化してあるので射影 $p_i$ の平均は0であり、その不偏標本分散は

$$\sigma^2 = \frac{1}{N-1}\sum_{i=1}^{N} p_i^2 = w_1 \left(\frac{1}{N-1}\hat{X}^{\top}\hat{X}\right) w_1^{\top}$$

となる。ここで中央の行列 $\mathcal{S} = \dfrac{1}{N-1}\hat{X}^{\top}\hat{X}$ の (i,j) 成分を計算すると

$$\hat{\sigma}_{ij} = \frac{1}{N-1}\sum_k (x_{ki} - \mu_i)(x_{kj} - \mu_j)$$

すなわち**共分散行列の不偏推定**そのものになる。この行列を $\mathcal{S}$ とおく。要するに「**射影後の分散 = $w_1 \mathcal{S} w_1^{\top}$**」という、共分散行列を挟んだ二次形式として表せた、というのがここでの要点である。

#### なぜ固有値問題に帰着するのか

- 求めたいのは$|w_1| = 1$ という制約のもとで、$\sigma^2 = w_1 \mathcal{S} w_1^{\top}$ を最大化する $w_1$

$$L(w_1, \lambda) = w_1 \mathcal{S} w_1^{\top} - \lambda(w_1 \cdot w_1^{\top} - 1)$$

を $w_1$ で偏微分して0とおくと

$$\frac{\partial L}{\partial w_1} = 2(\mathcal{S} - \lambda) w_1^{\top} = 0 \quad\Longrightarrow\quad \mathcal{S} w_1^{\top} = \lambda w_1^{\top}$$

が得られる
- これは「$\mathcal{S}$ の固有ベクトルを求めよ」という固有値問題
- つまり、PCAの「分散を最大化する軸を探す」という問題が、**共分散行列の固有値問題**へときれいに帰着する
- 固有値は $\det(\mathcal{S} - \lambda) = 0$ を解いて求めるが、一般に複数ある。そこで $\mathcal{S}$ の固有値を大きい順に $\lambda_1 \ge \lambda_2 \ge \cdots$、対応する固有ベクトルを $\omega_1, \omega_2, \dots$ とする。すると射影後の分散はちょうど

$$\sigma^2 = w_1 \mathcal{S} w_1^{\top} = \lambda$$

となるので、**分散を最大にするには最大固有値 $\lambda_1$ の固有ベクトル $\omega_1$ を選べばよい**ことが分かる。結論として、**第1主成分は最大固有値の固有ベクトル $\omega_1$、第2主成分は次に大きい固有値の固有ベクトル $\omega_2$**、と順に対応する。

意味を言葉で整理すると、$\omega_1$ に射影したとき分散が最大になりデータを最も広く区分できる。一方、それと直交する方向の情報は失われるが、その「失われた中で最も広く区分できる軸」が $\omega_2$、というように続いていく。

#### 利点・欠点と応用

**利点**
- 相関のある変数を取り除けること
- 次元を下げられること（単純化と高速化の利益が大きい）

**欠点**
- **線形性に強く依存**しているため、線形では捉えられない構造には適用できないこと

**応用例**
- 画像圧縮
	- 1024×640の画像を64×64ブロックに分割する。
	- 各ブロックを $16\times10\times3=480$ 次元の点として表現すると、4,096個の点が得られる。
	- 480次元データをPCAで $m$ 次元に圧縮する。
	- 圧縮した $\mathbf{y}_i$ から $y_{i1}w_1 + \cdots + y_{im}w_m$ として元の次元へ戻せる。
	- 完全には復元できず、多くの色を使う領域では大きく劣化する場合がある。

## PAC-Learning, SVM

### 帰納推論と「極限による同定」

- 「いくつかの具体例から一般法則を導く」こと

問題設定はこうである。中身の見えないプログラム関数 $F(x)$（出力は $+1, -1$ のみ）があり、入出力のペア $(x_1, F(x_1)), (x_2, F(x_2)), \dots$ をいくつか観測して、**元のプログラムの正体を当てられるか?** $y=1$ のデータを正例、$y=-1$ を負例と呼ぶ（教師あり学習）。

**極限による同定（Gold 1964）**は、これを厳密に定義したものである。アルゴリズム $\mathcal{A}$ にデータを順に入れていくと仮説 $h_1, h_2, \dots$ を順に出力し、ある時点 $N$ 以降は**ずっと同じ正解 $h$ を出し続ける**——このとき「極限で同定可能（学習可能）」と言う。

この理論はもともと**言語学習の問い**から来ている。「幼児は正例（正しい言葉）だけを与えられ、負例（誤った言葉の指摘）はほとんど受けないのに、なぜ自然言語を獲得できるのか?」を理論的に証明できるか、という問いであった。答えは「正例だけでは NO、負例があればある程度 YES」。後に [Blum75] がこの研究を「プログラムによる学習可能性と計算量」に結びつけ、計算論的学習理論が発展していく。

ただし極限同定には限界があった。「**完全に正しくなる**」という条件が厳しすぎるのである。実際には訓練データがどう与えられるか分からず、データに誤差もある。「極限で正解」と言われても実用には遠い——そこで **PAC学習** が登場する。

### PAC学習

- 名前のとおり「**たぶん（Probably）・だいたい（Approximately）・正しい（Correct）**」学習
- 完璧を諦めて、現実的な合格ラインを設定
- 設定の要点は、入力 $x$ が**確率分布 $\mathcal{D}$ に従って現れる**と考えること
- $P_x$ をその確率とすると $\sum_x P_x = 1$
- 仮説の集合 $H$ から正解の判別関数 $h^*$ を探す
- 真の関数 $h^*$ と候補 $h$ の**誤差**は、両者の答えが食い違う $x$ の確率を足し合わせたものと定義

$$d_X(h^*, h) = \sum_{x:\, h^*(x)\neq h(x)} P_x$$

- **よく出るデータで間違えると大きな誤差、めったに出ないデータの間違いは小さな誤差**として扱われる

- PAC学習可能の定義は任意の分布 $\mathcal{D}$、任意の $0 \leq \delta, \varepsilon \leq 1$ に対し、多項式時間のアルゴリズム $\mathcal{L}$ が存在して

$$\Pr\big[ d_X(h^*, h) \leq \varepsilon \big] \geq 1 - \delta$$

- **$\varepsilon$（近似度）**: 誤差の許容量。めったに来ないデータは間違えても仕方ない、という妥協。
- **$\delta$（信頼度）**: 失敗してもよい確率。データが確率的に来る以上、たまたま悪いデータ列を引けば良い $h$ を出せないこともある。だから「$1-\delta$ の確率で成功すればよい」とする。

- **$\varepsilon$ で許した部分こそ本当は正解させたい → Boosting へ**
- **失敗するなら何度も試して多数決すればよい → アンサンブル学習へ**

### VC次元と「次元の呪い」

#### VC次元とは

- 「いつでも分類できるのか」に答えるための指標が **VC次元** である。直感はこうである。2次元平面で直線を使って点を分けることを考えると、**3点なら（どんなラベルの付き方でも）必ず直線1本で分けられる**が、4点になると分けられない配置が出てくる。

正確には、サンプル $S$（$|S|=N$）の $2^N$ 通りすべてのラベル分け方を、$H$ の中の判別関数で表現できるとき「$H$ は $S$ を**細分化できる（shatter）**」と言う。そして

$$\mathrm{VCdim}(H) = \big( H \text{ で細分化できるサンプルの最大サイズ} \big)$$

がVC次元である。**n次元空間の超平面の集合なら $\mathrm{VCdim}(H) = n+1$** という綺麗な結果が成り立つ（2次元なら3、というのがさっきの話）。

VC次元が決まると、PAC学習に必要な訓練データ数 $m$ の下限も与えられる（$d = \mathrm{VCdim}(H)$ を使い、$m \geq \max\big(\tfrac{4}{\delta}\log\tfrac{2}{\delta},\ \tfrac{8d}{\varepsilon}\log\tfrac{13}{\varepsilon}\big)$）。これで疑問①にも目安がつく。

#### 次元の呪いと過学習

ここで素朴な疑問が出る。「VC次元が大きい（＝高次元）ほど分類能力が高くてよいのでは?」——**答えは「そうとも言えない」**。

- 学習で求めるのは、訓練データだけでなく未知データも適切に分類する**汎化能力**である。
- 次元とパラメータ数を増やすと、訓練データに過度に適合しやすくなる。
- その結果、訓練データに含まれる誤差や雑音まで学習する場合がある。
- 訓練データでは正しくても未知データでは誤る現象を、**過学習（overfitting）**と呼ぶ。
- 高次元で生じるこのような弊害を、**次元の呪い（curse of dimensionality）**と呼ぶ。

ここから方針が決まる。「**判別はできて、かつ VC次元がなるべく小さい（＝シンプルな）空間で分けたい**」。この方針を具体的に実現するのが SVM の「マージン最大化」である。

### SVM ①：マージン最大化（ハードマージン）

- マージンを最大化することが、なぜ良いのか
- 広い余白をとる＝境界をシンプルに保つことであり、これが**VC次元を実質的に抑え、過学習を防ぐ**ことにつながるから
- 先ほどの「VC次元の小さい空間で分けたい」という方針の実装になっている

#### 最適化問題への定式化

- マージンは「全サンプル中、平面に最も近い点までの距離」なので、これを最大化したい
$$\max_{\boldsymbol{w}, b}\ \min_{\boldsymbol{x}\in S}\ \frac{|f(\boldsymbol{x})|}{|\boldsymbol{w}|}$$
- ここで、$f$ は定数倍しても同じ平面を表すので、**最も近い点で $|f(\boldsymbol{x})| = 1$ になるようスケールを固定**してしまう
- マージンは $\frac{1}{|\boldsymbol{w}|}$ になり、これを最大化＝ $|\boldsymbol{w}|$ を最小化、と言い換えられる
- したがって、以下の**2次計画問題**に帰着できる

$$\min_{\boldsymbol{w}, b}\ \frac{1}{2}|\boldsymbol{w}|^2 \quad \text{s.t.}\quad y_k f(\boldsymbol{x}_k) \geq 1 \ \ (\forall \boldsymbol{x}_k \in S)$$
### SVM ②：ソフトマージン

- 「すべての点で $y_k f(\boldsymbol{x}_k) \geq 1$」という仮定は、現実には強すぎる
- **雑音があり、きれいに分離できないデータが普通**だから
- そこで各点に**スラック変数 $\zeta_k \geq 0$**（はみ出しを許す“ゆるみ”）を導入して制約を緩める

$$y_k f(\boldsymbol{x}_k) \geq 1 - \zeta_k$$

$\zeta_k$ が大きいほどマージン内に食い込んだり、反対側にはみ出したりを許す（$\zeta_k > 1$ で実際の誤分類）。ただしはみ出しは小さく抑えたい。そこで目的関数にペナルティを足す:

$$\min_{\boldsymbol{w}, b, \boldsymbol{\zeta}}\ \frac{1}{2}|\boldsymbol{w}|^2 + C\sum_k \zeta_k \quad \text{s.t.}\quad y_k(\boldsymbol{w}\boldsymbol{x}_k^T + b) \geq 1 - \zeta_k,\ \ \zeta_k \geq 0$$

ここで **$C$ は正則化係数**で、「マージンの広さ」と「誤分類の少なさ」のバランスを取るつまみである。

- **$C$ が大きい**: 誤分類への罰が重く、許しにくい（極端に大きいと事実上ハードマージン）。
- **$C$ が小さい**: 誤分類を許容し、その代わりマージンを広くとる。

$C$ の最適値は理屈では決まらず、**交差検定（cross validation）**で実験的に選ぶ。

#### ラグランジュ・KKT・双対問題

- **ラグランジュの未定乗数法**は、制約 $g_k(\boldsymbol{x})=0$ のもとで $f$ を最適化する古典的手法
- これを**不等式制約 $h_j(\boldsymbol{x}) \geq 0$ にも拡張**したのが **Kuhn-Tucker（KKT）定理**で、最適解の必要十分条件を与える
- 特に $\xi_j h_j(\boldsymbol{w}^*) = 0$ という**相補性条件**が重要で、「制約が効いていない点では乗数がゼロになる」ことを意味する
- SVM のラグランジュ関数を $\boldsymbol{w}, b, \boldsymbol{\zeta}$ で偏微分してゼロと置くと、**最適な重み $\boldsymbol{w}$ は、サンプルベクトルの重み付き和で書ける**、という事実が導かれる

$$\boldsymbol{w} = \sum_k \xi_k y_k \boldsymbol{x}_k$$

- これを代入して整理すると、主問題が**双対問題**に変わる

$$\max_{\boldsymbol{\xi}}\ \sum_k \xi_k - \frac{1}{2}\sum_{k,j} \xi_k y_k \xi_j y_j \boldsymbol{x}_j \boldsymbol{x}_k^T \quad \text{s.t.}\quad 0 \leq \xi_k \leq C,\ \ \sum_k \xi_k y_k = 0$$

- この双対形式には、次の2つの利点がある。
  - $\xi_k$ の値からサンプルの役割を判断できる。
    - $0 \leq \xi_k < C$ なら正しく分類される。
    - $\xi_k=0$ の点は、境界を定めるサポートベクトルとなる。
    - $\xi_k=C$ の点は、マージンを越えた点である。
  - 目的関数には $\boldsymbol{x}_j \boldsymbol{x}_k^T$ という内積の形しか現れない。
- 内積だけで表現される性質が、カーネルトリックを可能にする。

#### カーネルトリック

- 線形の超平面で分けられるデータには限界がある
- そこで発想を変え、「**低次元では直線で分けられなくても、高次元に写像すれば分けられるかもしれない**」と考える
- ただし素朴にやると、高次元への写像 $\phi(\boldsymbol{x})$ の計算コストが跳ね上がる
- ここで §8 の「双対問題には内積しか出てこない」が効いてくる。写像後の内積が

$$\phi(\boldsymbol{x}_j)\,\phi(\boldsymbol{x}_k)^T = K(\boldsymbol{x}_j, \boldsymbol{x}_k)$$

という**元の空間で計算できる関数 $K$**（カーネル関数）で書けるなら、$\phi$ を陽に計算せず、$K$ だけで高次元での分類ができてしまう。これが**カーネルトリック**である。

- $K$ が満たすべき条件を、**正定値カーネル**の条件と呼ぶ。
  - 対称性：$K(x,y)=K(y,x)$
  - 任意の点から作るグラム行列が正定値であること
- 双対問題の内積 $\boldsymbol{x}_j\boldsymbol{x}_k^T$ を $K(\boldsymbol{x}_j,\boldsymbol{x}_k)$ に置き換えるだけで、非線形分類が可能になる。
- これがSVMの強力さの核心である。

**SVM のまとめ（利点・欠点）**: 利点は、分類能力が高く高速で、線形なら結果の解釈が容易、最大マージンにより過学習に強く、カーネルで非線形も扱えること。欠点は、線形分離だけでは限界があり、非線形だと解釈が難しく、データ数が多い（1万以上など）と急速に遅くなることである。

### アンサンブル学習

#### 問題

> 精度の低い弱学習器を組み合わせて、精度の高い学習器を作れるか。

#### 解答

- 弱学習器には、線形分離器や浅い決定木がよく使われ、極端には深さ1の決定木も使われる
- `Bagging`と`Boosting`の2つある
- **Bagging**は、サンプルを何組も抽出してそれぞれから弱学習器を作り、多数決や平均で統合し
	- 各学習器が独立なので**並列化が可能**
	- ランダムフォレストはこの一例
- **Boosting**は**逐次的**
	- 前の学習器が間違えた訓練データの重みを大きくして、次はそこを重点的に学習する
	- **弱点を順に補強していく**方式
	- AdaBoost や GBDT がこの系統
	- Boosting は元々「**弱PAC学習**（誤差が $1/2$ よりわずかに小さい程度の弱い保証）を、理論的に通常の PAC学習に押し上げられるか」という問いから生まれた点が重要

### AdaBoost のアルゴリズムとその意味

AdaBoostの手順は次のとおりである。データ $X$ 上の初期分布 $\mathcal{D}_1$ を等確率（$1/m$）とし、各ラウンド $t$ で:

1. 現在の分布 $\mathcal{D}_t$ に基づくサンプルから弱学習器 $h_t$ を得る。
2. 誤差を計算: $\varepsilon_t = d_X(h^*, h_t)$。
3. $h_t$ の重要度を $\displaystyle \alpha_t = \frac{1}{2}\ln\frac{1-\varepsilon_t}{\varepsilon_t}$ とする。
4. 分布を更新: $\displaystyle \mathcal{D}_{t+1}(\boldsymbol{x}_j) = \frac{\mathcal{D}_t(\boldsymbol{x}_j)\exp(-\alpha_t y_j h_t(\boldsymbol{x}_j))}{Z_t}$（$Z_t$ は合計を1にする正規化定数）。

これを $T$ 回繰り返し、最終出力を**重み付き多数決** $\displaystyle H(\boldsymbol{x}) = \mathrm{sgn}\Big(\sum_{t=1}^{T}\alpha_t h_t(\boldsymbol{x})\Big)$ とする。

**分布の更新（重要度の付け替え）**: 更新後の分布では、$h_t$ が**正解した点の合計確率と、間違えた点の合計確率がちょうど 1/2 ずつ**になるよう設計されている。つまり次のラウンドで $h_t$ を使っても**ランダム（情報ゼロ）になるようにわざと調整**し、$h_t$ が苦手だった部分に重みを移す。これにより次の学習器は前の弱点に集中せざるを得なくなる。

- **重要度 $\alpha_t$ の意味**
  - 誤差 $\varepsilon_t$ が小さい学習器ほど $\alpha_t$ が大きく、最終結果へ強く反映される。
  - $\varepsilon_t=0.5$ なら $\alpha_t=0$ となり、ランダムな学習器は寄与しない。
  - $\varepsilon_t>0.5$ なら、学習器の符号を反転した $-h_t$ を使うことで、誤差を $1-\varepsilon_t<0.5$ にできる。
  - 半分より高い精度を持つ弱学習器を積み上げる、弱PAC学習の発想が現れている。

### AdaBoost の数値例

> 7人分のデータ（年代・年収・住宅ローン有無 → 貸し倒れ Yes/No）で計算
> 貸し倒れ Yes を $+1$、No を $-1$ とし、初期重みは全員 $1/7$とする


#### ラウンド1

- 弱学習器 $h_1$ ＝「年代32以下なら $+1$、33以上なら $-1$」
- 重み付き誤差の合計は $2/7$（2人を誤分類）

$$\alpha_1 = \frac{1}{2}\ln\frac{1-2/7}{2/7} = \frac{1}{2}\ln\frac{5}{2} = 0.4581$$

誤分類した2人（重み $1/7$）の重みを $\exp(+\alpha_1)$ 倍して増やし、正解した5人は $\exp(-\alpha_1)$ 倍して減らし、正規化（$0.9038$ で割る）。結果、誤った2人の重みは $0.25$ に跳ね上がり、正解した人は $0.10$ 前後に下がる。

**ラウンド2**: 更新後の重みで弱学習器 $h_2$ ＝「年収700未満なら $-1$、700以上なら $+1$」。重み付き誤差は $0.4000$ なので

$$\alpha_2 = \frac{1}{2}\ln\frac{1-0.4}{0.4} = \frac{1}{2}\ln\frac{0.6}{0.4} = 0.2027$$

- **最終分類器**

  $$H(\boldsymbol{x}) = \mathrm{sgn}\big(0.4581\, h_1(\boldsymbol{x}) + 0.2027\, h_2(\boldsymbol{x})\big)$$

- 例えば、$x_2$ は次のように計算する。

  $$H(x_2)=\mathrm{sgn}(0.4581\cdot(-1)+0.2027\cdot1)=\mathrm{sgn}(-0.2554)=-1$$

- 2つの弱い規則の重み付き多数決で各サンプルを判定する。
- 一部は不一致であり、本来はさらにラウンドを重ねる。

この例の良いところは、**「外したデータの重みが次ラウンドで実際に増える」様子が数字で追える**ことである。$1/7 \to 0.25$ という重みの上昇が、Boosting の「弱点を強化する」という言葉の正体である。

### AdaBoost の利点・欠点と課題

**利点**
- 実装が容易で高速
- 調整パラメータが少ない（ほぼ繰り返し回数 $T$ のみ）
- 弱学習器との独立性が高い
- 汎化能力が高い
- 他手法より過学習が起きにくい

**欠点**
- 解の最適性に難
- **ノイズデータの影響を受けやすく**（外れ値の重みが上がり続けて過学習になりうる）、データが弱学習器に対して複雑すぎると判別がうまくいかない

## 知識表現

記号ベースで知識を計算機に表現し、推論・説明を可能にする枠組み（Topic 9）。講義内容の全文記録。蒸留版は [[知識表現]] を参照。


### 知識表現とは何か、なぜ必要か

- 知識表現とは、人間が言語的・記号的に保持している知識を計算機内部に表現し、それを用いた推論や学習を実現するための枠組み
- 計算機システム上に実現された知識の集まりを**知識ベース** と呼ぶ
- 知識表現が必要とされる背景には、統計的学習だけでは扱いにくい性質がいくつか存在することがある
	- 具体的には、人間は多数の例を見ずとも学習・認知を行えること
	- 専門知識のような高度な知識を獲得できること
	- そして推論や学習の根拠を説明できることが挙げられる
- さらに「時間の概念」「因果関係」「他者に関する知識」なども、統計的処理のみでは表現が困難である

- 知識表現は**主に記号ベース**であり、統計的・ニューラルネットワーク的手法と対比される
- エージェント間や人とのコミュニケーションでは明示的な内容が必要となるため、社会性を実現するには記号的表現が不可欠
- ただし両者は排他的ではなく、組み合わせが重要
- 例えばニューラルネットワークの学習において、禁止事項をルールとして加えて禁じ手を回避する、といった併用が考えられる
### 人間の記憶モデル(二重貯蔵モデル)

- 知識表現の各手法は、人間の記憶に関する心理学的モデルを土台としている
- 基礎となるのが**二重貯蔵モデル** であり、記憶を以下の二種に分ける

| 記憶の種類                        | 内容                                                             |
| ---------------------------- | -------------------------------------------------------------- |
| **短期記憶 (short-term memory)** | 五感からの情報の一時的貯蔵、および認知活動に必要な情報処理のための記憶。動作記憶(working memory)とも呼ばれる |
| **長期記憶 (long-term memory)**  | 宣言的記憶と手続的記憶に分類される                                              |

- 長期記憶はさらに次の二つに分けられる
	- **宣言的記憶**: 言語的に表現可能な知識。さらにエピソード記憶(特定の場面・出来事に結びつく)と意味記憶(「酸素はO₂」のような一般的概念知識)に分かれる。
	- **手続的記憶**: 動作や活動の認知として表現される知識(自転車に乗るなど)。料理の手順や積分の求め方などがこれにあたる。

- **プロダクションルール**は二重貯蔵モデル全体に、意味ネットワークは意味記憶に、フレームはエピソード記憶に、それぞれ基づいている

### プロダクションシステム

#### 構成

- プロダクションシステムは、Newellによって1973年に提案された
- 人間の問題解決行動の研究をもとに計算機用に実現されたもの
- 二重貯蔵モデルと直接対応する三要素から構成される
	- **プロダクションメモリ (PM)**: 長期記憶に相当する知識ベース。ルールや一般的データの集合。
	- **ワーキングメモリ (WM)**: 短期記憶に相当する作業記憶。観測事象や推論の途中結果を一時保存する。
	- **推論エンジン**: WMの情報にPMの知識を適用し、新しい結果を導く機構。

- エージェントの観点では、センサーからのデータをWMに蓄え、PMのルールを用いて推論し、エフェクターを通じて行為を選択する

#### プロダクションルール

- PMはIF-THEN形式のルールの集合として構成される
- 形式は次のとおり

```
C1, C2, …, Cm  ⟹  A1, A2, …, An
```

- C側を**条件部**または**LHS (left hand side)**
- A側を**実行部**または**RHS (right hand side)** 
- 「LHSが成立するならRHSを実行せよ(行為の実行、あるいはWMの書き換え)」と解釈される

- あいまいさを表現するため、ルールに**確信度**を付与する場合もある。

```
ライトが暗い, セルモータがよく動かない, バッテリーは新しい ⟹ 発電機が故障している [0.7]
```

#### 後ろ向き推論 

- 推論には前向きと後ろ向きの二方式がある
- まず後ろ向き推論を説明する
- これはゴールから出発し、それを達成するルールの繋がりを逆向きに探索する方式

1. ゴールをWMに与え、そのゴールを実行部に含むルールをPMから選択する
2. その条件部を参照し、WMに含まれない条件を取り出す
3. その各条件を達成するルールをさらに探す
4. これらの操作を繰り返す

- 繰り返しの末に、どのルールの実行部によっても達成できない条件に至った場合は、環境からセンサーで取得するか、人間に直接問い合わせてWMに格納する
- ゴールを与えることで推論が開始されるため**ゴール駆動推論**とも呼ばれる
- なお後ろ向き推論では実行部が実際の実行を伴わないため、実行部を**結論部**と呼ぶのが通常

##### 動物当ての例でみる後ろ向き推論

```
P1: 体毛を持つ ⟹ 哺乳動物
P2: 授乳する ⟹ 哺乳動物
P5: 哺乳動物 ∧ 肉食 ⟹ 肉食動物
P6: 哺乳動物 ∧ 鋭い歯 ∧ 鋭い爪 ⟹ 肉食動物
P8: 肉食動物 ∧ 黄褐色 ∧ 黒い縞 ⟹ トラ
```

###### 問題

> 体毛を持ち、鋭い牙と爪を持ち、黄褐色で黒い縞がある動物はトラか。

###### 解答

1. 下線部の事実がWMに追加され、ゴールは「トラであること」となる。
2. 結論部にトラを持つのはP8なので、その条件部を示せばよい。
3. WMより黄褐色・黒い縞は既知。残りは「肉食動物であること」を示す。
4. 結論部に肉食動物を含むのはP5とP6。
5. P5の条件部「肉食」はWMにないため不明。哺乳動物はP1で分かるが、肉食かどうかが分からない。
6. そこでP6を考える。鋭い歯・爪はWMにあり、哺乳動物はP1で分かる。よって証明された。

- なお項番5で肉食か不明な場合、人間に「その動物は肉食であるか?」と問い合わせる方法もある
- 回答が「はい」ならP5成立、「いいえ」なら不成立、「分からない」なら一応P5成立とするが他のルールも試み、それでも証明できなければ条件付き(肉食であれば)でトラと提示して終了する
- このような問い合わせ中心のシステムは、動物当てでは不要かもしれないが、**故障診断**(自動車など、すべての状況を初めに入力できない場面)では有益

##### 後ろ向き推論の特徴

- ルールを逆向きにたどるだけなので効率がよく実装も容易である
- 基本的に環境への行為は問い合わせか観測のみで、環境に影響を与える制御(ロボットの行動など)は行えない
- このため、環境に影響を与えない**診断**に用いられる

#### 前向き推論

- 前向き推論はゴールに向かってWMの内容から新しい事実を導く方式であり、次の三段階を一サイクルとする。

- **照合 (matching)**: WMを参照し、条件部が成立するルールをPMから探す。このルール集合を**競合集合 (CS)** という。
- **競合解消 (conflict resolution)**: CSが空なら終了。複数あれば、何らかの**競合解消戦略**で一つだけ選ぶ。
- **実行 (action)**: 選択したルールの実行部を実行し、WMを書き換える。

この三段階を各一回行うことを**プロダクションサイクル (PC)** といい、ゴール状態になるまで繰り返す。

**動物当ての例**(前向き)を追う。問い「体毛があり、鋭い歯と爪を持ち、黄褐色で黒い斑点を持つ動物は何か?」、WM = {体毛, 鋭い歯, 鋭い爪, 黄褐色, 黒い斑点} とする。

```
PC1: CS={P1} → 実行 → WM = WM ∪ {哺乳動物}
PC2: CS={P1, P6} → 戦略によりP6選択 → WM = WM ∪ {肉食動物}
PC3: CS={P1, P6, P9} → 戦略によりP9選択 → WM = WM ∪ {チータ}
```

よってこの動物はチータと結論される(P9: 肉食動物∧黄褐色∧黒い斑点 ⟹ チータ)。

- PCの「実行」はWMの内容を変更するだけであり、必ずしも環境への行為を意味しない。
- 実世界で行為するタイミングは、推論とは独立した問題である。
- ロボットでは、WMに「右足を出す」と記録しても、数ステップ先まで推論してから実際に行為してよい。
- 前向き推論は、適切な場所で推論を中断し、実際の行為を発行できる。
- 一方、不適切なルールを選んでも**バックトラック（推論の取り消し）は行わない**。
- したがって、競合解消戦略の選択が重要となる。

#### 競合解消戦略

CSから適切なルールを選ぶことが重要であり、適切な適用順序が結論を早く導く。戦略の例として以下がある。

- ルールに重要度を予め与え、重要度の大きいものを優先する。
- 条件部の多いもの(より詳細と考えられる)を優先する。
- 最も最近使われていないものを優先する。
- 同じルールは続けて使用しない。
- 最近WMに加えられた事実を使用する条件を持つものを優先する。
- 実行してWMに新しい事実が加わるものを優先する。

通常はこれらを組み合わせて用いる。有名な例が**LEX戦略 (Lexicographic sort)** であり、以下を(1)から順に適用し、ルールが一つになった時点で処理を終える。(1)適用後にCSが空になれば推論は終了する。

```
(1) すでに実行したルールをCSから除去する。
(2) WMの最新のデータを参照するルールをCSから選択する。
(3) 条件部の個数が多いものをCSから選択する。
(4) 任意のルールをCSから選択する。
```

これは一例であり、戦略は問題領域やルールの作り方に依存する。

#### Reteアルゴリズム

PCごとに照合が行われるため、WMを参照して条件に合うルールを効率的に選択する必要がある。この高速化を担うのが**リートアルゴリズム (Rete algorithm)** である。「Rete」は(体内の)網を意味する。

基本的な発想は、ルールを適用しても実行部によってWMは少ししか変わらない、という仮定にある。PMをコンパイルして**リートと呼ばれるネットワーク**を生成し、ルール適用のたびにWMの変化に合わせてリートをわずかに書き換える。網を参照すれば条件部を満足するルールが容易に分かる。

具体例で動作を見る。

```
PM: R1: A(x)∧B(x)∧C(y) ⟹ add D(x)
    R2: A(x)∧B(y)∧D(x) ⟹ add E(x)
    R3: A(x)∧B(x)∧E(y) ⟹ delete A(x)
WM: {A(1), A(2), B(2), B(3), B(4), C(5)}
```

多く共通に含まれる条件(この場合A(x))から順にネットワークをコンパイルする。

1. add Dに達するルート①(R1相当)があるのでR1を適用し、D(2)が新規に加わる。
2. add Eに達するルート②(R2相当)ができ、E(2)が新規に加わる。
3. delete Aに達するルート③(R3相当)ができ、A(2)が消える(この場合はネットワークの変更が多くなる)。

Reteの性質と問題点としては、実行部による一度の変更量が少ないことを前提とするため、多数の実行部を持つルールが多いと効率が下がる。また多くのルールの条件部にある事実に変更があると、修正がネットワーク全体に広がり時間がかかる。必要メモリ量も大きめである。しかしWMを毎回参照するより効率的で、現在でも有効な手段として使われている。なお原典はCharles Forgy (1982) である。

- Reteを用いた代表的なプロダクションシステムが、**OPS5（Official Production System 5）**である。
- Charles Forgyがカーネギーメロン大学で考案した。
- 改良版のRete IIを実装したJava版としてOPSJがある。
- OPS5を利用したシステムに、後にXCONと改名された**R1**がある。
  - DEC社の大型ミニコンピュータVAXについて、構成提案と発注助言を行った。
  - 初の商用エキスパートシステムである。
  - 2,500個のルールを持ち、8万件以上の適用事例があった。

#### 説明機能

推論に利用したルールを提示することで、結論の根拠を説明できる。前述のチータの例では、結論はP9を用いて得られた。「なぜチータか」と問われればP9を提示し、「なぜ肉食動物と分かるか」と問われればP6を提示する、という形で根拠を遡って示せる。複雑な問題解決の根拠を示すという意味で、説明機能は人工知能アルゴリズムとして重要な機能である。

#### 長所と短所

**長所**としては、IF-THEN形式が意味的に分かりやすいこと、ルールが独立しており追加・変更が独立に行えること、前向き・後ろ向き両方の推論が可能なこと、説明機能を持つことが挙げられる。

**短所**としては、独立性の反面として知識の相互関係が分かりにくいこと(自動的にルールが選択されるため、関連する知識と無関係に実行される可能性がある)、そして系統だった知識を記述できないこと(基本的に平坦的な記述となり、後述の階層的表現を含まない)が挙げられる。

### 意味ネットワーク

#### 基本概念

- 意味ネットワークは、意味記憶とりわけ**連想記憶**という心理学的モデルをベースに、Quillianによって1966年頃に提案された
- 連想記憶とは、ある概念や対象を考えると、それが持つ属性の値を通して他の概念が連想される、という仕組み

- これを形式的に〈概念, 属性, 値〉、すなわち〈o, a, v〉として有向グラフで表す
- **節点(ノード)** ⟺ 概念、対象、状況、属性値
- **枝(エッジ、リンク)** ⟺ 属性、関係(メタデータとも呼ぶ)
- 例えば「人間 —is-a→ 動物」のように、概念の属性や概念間の関係を表し、全体としてネットワーク構造を成す

#### is-a と part-of

**概念的階層構造**

- is-a関係
- 概念間の上位下位の関係
- 下位 is a 上位
- この関係に基づいて**継承**が行われる

**構造的階層構造**

- part-of関係
- 全体・部分の関係
- 部品 part of 全体

#### 推論: グラフの照合と推移律

- 意味ネットワークは知識を表現するのみで、推論方法までは規定しない
- 通常は**ネットワークの照合**を行う
- 照合では属性の意味に合わせてまとめることもあり、例えば「is-aのis-aはis-a」(推移律)が成立する。part-ofも同様である。他の属性については、その設計上の意味から推移律が成立するかを個別に考える。

例えば「府中市に住み、鳥を飼っている人は誰か?」という問いは、

```
人間 —is-a→ x? —live→ 府中市
              x? —breed→ 鳥
```

というパターンとしてグラフ照合により解かれる。さらに設計者の定義によって属性間の波及も扱える。「東京に住み、鳩が好きな人は誰か?」では、liveがpart-ofを伝わる(府中市に住む=東京に住む)性質や、breedがあれば自動的にlikeも成立する(飼っているなら好きであろう)性質を、設計者が与えておく。

#### 継承と例外

- **継承** とは、is-a関係を通して、否定されない限り他の属性・関係が伝達されること
- 例えば「動物 ←is-a— 鳥」「鳥 —part-of→ 羽」のとき、「動物である」「羽を持つ」はis-a関係を通じて全ての鳥に継承される
- 一方「飛ぶ」については、ダチョウとペンギンのノードで「cannot(飛ぶの否定)」が定義されているため、これらは飛べず、それ以外の鳥は継承により飛べる。canの否定がcannotであることは予め与えておく
- このように継承された値は**例外を含む値**として解釈する
- 鳥は通常飛べるとし、ペンギン・ダチョウなどの例外のみ付け加えればよい
- また継承された値は**暗黙値** とも解釈できる
- 暗黙値とは特に指示がないときに使われる代用値
- 一般に正しいと考えられる性質を暗黙値として定義し例外のみ記述することで、記述量を減らせる

#### 動作の表現

- 静的な記述だけでなく動作も表現したい
- 動作の主格・目的格を属性関係で結んで表す
- 例えば「与える」は「所有する」の主格を変える操作として、前提条件をグラフの照合で、行為をグラフの張り替え規則で表現する
- 具体的には「太郎が花子に花をプレゼントする」を動的にネットワークとして構成する。プレゼントするは与えるの特別な場合なのでis-aで結び、「所有する」部分が成立するかネットワークを参照して確かめ、ネットワークを書き換える。

#### 知識グラフ・URI・RDF

意味ネットワークのエッジ属性の種類を制限し、代わりにグラフ理論の形式的操作を可能としたものが**知識グラフ**である。推論でグラフ操作が必要なため、現状では意味ネットワークと同一視されることが多い。類似概念としてGoogleのナレッジグラフがあり、関係性に意味(属性)を付与した意味検索を実現する。

関連概念として以下がある。

- **URL (Universal Resource Locator)**: リンクにより情報の在処を関連づける。
- **URI (Universal Resource Identifier)**: URLの拡張概念。情報や計算資源の場所に加え、単純に概念(都市名、過去の人名など)を指し示す。URLを含む。
- **RDF (Resource Description Framework)**: URI間の関係(メタデータ)を記述する枠組みであり、まさに意味ネットワークを表現する。`<subject relation object>` の形式をとり、subject・object・relation はいずれもURIである。relationは**述語 (predicate)** とも呼ぶ。

当初ノードは計算機上のテキストにすぎず多様な知識の表現が難しかったが、現在ノードは主にWWW上の知識とリンクで表現され、それを目指してURIやRDFが整備された。subjectを一つ固定すると整理された形式になり、これが次のフレームの発想(ネットワーク形式の知識を概念ごとにまとめる)へとつながる。

#### 長所と短所

- **長所**は、ネットワーク構造で図として見やすいこと、知識の追加が容易なこと、ポインターをたどれるので推論・処理が速いこと
- **短所**は、推論規則があまり強力でなく別途用意が必要なこと、知識の矛盾が発生しうること、一貫性の検証が困難なこと、大規模システムでは複雑さが増すことである

### フレーム（参考）

#### 基本概念

フレームは1974年にM. Minskyによって提唱された。エピソード記憶と関連し、ある場面を思い浮かべるとそれに結びついた時間・空間・事柄が連想されることを表す。整理された知識をフレームという枠組みで表現し、関連する知識・概念間の関係・その事象に結びついた手続きをまとめる。例えば自室を思い浮かべると、「窓」がその部屋の特定の窓を、「ぬいぐるみ」が特定のクマのぬいぐるみを自然に指す、というように特定のものが結びつく。

#### 構造: スロットとファセット

フレームは複数の**スロット**とその**スロット値**からなり、各スロットは任意個数の**ファセット**とその値を持つ。

```
〈frame〉
  〈slot1〉〈slot value1〉
    <facet1-1> <facet-value1-1>
    …
  〈slot2〉〈slot value2〉
    …
```

スロットは属性名、スロット値はその属性値を表す。具体例を示す。

```
Bird                  ; 一般的な「鳥」概念(クラスフレーム)
  wing: Yes           ; 羽を持つ
  lay-egg: Yes        ; 卵を産む
  legs: 2             ; 脚は2本
  flies: yes          ; 飛べる
  size:               ; 不明
  name:               ; 不明

Bird1                 ; ある特定の鳥(インスタンスフレーム)
  wing: Yes
  legs: 2
  flies: yes
  size: 70            ; 体長70mm
  name: warbler       ; 鳥の名前
```

一般的概念を表す**クラスフレーム**と、特定の対象を表す**インスタンスフレーム**がある(一般的な「部屋」ならクラス、あなたの特定の部屋ならインスタンス)。

#### IS-A と Member 関係、継承

フレーム間はIS-A関係とmember関係で結ぶ。専用のスロットを作り、指し示すフレーム名をスロット値に格納する。

- **A —IS-A→ B**: A・Bともクラスフレームで、概念Bが概念Aに含まれる(動物→鳥→カナリアなど)。AKO (a kind of) 関係とも言う。
- **A —member→ B**: Aはクラス、Bはインスタンスで、Bが概念Aの特定対象であることを示す(Instance-of関係とも)。

これらの関係を通じてスロット値が**継承**される。継承方法には以下が設定される。

1. 定義がなければ上位概念の値がそのまま継承される(意味ネットワークと同じ考え)。
2. 上位概念の値が集合(数値区間)なら、下位はその部分集合でなくてはならない。
3. 上位で集合が与えられていても、インスタンスフレームではそのうちの一つでなくてはならない。

2・3では具体的な値を与える必要がある。実際のフレームシステムでは、こうした継承の仕様を**インヘリタンスロール (inheritance role)** という特殊なファセット値で管理する。

#### 多重継承

IS-A・Member関係を複数許すと**多重継承**の問題が生じる。多重継承が望ましい例として、戦闘機は飛行機かつ武器、iPhoneは電話かつ動画再生器かつスケジューラ、などがある。同名スロットを複数のフレームから継承する場合、継承方法に多様性がある。

- 和集合を取る (union inheritance)
- 共通集合を取る (intersection inheritance)
- 一方のみを選択する (selective inheritance)

一般には両方の可能性があり、内容が矛盾する場合はどちらかを優先する。これは応用依存となり、複雑で分かりにくくなる恐れがある。

#### 手続き的知識とデーモン

- スロット値には、宣言的知識だけでなく**手続き的知識**も格納できる。
- スロットへ格納するプログラムや関数を、**付加手続き（attached procedure／procedural attachment）**と呼ぶ。
- スロットにメッセージを送ると手続きが起動するため、オブジェクト指向に近い。
- フレームシステムの実装には、主にLISPが使われた。
- LISPはプログラムをデータとして生成・実行でき、ラムダ式でプログラムを記述できるため、この用途に適している。

**ファセット**はスロットに関する属性・付随情報を記述する領域であり、利用法として以下がある。

- **インヘリタンスロール**: 継承の仕様や制限をシステムがチェックする。
- **データタイプ**: スロットのデータ型をチェック(例: integer)。
- **デーモン (daemon)**: 上記2つもこれで実現できる。

**デーモン**とは、特別な名前のファセットに手続き的知識を格納し、対応するスロット値へのアクセス・変更の前後に手続きを起動する仕組みである(UNIXの常駐デーモンとは別概念だが、「出番を待ち受けて必要なときに動作する」という広い意味では似ている)。代表例は次の三つである。

- **If-added**: スロット値が追加されたときに起動。
- **If-needed**: スロット値を求める(アクセスする)ときに起動。
- **If-removed**: スロット値が消去されたときに起動。

このほか **default**(代用値を計算)、**require**(格納時に値の妥当性を判断、例: 身長は正数)なども付加手続きの一例である。デーモンの利用法としては、値がないときの代用値計算(継承以外。例: 体重不明時に身長と性別から標準値を計算)や、値変更時に関連スロットとの矛盾が起こらないよう設定する、といったものがある。

#### 意味ネットワークとの関係、長所と短所

意味ネットワークのエッジは、主ノードをフレームのノードとし、関係名のスロットを作ってその値に反対側のノードを記載することで表せる。この観点では意味ネットワークの表現はフレームでも可能である。ただしフレームは継承(is-a・member-of関係)に基づく構造化が明確に定義されている点で、知識がより整理されている。

**まとめると**、フレームシステムは典型的な知識を関連するものごとにまとめて表現でき、宣言的知識が中心だがスロットに手続きを格納して手続き的知識にも対応でき、継承により性質の記述をまとめられる。一方で、大規模な知識体系をフレームのみで表現するのは難しく、フレーム間の矛盾管理も困難で利用者に委ねられている。
